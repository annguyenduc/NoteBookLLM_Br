# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Kiến trúc ứng dụng Web Python (STEM Education) & Kỹ thuật Prompt Engineering cho AI Development/Tutoring.
- **Vấn đề / Câu hỏi:** Thiết kế kiến trúc, tối ưu chi phí (ưu tiên miễn phí) cho app quản lý dự án STEM bằng Python, và xây dựng prompt chuẩn để AI hỗ trợ code, học tập từ sách chuyên ngành, hoặc nghiên cứu bảo mật mạng.
- **Cách giải quyết / Logic lõi:** 
  1. **Kiến trúc & Cấu hình Tech Stack (Tối ưu chi phí):**
     ```yaml
     Backend: Flask (nhẹ, dễ học) hoặc FastAPI (hiệu năng cao, async)
     Database: SQLite (dev/local) → PostgreSQL/MySQL (prod, dùng free-tier: Supabase, Neon)
     Frontend: HTML/CSS/JS + Bootstrap/Tailwind (Mobile-first, responsive)
     Storage: Tách file nặng → nhúng link Google Drive/OneDrive hoặc dùng Cloudinary/AWS S3 (free tier)
     Deployment: PythonAnywhere, Render, hoặc Docker + VPS giá rẻ
     ```
  2. **Cấu trúc Prompt Engineering chuẩn cho AI Code/Tutor:**
     ```text
     [Vai trò] Chuyên gia [Lĩnh vực]
     [Bối cảnh & Mục tiêu] ...
     [Yêu cầu chức năng] (User roles, core features, input/output formats)
     [Yêu cầu phi chức năng] (UI/UX, Security, Performance, Cost/Free-tier)
     [Công nghệ đề xuất] ...
     [Ràng buộc & Đạo đức] (Tuân thủ pháp lý, nguồn dữ liệu duy nhất nếu học từ sách)
     [Định dạng đầu ra] ...
     ```
  3. **Prompt học tập từ tài liệu cố định (NotebookLM/LLM):**
     - Ràng buộc cứng: `"Chỉ sử dụng thông tin từ file [Tên sách]. Không dùng nguồn ngoài."`
     - Luồng xử lý: Đi theo cấu trúc Phần/Chương → Tóm tắt → Giải thích khái niệm → Ví dụ mã → Cảnh báo (gotchas) → Câu hỏi kiểm tra hiểu biết.
- **Ghi chú quan trọng:** 
  - **Tối ưu chi phí:** Ưu tiên dịch vụ Free Tier, tách biệt lưu trữ file nặng sang cloud bên thứ 3, dùng ORM để dễ dàng migrate DB sau này.
  - **Phát triển với AI (Cursor/LLM):** Chia nhỏ task (xây MVP trước), yêu cầu AI sinh code theo module, viết unit test, và luôn review code để tránh hallucination. Tuân thủ PEP 8 & cấu trúc thư mục rõ ràng.
  - **Bảo mật & Pháp lý:** Khi nghiên cứu công cụ mạng (ví dụ: WiFi), luôn nhấn mạnh mục đích giáo dục/kiểm thử hợp pháp, có sự cho phép của chủ sở hữu. Card WiFi Killer AX1650i có hạn chế `monitor mode` trên Windows, cần lưu ý khi bắt handshake WPA/WPA2.
