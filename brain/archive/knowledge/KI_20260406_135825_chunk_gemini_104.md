# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Cơ chế tiêu tốn Token của MCP (Model Context Protocol), cấu hình MCP Hub và chiến lược tối ưu chi phí cho Cursor Pro.
- **Vấn đề / Câu hỏi:** Việc tích hợp nhiều công cụ MCP có làm tăng chi phí token không? Cách hoạt động của context trong Cursor và phương án cấu hình để giảm thiểu/maximize chi phí (ưu tiên miễn phí).
- **Cách giải quyết / Logic lõi:** 
  - **Cơ chế Token của MCP:** Mọi định nghĩa công cụ (tool definitions) đều được đóng gói vào "meta-prompt" và gửi đến LLM trong *mỗi lần request*. Do đó, số lượng tool càng nhiều, chi phí token nền (baseline) càng cao, bất kể tool đó có được gọi hay không.
  - **MCP Hub (Tối ưu Context):** Gom nhiều server MCP vào một hub trung tâm, chỉ expose 2 tool (`list-all-tools`, `call-tool`) vào context chính. Giúp giảm mạnh token nền cho các tác vụ thông thường, nhưng đánh đổi bằng độ trễ và chi phí token cao hơn khi AI phải khám phá tool mới.
  - **Cấu hình MCP Hub:**
    ```json
    // .cursor/mcp.json
    { "mcpServers": { "mcp-hub": { "command": "npx", "args": ["mcp-hub-mcp", "--config-file", "~/.cursor/mcp-hub.json"] } } }
    // .cursor/mcp-hub.json
    { "mcpServers": { /* Cấu hình các server MCP con tại đây */ } }
    ```
  - **Quản lý Context trong Cursor:** AI chỉ xử lý những gì được cung cấp: lịch sử chat, `@file`, `@symbol`, code bôi đen, `@docs`. Mỗi lần nhấn Enter, toàn bộ context hiện tại được gửi lại.
  - **Chiến lược Hybrid & BYOK (Miễn phí/Tối ưu):**
    - Dùng BYOK với Google Gemini (Free Tier) cho tác vụ đơn giản hoặc thu thập dữ liệu.
    - Cấu hình BYOK trong `settings.json`:
      ```json
      { "externalServices": [{ "name": "google", "apiKey": "YOUR_GEMINI_API_KEY" }] }
      ```
    - Phân luồng tác vụ: Đơn giản → Gemini/Claude Sonnet; Phức tạp/Debug sâu → Claude Opus/GPT-4o.
    - Query MCP chi tiết (`@GitHub(issue:123)` thay vì `@GitHub`), dùng `@symbol` thay vì `@file`, tắt server MCP không dùng, tạo `.cursorrules` để cố định ngữ cảnh dự án.
- **Ghi chú quan trọng:** 
  - Token được tính trên cả Input (Context + Prompt) và Output. Context càng dài, chi phí càng tăng tuyến tính.
  - MCP Hub không "miễn phí" token mà chỉ "trì hoãn" chi phí: bạn chỉ trả token khám phá khi thực sự cần gọi tool ẩn.
  - Ưu tiên `Ctrl+K` (Edit/Generate) thay vì `Ctrl+L` (Chat) cho chỉnh sửa nhỏ để context tập trung, giảm token thừa.
  - Luôn `Clear Chat` khi chuyển chủ đề để reset context window, tránh lãng phí token cho lịch sử không liên quan.
