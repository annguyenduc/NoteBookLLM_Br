# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Kiến trúc & Tối ưu hóa phát triển Chatbot/Tác nhân AI với Google Gemini API
- **Vấn đề / Câu hỏi:** Làm thế nào để lựa chọn mô hình, cấu hình tham số, quản lý ngữ cảnh và tích hợp kỹ thuật nâng cao (RAG, Agent, Structured Output) nhằm cân bằng giữa tốc độ, chi phí và độ chính xác khi xây dựng ứng dụng AI?
- **Cách giải quyết / Logic lõi:** 
  1. **Lựa chọn mô hình cốt lõi:**
     - `Gemini 2.5 Flash`: Ưu tiên tốc độ (~270 tok/s), độ trễ thấp (0.4s), chi phí rẻ. Phù hợp chatbot thời gian thực, tóm tắt, tác vụ khối lượng lớn.
     - `Gemini 2.5 Pro`: Ưu tiên chất lượng suy luận, độ chính xác cao. Phù hợp phân tích dữ liệu, mã hóa phức tạp, toán học. Độ trễ cao hơn, chi phí đắt hơn.
  2. **Cấu hình tham số & Chế độ đặc biệt:**
     - *Quy trình tư duy (Thinking):* Dùng `thinkingBudget` điều khiển độ sâu suy luận. `-1` (động), `0` (tắt/ưu tiên tốc độ), `128-32768` (cố định). Bật `includeThoughts: true` để debug logic nội bộ.
     - *Đầu ra có cấu trúc (Structured Output):* Ép mô hình trả về JSON/XML để tích hợp hệ thống.
       ```json
       {
         "generation_config": {
           "response_mime_type": "application/json",
           "response_schema": {
             "type": "OBJECT",
             "properties": { "intent": {"type": "STRING"}, "confidence": {"type": "NUMBER"} }
           }
         }
       }
       ```
     - *Hướng dẫn hệ thống (System Instructions):* Đặt persona, tone, ràng buộc ngay khi khởi tạo `GenerativeModel`. Tách biệt khỏi prompt người dùng cuối.
     - *Chế độ Agent:* Cho phép mô hình tự động gọi công cụ/API bên ngoài, thực thi đa bước. Cần kiểm soát quyền truy cập (user approval).
  3. **Quản lý ngữ cảnh & Trạng thái (State Management):**
     - LLM mặc định `stateless`. Dùng `Context Caching` để giảm chi phí token cho ngữ cảnh dài.
     - Kỹ thuật: `Sliding Window` (giữ tin nhắn gần nhất), `Specialized Fields` (lưu metadata vào DB ngoài), `Smart Write/Read` (truy xuất động), `Dialogue State Tracking (DST)` cho chatbot định hướng mục tiêu.
  4. **Tích hợp RAG & Framework:**
     - Dùng Vertex AI Search hoặc tự xây dựng pipeline RAG (Vector DB + Re-ranker) để giảm hallucination.
     - Framework: `LlamaIndex` (tối ưu RAG/dữ liệu), `LangGraph` (workflow đa tác nhân có trạng thái), `Composio` (kết nối API bên ngoài).
  5. **Prompt Engineering & Xử lý giới hạn:**
     - Chia nhỏ tác vụ phức tạp (iterative prompting) do giới hạn đầu ra (~65k tokens).
     - Meta-prompting: Gán vai trò, cung cấp ví dụ (few-shot), chỉ định định dạng rõ ràng.
     - Điều chỉnh `temperature` (0.2 cho logic/mã, 1.5-2 cho sáng tạo).
- **Ghi chú quan trọng:**
  - ⚠️ **Giới hạn đầu ra:** Lỗi "kiểm tra internet" thường là do vượt quá giới hạn token đầu ra hoặc timeout, không phải lỗi mạng. Luôn chia nhỏ yêu cầu tạo mã/dữ liệu lớn.
  - 💡 **Tối ưu chi phí:** Kết hợp `Context Caching` + chọn mô hình Flash cho tác vụ thường xuyên + dùng `thinkingBudget` thấp cho truy vấn đơn giản.
  - 🛡️ **Giảm Hallucination:** Bắt buộc dùng RAG cho dữ liệu thực tế, áp dụng `response_schema` để ràng buộc định dạng, và thiết lập ngưỡng tin cậy/cơ chế human-in-the-loop.
  - 🔒 **Bảo mật Agent:** Chế độ Agent có thể thực thi hành động hệ thống. Luôn yêu cầu xác nhận người dùng cho các thao tác sửa đổi (trừ chế độ `yolo` đã cấu hình sẵn).
