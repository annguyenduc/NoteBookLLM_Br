# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Đánh giá phần cứng Yolo UNO (ESP32-S3), nền tảng Google Antigravity IDE và chiến lược lựa chọn Firmware cho giáo dục STEM & phát triển AI/IoT.
- **Vấn đề / Câu hỏi:** 
  1. Có nên mua kit xe Arduino giá 250k để dạy học/nghiên cứu?
  2. Khả năng tích hợp Google Antigravity với Yolo UNO.
  3. Nên giữ firmware gốc OhStem hay cài lại firmware chuẩn (Arduino/ESP-IDF)?
- **Cách giải quyết / Logic lõi:** 
  - **Đánh giá Kit 250k:** Giá thấp hơn giá linh kiện rời (~350k+). Thường dùng động cơ TT (vàng) thay vì N20, độ rơ cơ khí cao → khó dạy thuật toán PID chính xác. MCU thường hàn chết Atmega328 → không tương thích trực tiếp với Yolo UNO. Chỉ nên mua để "mổ xẻ" tham khảo code mẫu & bố trí mạch.
  - **Kiến trúc Yolo UNO:** Chip ESP32-S3 (2 lõi Xtensa LX7 240MHz, hỗ trợ tập lệnh vector cho AI/DSP), 16MB Flash + 8MB PSRAM (bắt buộc cho AI/OTA/LittleFS), 12 cổng Grove (4A/4D/4I2C). Logic hoạt động 3.3V (cần lưu ý khi ghép module 5V cũ).
  - **Google Antigravity IDE:** Nền tảng phát triển "Agent-first" (AI tự chủ lập kế hoạch, sinh code, đọc datasheet tạo driver, tự sửa lỗi biên dịch). Yêu cầu môi trường mở (PlatformIO/ESP-IDF/Arduino CLI) để hoạt động tối ưu.
  - **Chiến lược Firmware:**
    - *Giữ Firmware OhStem:* Dành cho giáo dục STEM, người mới. Ưu tiên kéo thả (Blockly), tương thích module Grove ngay lập tức, ổn định. Nhược điểm: Hiệu năng thấp, khó tùy biến sâu, không tương thích Antigravity.
    - *Cài lại Firmware Chuẩn (Arduino/ESP-IDF/MicroPython gốc):* Dành cho nghiên cứu, AI, IoT chuyên sâu. Bắt buộc để dùng Antigravity, khai thác tối đa 16MB Flash/8MB PSRAM, hỗ trợ FreeRTOS & TensorFlow Lite.
  - **Quy trình kỹ thuật quan trọng:**
    - Backup firmware gốc: `esptool.py --chip esp32s3 --port <PORT> read_flash 0x0 0x1000000 yolo_uno_backup.bin`
    - Nạp code qua Antigravity/PlatformIO: `pio run --target upload`
    - Chế độ Boot thủ công (khi nạp tự động thất bại): Giữ nút `BOOT` → Nhấn & thả `RESET` → Thả `BOOT`.
- **Ghi chú quan trọng:** 
  - Luôn backup firmware gốc trước khi flash lại để có thể khôi phục nhanh cho mục đích giảng dạy.
  - Khi dùng Antigravity, bắt buộc cung cấp file `CONTEXT.md` chứa thông tin chip (ESP32-S3 N16R8) và bản đồ chân Grove cụ thể của Yolo UNO để AI sinh code chính xác, tránh lỗi "ảo giác" phần cứng.
  - Quyết định firmware phụ thuộc hoàn toàn vào đối tượng: Học sinh/K-12 → Giữ OhStem. Kỹ sư/Nghiên cứu AI → Flash chuẩn + dùng Antigravity.
