# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Triển khai ứng dụng Flask Production, Kiến trúc tự động hóa PyQt6+Selenium, và Lộ trình phát triển Python Backend/eCommerce.
- **Vấn đề / Câu hỏi:** 
  1. Làm thế nào để chuyển ứng dụng Flask từ môi trường phát triển sang production an toàn và hiệu quả?
  2. Cấu trúc luồng hoạt động của ứng dụng desktop tự động hóa (PyQt6 + Selenium + API)?
  3. Lộ trình học tập và cấu trúc code chuẩn cho vị trí Python Developer (Backend, Automation, eCommerce)?
- **Cách giải quyết / Logic lõi:** 
  ### 1. Triển khai Flask lên Production
  - **Web Server:** Thay thế `app.run()` bằng WSGI server (Gunicorn/uWSGI).
    ```bash
    # Procfile hoặc lệnh chạy
    web: gunicorn app:app --workers 4 --bind 0.0.0.0:8000
    ```
  - **Reverse Proxy:** Dùng Nginx/Apache phía trước để xử lý static files, SSL/TLS, caching và cân bằng tải.
  - **Cơ sở dữ liệu:** Chuyển từ SQLite sang PostgreSQL/MySQL. Sử dụng `Flask-Migrate` (Alembic) để quản lý schema an toàn.
  - **Bảo mật & Cấu hình:** 
    - `app.config['DEBUG'] = False`
    - Lưu `SECRET_KEY`, `DATABASE_URL` trong biến môi trường (`.env`), không hardcode.
    - Áp dụng HTTPS, chống XSS/CSRF/SQL Injection, quản lý JWT an toàn.
  - **CI/CD & Logging:** Tự động hóa deploy qua GitHub Actions, tích hợp logging tập trung (Sentry/ELK).

  ### 2. Kiến trúc Ứng dụng Tự động hóa (PyQt6 + Selenium)
  - **Luồng xử lý bất đồng bộ:** Sử dụng `QThread` (`WorkerThread`) để chạy tác vụ nặng, tránh treo giao diện chính (`MainWindow`).
  - **Phân tách module:**
    - `main.py`: Điều phối UI, đọc/ghi Excel (`openpyxl`), khởi tạo thread.
    - `Bot.py`: Dùng Selenium điều khiển Chrome để điền form, kết hợp `Faker` tạo dữ liệu giả.
    - `ChangePasswordBot.py`: Dùng `requests.Session` gửi HTTP trực tiếp đến API (nhanh & nhẹ hơn Selenium).
    - `TempMailClient.py`: Wrapper gọi API dịch vụ mail tạm thời để nhận/đọc mã xác thực.
  - **Cơ chế dừng an toàn:** Dùng cờ `self.is_running` trong vòng lặp của `WorkerThread` để ngắt tác vụ khi người dùng nhấn "Dừng".

  ### 3. Cấu trúc Prompt & Model cho Ứng dụng Quản lý Dự án (Flask)
  - **Database Models (SQLAlchemy):** Thiết kế quan hệ 1-N và 1-1 rõ ràng.
    ```python
    # Ví dụ quan hệ Submission - Grade (1-1)
    class Submission(db.Model):
        id = db.Column(db.Integer, primary_key=True)
        grade = db.relationship('Grade', backref='submission', uselist=False)
    ```
  - **Xác thực & Phân quyền:** Dùng `Flask-Login`, `@login_required`, kiểm tra `current_user.role` để điều hướng dashboard.
  - **Upload File:** Form HTML bắt buộc `enctype="multipart/form-data"`, lưu file theo cấu trúc `uploads/<user_id>/<project_id>/` để tránh trùng tên.
  - **UI/UX:** Kế thừa template `base.html`, tích hợp Bootstrap 5 qua CDN, hiển thị flash messages.

  ### 4. Lộ trình học Python Developer (Theo JD)
  1. **Nền tảng:** Python nâng cao (OOP, Decorators, Asyncio), Git, Cấu trúc dữ liệu & Giải thuật.
  2. **Backend Web:** Django/FastAPI/Flask, RESTful API, ORM (SQLAlchemy).
  3. **Database & Queue:** PostgreSQL/MySQL, Redis (cache/broker), Celery/RabbitMQ (xử lý nền bất đồng bộ).
  4. **Automation & Integration:** Selenium/Scraping, tích hợp API bên thứ 3 (Shopify, Amazon, OAuth2).
  5. **Domain Knowledge:** Hiểu nghiệp vụ ERP, CRM, OMS, Data Platform cho eCommerce.
- **Ghi chú quan trọng:** 
  - Tuyệt đối không dùng `app.run()` hoặc SQLite cho môi trường production nhiều người dùng đồng thời.
  - Trong PyQt6, mọi tác vụ I/O hoặc network phải chạy trên thread riêng để tránh UI `Not Responding`.
  - Ưu tiên dùng HTTP Request (`requests`) thay vì Selenium khi tương tác với API có sẵn để tối ưu hiệu suất và tài nguyên.
  - Luôn quản lý dependency bằng `requirements.txt` hoặc `Poetry`, và dùng biến môi trường cho mọi thông tin nhạy cảm.
