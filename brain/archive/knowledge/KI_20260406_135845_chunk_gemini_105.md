# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Quản lý giới hạn công cụ MCP trong Cursor, tối ưu hóa cửa sổ ngữ cảnh (Context Window) và quy trình phát triển AI-driven tiết kiệm chi phí.
- **Vấn đề / Câu hỏi:** 
  1. Cursor áp dụng giới hạn cứng ~80 công cụ MCP (cảnh báo từ ~40) do bão hòa token ngữ cảnh và gánh nặng nhận thức của LLM.
  2. Làm thế nào để vượt/quản lý giới hạn này, đồng thời xây dựng ứng dụng tối ưu token và chi phí gói Pro?
- **Cách giải quyết / Logic lõi:** 
  **1. Chiến lược xử lý giới hạn MCP & Ngữ cảnh:**
  - *Chính thức:* Bật/tắt toàn bộ server MCP. Tính năng quản lý chi tiết từng tool đang được Cursor phát triển.
  - *Offload ngữ cảnh (Khuyên dùng):* Dùng CLI `vibe-tools` để ủy thác tác vụ nặng (phân tích toàn bộ repo, lập kế hoạch) cho AI bên ngoài (Gemini/Perplexity) qua terminal, giữ context Cursor gọn nhẹ.
  - *Patch máy khách (Rủi ro cao):* Sửa file `resources/app/out/vs/workbench/workbench.desktop.main.js`, chèn logic override hàm giới hạn token:
    ```javascript
    async getEffectiveTokenLimit(e) {
      if(e.modelName && e.modelName.includes('claude-3.7')) return 200000;
      // ... logic gốc tiếp tục
    }
    ```
  - *Tối ưu ngữ cảnh:* Ưu tiên `@code` > `@file` > `@folder`. Dùng `.cursor/rules` để mã hóa quy tắc dự án, thay thế MCP tùy chỉnh. Tận dụng `@Docs`/`@Web` cho tài liệu động.

  **2. Quy trình làm việc & Tối ưu chi phí:**
  - *Công cụ AI:* `Ctrl+L` (Chat/Hỏi đáp), `Ctrl+I` (Agent/Đa tệp & Terminal), `Ctrl+K` (Inline Edit/Refactor nhanh), `Tab` (Autocomplete).
  - *Chiến lược tiết kiệm:* Chia nhỏ prompt, dùng model rẻ cho tác vụ đơn giản, kiểm tra mã sau mỗi bước nhỏ.
  - *Debug/Refactor:* Dán chính xác lỗi từ Console, dùng `console.log` có chủ đích để trace, refactor từng hàm riêng biệt thay vì yêu cầu chung chung.
- **Ghi chú quan trọng:** 
  - Giới hạn 80 tool là cơ chế bảo vệ hiệu suất, không phải lỗi. Tư duy đúng: "Quản lý ngữ cảnh chính xác" thay vì "Phá giới hạn".
  - Patch `CursorPlus` dễ bị ghi đè khi cập nhật, vi phạm ToS và có thể gây mất ổn định. Chỉ dùng cho môi trường thử nghiệm.
  - File `.cursorrules` giúp AI tự động tuân thủ convention, giảm đáng kể số lượng prompt lặp lại và token tiêu thụ.
  - Git là "lưới an toàn" bắt buộc: Commit thường xuyên để rollback nếu AI sinh mã sai, tránh lãng phí token sửa lỗi.
