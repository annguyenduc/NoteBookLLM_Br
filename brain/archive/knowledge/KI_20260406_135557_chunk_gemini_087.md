# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Lập trình Agent trong Minecraft MakeCode để xây dựng cấu trúc hình học (Mặt trời dạng lục giác + tia nắng) đồng bộ mặt phẳng và tương thích chế độ Block.
- **Vấn đề / Câu hỏi:** 
  1. Lõi và tia nắng bị lệch mặt phẳng (trục Z/Y không khớp) do dùng sai hướng đặt khối (`place(FORWARD)` vs `place(BACK)`).
  2. Lỗi cú pháp khai báo biến nhiều dòng trên cùng một hàng không tương thích với trình soạn thảo Block của MakeCode.
  3. Yêu cầu chuyển đổi linh hoạt giữa xây dựng thẳng đứng (trục Y) và nằm ngang (mặt phẳng X-Z).
- **Cách giải quyết / Logic lõi:** 
  - **Đồng bộ mặt phẳng:** Thống nhất hướng đặt khối. Bản dọc dùng `agent.place(FORWARD)`, bản ngang dùng `agent.place(DOWN)`. Tuyệt đối tránh `place(BACK)` gây lệch trục.
  - **Tính toán tọa độ neo (Anchor) bằng toán học:** Không lưu điểm trong lúc xây. Sau khi hoàn thành lõi, dùng `buildCenter` làm gốc tuyệt đối để tính 12 điểm viền:
    ```typescript
    const R = Math.floor(totalWidth / 2);
    const corner_x = Math.floor(canh / 2);
    const corner_y = R - corner_x;
    
    // Ví dụ tính điểm neo (trục Y cho bản dọc, trục Z cho bản ngang)
    anchor_N = positions.add(buildCenter, pos(0, R, 0));
    anchor_NE = positions.add(buildCenter, pos(corner_x, corner_y, 0));
    ```
  - **Sửa lỗi cú pháp MakeCode:** Tách khai báo biến `Position` thành từng dòng riêng và gán `null` để Block Editor nhận diện:
    ```typescript
    let anchor_N: Position = null;
    let anchor_S: Position = null;
    // ...
    ```
  - **Hàm vẽ tia chuẩn hóa (loại bỏ boilerplate):**
    ```typescript
    function buildRay(length: number, x_step: number, y_step: number) {
        for (let i = 0; i < length; i++) {
            if (x_step > 0) agent.move(RIGHT, x_step);
            if (x_step < 0) agent.move(LEFT, -x_step);
            if (y_step > 0) agent.move(UP, y_step);
            if (y_step < 0) agent.move(DOWN, -y_step);
            agent.place(FORWARD); // Giữ nguyên mặt phẳng với lõi
        }
    }
    ```
- **Ghi chú quan trọng:** 
  - Luôn dùng `buildCenter` làm gốc tọa độ cố định. Không phụ thuộc vào vị trí cuối cùng của Agent sau khi xây lõi để tránh sai số tích lũy.
  - Cấu trúc code nên chia rõ 3 pha: `Căn giữa & Thiết lập` → `Xây lõi` → `Tính toán neo & Vẽ tia`.
  - Khi chuyển sang mặt phẳng ngang (X-Z), thay thế trục `UP/DOWN` bằng `FORWARD/BACK` và đổi `place(FORWARD)` thành `place(DOWN)`.
