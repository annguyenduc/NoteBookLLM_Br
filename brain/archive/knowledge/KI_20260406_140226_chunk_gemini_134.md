# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Triển khai MCP (Model Context Protocol) cho môi trường Local/LAN & Kiến trúc hệ thống "Vietnamese Class Manager".
- **Vấn đề / Câu hỏi:** Xác định tính cần thiết của MCP trong mạng LAN, cách xây dựng MCP server cục bộ, và lựa chọn loại MCP phù hợp để tích hợp AI Agent vào dự án quản lý lớp học.
- **Cách giải quyết / Logic lõi:** 
  - **Điều kiện áp dụng MCP:** Chỉ cần thiết khi muốn AI Agent (Cursor, LLM) tương tác thông minh hoặc tự động hóa tác vụ qua ngôn ngữ tự nhiên. Nếu chỉ quản lý thủ công qua GUI trên LAN → Không cần MCP.
  - **Giao thức vận chuyển (Transport):** Ưu tiên `stdio` (Standard Input/Output) cho môi trường local. AI Client giao tiếp trực tiếp qua luồng stdin/stdout của process Python, không cần mở port mạng.
  - **Mẫu triển khai MCP Server (Python SDK):**
    ```python
    from mcp.server import Server
    from mcp.server.stdio import stdio_server
    import asyncio

    app = Server("VCM_MCP_Server")

    @app.tool()
    async def get_attendance_report(class_id: str, start_date: str, end_date: str):
        # Gọi trực tiếp Service/Repository layer của dự án
        # report = attendance_service.get_report(class_id, start_date, end_date)
        return {"success": True, "data": "..."}

    async def main():
        async with stdio_server() as (read_stream, write_stream):
            await app.run(read_stream, write_stream)

    if __name__ == "__main__":
        asyncio.run(main())
    ```
  - **Kiến trúc MCP đề xuất cho dự án:**
    1. `Database MCP`: Wrapper cho `Repository` layer, cho phép AI truy vấn SQLite (điểm danh, học sinh, bài tập).
    2. `Document MCP`: Quản lý tài liệu local hoặc tích hợp Google Drive.
    3. `Task/Assignment MCP`: Tự động hóa việc giao, theo dõi tiến độ và chấm bài tập.
    4. `Context MCP`: Lưu trữ ngữ cảnh lịch sử tương tác cá nhân hóa cho giáo viên/học sinh.
  - **Kiến trúc nền tảng dự án:** Clean Architecture + Service Layer Pattern. Cơ chế phân tách DB SQLite riêng cho từng giáo viên (`teacher_databases/`). Đã hoàn thiện hệ thống Migration (`LegacyDataReader` → `DataTransformer` → `MigrationRunner`) và áp dụng TDD.
- **Ghi chú quan trọng:** 
  - Tận dụng lại `Service` và `Repository` hiện có khi viết MCP tool để đảm bảo logic nghiệp vụ nhất quán, tránh duplicate code.
  - Cấu hình Cursor IDE: `Settings > Features > MCP > Add New` → Transport: `stdio` → Command: `python /path/to/server.py`.
  - Bảo mật & Phân quyền: MCP server chạy cục bộ nhưng vẫn cần validate input kỹ và áp dụng cơ chế phân quyền (Admin/Teacher/Student) trước khi expose tool cho AI, đặc biệt khi thao tác ghi/xóa dữ liệu.
