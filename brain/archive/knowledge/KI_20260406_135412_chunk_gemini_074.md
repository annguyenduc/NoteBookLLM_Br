# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Kiến trúc và chiến lược xây dựng công cụ chuyển đổi mã Python sang định dạng JSON khối lệnh (Blockly) cho nền tảng `app.ohstem.vn`.
- **Vấn đề / Câu hỏi:** Làm thế nào để "dịch ngược" mã Python (text-based) sang cấu trúc JSON của Blockly một cách chính xác, khả thi và dễ bảo trì, khi bản chất chuyển đổi là bất đối xứng và nhiều cú pháp Python không có khối tương đương?
- **Cách giải quyết / Logic lõi:** 
  - **Kiến trúc 3 lớp tách biệt:** `Parser` (dùng `ast` module) → `Mapper` (ánh xạ đệ quy AST sang cấu trúc trung gian Python dict) → `Serializer` (xuất chuỗi JSON chuẩn Blockly).
  - **Phân tích cấu trúc bằng AST:** Không xử lý chuỗi văn bản. Dùng `ast.parse()` để lấy Cây Cú pháp Trừu tượng, loại bỏ chi tiết bề mặt, chỉ giữ logic.
  - **Ánh xạ đệ quy (Mapper):** Duyệt cây AST, mỗi loại nút (`ast.Assign`, `ast.For`, `ast.If`, `ast.Call`) được chuyển thành `opcode`, `fields`, `inputs` của Blockly. Xử lý riêng biệt `statement` (khối nối trên/dưới) và `expression` (khối trả giá trị).
  - **Quản lý API bằng cấu hình động:** Dùng tệp `api_map.json` để ánh xạ các hàm OhStem sang khối lệnh, tránh hardcode, dễ mở rộng khi thêm thiết bị mới.
  - **Tạo khối tùy chỉnh (Blockly):** Định nghĩa UI trong `definition.js`, viết logic sinh mã Python trong `generator.js` thông qua `pythonGenerator.forBlock`.
  ```python
  # 1. Phân tích AST (Parser)
  import ast
  tree = ast.parse("x = 10\nfor i in range(x): print('Hello')")
  
  # 2. Cấu hình API động (api_map.json)
  {
    "yolobit.led_show": {
      "opcode": "yolobit_led_show_string",
      "args_to_inputs": {"0": "TEXT"}
    }
  }
  
  # 3. Logic Generator.js cho khối tùy chỉnh
  pythonGenerator.forBlock['yolobit_led_show_string'] = function(block, generator) {
    const text = block.getFieldValue('TEXT');
    return `yolobit.led_show("${text}")\n`;
  };
  ```
- **Ghi chú quan trọng:** 
  - Chuyển đổi Python → Blockly là bài toán phân tích & tái cấu trúc, **không phải dịch 1-1**. Nhiều cấu trúc (list comprehension, lambda, try/except, class) không hỗ trợ ở v1.0.
  - Cấu trúc JSON đầu ra phải tuân thủ chuẩn Blockly: `blocks` chứa `opcode`, `fields`, `inputs`, `next`, `parent`, `topLevel`. ID biến và khối phải duy nhất.
  - Ưu tiên **xử lý lỗi rõ ràng** khi gặp nút AST không hỗ trợ (dừng chuyển đổi + báo dòng/lỗi) thay vì cố sinh mã sai.
  - Blockly là **"nguồn chân lý" (source of truth)**. Chỉnh sửa trực tiếp trên mã Python sẽ bị mất khi quay lại chế độ khối.
  - Lộ trình triển khai: Reverse-engineer file JSON mẫu → Xây dựng Parser/Mapping core → Tích hợp API dictionary → Xử lý lỗi & đóng gói CLI/Web UI.
