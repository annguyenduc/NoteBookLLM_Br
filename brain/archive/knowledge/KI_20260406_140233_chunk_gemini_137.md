# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Xây dựng & Tối ưu hóa Model Context Protocol (MCP) trong Cursor IDE.
- **Vấn đề / Câu hỏi:** Làm thế nào để định hình và quản lý ngữ cảnh (context) cho Cursor AI hiểu sâu kiến trúc, lịch sử và quy ước của dự án, nhằm giảm thiểu ảo giác (hallucination), tăng độ chính xác khi sinh mã/gỡ lỗi và tối ưu tài nguyên tính toán?
- **Cách giải quyết / Logic lõi:** 
  1. **Cấu hình file `.cursor/context.json`** (đặt ở thư mục gốc dự án):
     ```json
     {
       "files": ["README.md", "config.json", "main.py", "BAO_CAO_DU_AN.md"],
       "directories": [
         { "path": "app/", "priority": "high" },
         { "path": "data/", "priority": "medium" },
         { "path": "tests/", "priority": "low" }
       ],
       "exclude": ["app/database/legacy_models.py", "tests/legacy_tests/"],
       "focus": ["app/services/teacher_service.py"]
     }
     ```
  2. **Chuẩn hóa "nguyên liệu" ngữ cảnh:**
     - **Docstrings & Type Hints:** Mô tả rõ `Args`, `Returns`, `Raises` cho hàm/lớp. Dùng type hinting triệt để để AI hiểu luồng dữ liệu.
     - **Tài liệu cấp cao:** `README.md`, báo cáo kiến trúc, nhật ký lỗi phải có cấu trúc: `Vấn đề -> Giải pháp -> Tác động`.
     - **Mã nguồn sạch:** Tuân thủ PEP 8, đặt tên file/module mô tả đúng chức năng, tách biệt rõ ràng các tầng (Clean Architecture, Service Layer).
  3. **Quy trình tương tác & "Huấn luyện" AI:**
     - Dùng `@mention` để ghim file cụ thể vào prompt, ép AI tham chiếu đúng nguồn.
     - Dùng `Ctrl/Cmd + K` để yêu cầu AI giải thích, refactor hoặc viết test trực tiếp trên vùng code đang chọn.
     - **Phản hồi sửa lỗi:** Khi AI sai, cung cấp ngữ cảnh chính xác: `"Bạn nhầm ở X. Theo @file.md, đúng là Y. Hãy ghi nhớ cho lần sau."`
     - Cập nhật động trường `"focus"` trong `context.json` mỗi khi chuyển module/tính năng làm việc.
- **Ghi chú quan trọng:**
  - MCP trong Cursor là **phương pháp luận quản lý ngữ cảnh**, không phải plugin hay chế độ riêng biệt.
  - AI tự động quét codebase, nhưng `context.json` giúp nó ưu tiên tài nguyên vào phần quan trọng nhất, loại bỏ nhiễu (legacy, log, temp files).
  - Duy trì MCP là quá trình liên tục: cập nhật tài liệu/docstring mỗi khi có thay đổi kiến trúc hoặc sửa lỗi lớn.
  - Áp dụng thực tế cho dự án: Tận dụng tài liệu Clean Architecture, cơ chế DB riêng cho giáo viên, và nhật ký sửa lỗi (BUG Double Hashing, Filter, DB path) làm nền tảng ngữ cảnh để AI hỗ trợ chính xác.
