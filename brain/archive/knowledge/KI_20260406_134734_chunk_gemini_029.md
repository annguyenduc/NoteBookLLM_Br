# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Khắc phục lỗi treo giao diện "About Machine", mất kết nối Mainboard và xung đột Firmware trên máy in 3D Elegoo Neptune 4.
- **Vấn đề / Câu hỏi:** Máy in bị đơ màn hình cảm ứng, IP trống, cổng LAN không sáng, treo ở màn hình "About" sau khi cập nhật UI. Nguyên nhân cốt lõi do lệch phiên bản Firmware giữa Màn hình (UI) và Bo mạch chủ (Mainboard), hoặc lỗi boot hệ điều hành Klipper trên module eMMC.
- **Cách giải quyết / Logic lõi:** 
  1. **Chuẩn bị USB cập nhật:** Format `FAT32`, `Allocation unit size: 4096 bytes`. Tạo thư mục `ELEGOO_UPDATE_DIR` tại root, chứa các file cấu hình và gói cài đặt.
  2. **Kích hoạt cập nhật cưỡng bức (Force Update):** Nếu máy không tự quét thư mục, đổi tên file thực thi:
     ```bash
     # Đổi tên file kích hoạt và đặt tại root USB
     ELEGOO_APP_factory.deb -> Neptune4_Elegoo_Update.tst
     ```
  3. **Bypass lỗi cảm ứng:** Cắm chuột USB vào cổng máy in để điều khiển giao diện Linux, thoát màn hình "About" và nhấn `[Confirm]` để tiến hành flash Mainboard.
  4. **Cập nhật Firmware màn hình (khi đơ hoàn toàn):** Tháo nắp lưng màn hình, cắm thẻ microSD chứa firmware vào khe nội bộ trên bo mạch màn hình. Khởi động lại máy, bootloader sẽ tự động nạp trong ~90s mà không cần tương tác cảm ứng.
  5. **Chẩn đoán Mainboard qua mạng:** Cắm cáp LAN, quét mạng tìm thiết bị có hostname `neptune4`, `mks` hoặc `armbian`. Nếu có IP, truy cập giao diện web `Fluidd`/`Mainsail` để upload và update firmware trực tiếp.
  6. **Khắc phục lỗi eMMC (Boot failure):** Nếu không nhận IP và đèn LAN không sáng, hệ điều hành trên eMMC có thể bị corrupt. Cần tháo module, dùng USB-to-eMMC Adapter kết nối với PC để nạp lại OS Linux/Klipper từ bên ngoài (tương tự ghost ổ cứng).
- **Ghi chú quan trọng:** 
  - Neptune 4 chạy kiến trúc Linux nhúng (Klipper), bắt buộc cập nhật đồng bộ cả Mainboard và UI. Chỉ update UI sẽ gây treo do giao thức giao tiếp không khớp.
  - Luôn rút USB/thẻ nhớ ngay sau khi cập nhật thành công để tránh vòng lặp flash firmware khi khởi động lại.
  - Sau khi flash firmware, cấu hình máy bị reset, bắt buộc thực hiện lại quy trình cân bàn (Bed Leveling).
  - Kiểm tra công tắc chuyển đổi điện áp nguồn (đảm bảo ở mức `230V` tại Việt Nam) trước khi kết luận lỗi phần cứng/Mainboard.
