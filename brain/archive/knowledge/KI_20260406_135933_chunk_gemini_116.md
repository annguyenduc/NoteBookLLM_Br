# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Triển khai LLM cục bộ trên phần cứng hạn chế & Thiết kế Máy chủ MCP cho hệ sinh thái khép kín (MakeCode Minecraft Education).
- **Vấn đề / Câu hỏi:** Làm thế nào để chạy mô hình context 128k trên GPU 4GB VRAM? Và làm sao xây dựng MCP Server cho nền tảng không cung cấp API công khai (MCEE)?
- **Cách giải quyết / Logic lõi:** 
  1. **Tối ưu LLM cục bộ:** Sử dụng `llama.cpp` với mô hình lượng tử hóa GGUF (Q4_K_M). Tận dụng cơ chế offload GPU (`-ngl`) và YaRN scaling để mở rộng context vượt ngưỡng huấn luyện gốc.
  2. **Kiến trúc MCP cho hệ thống khép kín:** Do MCEE không cho phép kết nối API ngoài, chuyển sang mô hình **Tạo mã & Điều phối nội dung gián tiếp**. MCP Server đóng vai trò trợ lý sinh code TypeScript/Extension và kế hoạch bài giảng; người dùng tự sao chép/dán vào môi trường sandbox của game.
  3. **Code & Cấu hình cốt lõi:**
     ```bash
     # Lệnh llama.cpp tối ưu cho GTX 1650 & context dài
     ./main -m model.gguf -ngl 999 -c 131072 --rope-scaling yarn --rope-scale 3.2 --yarn-orig-ctx 40960 -t 8 --temp 0.7 --top-p 0.8 --jinja
     ```

- **Ghi chú quan trọng:** 
  - ⚠️ **VRAM & Context:** 4GB VRAM không đủ cho KV cache của context 128k. Nếu gặp lỗi OOM, giảm tham số `-c` xuống `8192` hoặc `16384`. `llama.cpp` tự động fallback phần layer thừa sang RAM hệ thống.
  - 🔒 **Bảo mật MCP:** Giao thức yêu cầu Host App phải có cơ chế **phê duyệt người dùng (user consent)** trước khi gọi tool. Kiến trúc gián tiếp của MCEE (sinh mã -> người dùng kiểm tra -> dán vào game) hoàn toàn tuân thủ nguyên tắc bảo mật này.
  - 🛠️ **Framework & Lộ trình:** Khuyến nghị dùng `FastMCP` (Python) thay vì `mcp.server` gốc để dễ quản lý production. Lộ trình phát triển: PoC (sinh code hợp lệ) → MVP (gỡ lỗi/giải thích) → Trao quyền (tạo block/kế hoạch bài) → Cộng đồng.
