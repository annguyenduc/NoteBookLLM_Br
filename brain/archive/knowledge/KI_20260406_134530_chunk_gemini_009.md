# TÓM TẮT TRI THỨC KỸ THUẬT

### KI 1: Thuật toán tối ưu duyệt mảng (Python)
- **Chủ đề chính:** Đếm số phần tử lớn hơn tất cả phần tử bên phải.
- **Vấn đề / Câu hỏi:** Tìm số ngôi nhà nhìn thấy biển (cao hơn mọi nhà bên phải) với độ phức tạp tối ưu.
- **Cách giải quyết / Logic lõi:** Duyệt ngược từ phải sang trái ($O(N)$), dùng biến `max_height` lưu giá trị lớn nhất đã gặp. Nếu phần tử hiện tại `> max_height` thì đếm và cập nhật `max_height`.
  ```python
  count = 0
  current_max = -1
  for i in range(n - 1, -1, -1):
      if heights[i] > current_max:
          count += 1
          current_max = heights[i]
  ```
- **Ghi chú quan trọng:** Tránh duyệt lồng nhau $O(N^2)$. Điều kiện `>` (không dùng `>=`) đảm bảo nhà bằng chiều cao max bên phải sẽ bị che khuất.

### KI 2: Quản lý đường dẫn tệp tin trong Python
- **Chủ đề chính:** Đường dẫn tuyệt đối vs Tương đối & Thư viện `pathlib`.
- **Vấn đề / Câu hỏi:** Code dễ lỗi `FileNotFoundError` khi chuyển môi trường do dùng đường dẫn tuyệt đối cứng.
- **Cách giải quyết / Logic lõi:** Ưu tiên đường dẫn tương đối kết hợp `pathlib` để tự động xử lý dấu phân cách (`/` hoặc `\`) và tương thích đa nền tảng.
  ```python
  from pathlib import Path
  base_path = Path(__file__).parent
  data_file = base_path / "data" / "results.csv"
  ```
- **Ghi chú quan trọng:** `pathlib` an toàn hơn việc cộng chuỗi thủ công. Giúp dự án dễ dàng chia sẻ và chạy trên máy khác mà không cần chỉnh sửa đường dẫn.

### KI 3: Phân biệt tên gói & module `scikit-learn`
- **Chủ đề chính:** Quy chuẩn đặt tên thư viện Machine Learning Python.
- **Vấn đề / Câu hỏi:** Nhầm lẫn giữa `scikit-learn` và `sklearn` khi cài đặt và import.
- **Cách giải quyết / Logic lõi:** 
  - `scikit-learn`: Tên chính thức dùng để cài đặt qua trình quản lý gói.
  - `sklearn`: Tên viết tắt dùng để import trong mã nguồn.
  ```bash
  pip install scikit-learn
  ```
  ```python
  import sklearn
  from sklearn.linear_model import LinearRegression
  ```
- **Ghi chú quan trọng:** Gói `sklearn` trên PyPI chỉ là dummy package chuyển hướng. Luôn dùng `pip install scikit-learn` để đảm bảo ổn định trong môi trường chuyên nghiệp.

### KI 4: Khắc phục lỗi hệ thống & Quy trình G-code cho máy in 3D (Klipper)
- **Chủ đề chính:** Flash firmware eMMC & Tối ưu hóa truyền file G-code.
- **Vấn đề / Câu hỏi:** Máy in 3D bị soft-brick, USB yêu cầu format liên tục khi chép file.
- **Cách giải quyết / Logic lõi:** 
  - Dùng mạch chuyển eMMC sang USB để flash lại OS Klipper (khôi phục phần mềm, không thay phần cứng).
  - USB chỉ cần format FAT32 (Allocation 4096 bytes) **một lần duy nhất**. Format liên tục là dấu hiệu rút nóng sai cách hoặc USB hỏng.
- **Ghi chú quan trọng:** Luôn dùng `Safely Remove/Eject` trước khi rút USB. Khuyến nghị chuyển sang kết nối LAN/Wi-Fi và quản lý qua giao diện Web (Fluidd/Mainsail) để loại bỏ rủi ro vật lý.

### KI 5: Kỹ thuật pha Moka Pot & Kiểm soát áp suất
- **Chủ đề chính:** Khắc phục hiện tượng bắn tia/sủi bọt khi pha cà phê.
- **Vấn đề / Câu hỏi:** Cà phê bắn dù đã hạ nhiệt bếp điện, do áp suất không ổn định và rò rỉ.
- **Cách giải quyết / Logic lõi:** 
  1. **Vệ sinh vành phễu:** Lau sạch bột bám trên mép và ren để ron cao su ép kín tuyệt đối.
  2. **Lấp đầy & Không nén:** Đổ đầy phễu, gạt phẳng mặt. Khoảng trống gây hiện tượng Channeling (nước đi đường tắt).
  3. **Kiểm soát nhiệt thủ công:** Bếp điện có cơ chế On/Off gây xung nhiệt. Khi cà phê bắt đầu chảy, nhấc ấm lên cách mặt bếp 1-2cm để tận dụng nhiệt dư, giúp dòng chảy êm.
- **Ghi chú quan trọng:** Dùng nước nóng ban đầu, xay cà phê thô hơn mức Espresso một chút. Ngắt nhiệt sớm (khi chảy được 1/3) hoặc xả lạnh đáy ấm để tránh over-extraction (đắng gắt).

### KI 6: Công thức viết Mục tiêu nghề nghiệp (CV)
- **Chủ đề chính:** Cấu trúc Career Objective chuyên nghiệp.
- **Vấn đề / Câu hỏi:** Viết mục tiêu CV lan man, thiếu trọng tâm hoặc chỉ tập trung vào lợi ích cá nhân.
- **Cách giải quyết / Logic lõi:** Áp dụng công thức: `[Vai trò] + [Kỹ năng/Kinh nghiệm] + [Giá trị mang lại] + [Mục tiêu phát triển]`. Ngắn gọn (3-5 dòng), định lượng kết quả nếu có, và khớp từ khóa với JD.
- **Ghi chú quan trọng:** Tránh dùng từ sáo rỗng ("năng động, sáng tạo"). Thay vào đó, chứng minh bằng dự án thực tế và kỹ năng cụ thể. Luôn hướng đến giá trị đóng góp cho nhà tuyển dụng.
