  **2. Xử lý Regex & So sánh chuỗi (Python):**
  ```python
  import re
  def compare_strings(s1, s2):
      s1, s2 = s1.lower().strip(), s2.lower().strip()
      # Escape dấu '-' để tránh lỗi 'bad character range'
      punctuation = r"[.?!,;:\-']" 
      s1 = re.sub(punctuation, "", s1)
      s2 = re.sub(punctuation, "", s2)
      return s1 == s2
  ```

  **3. Xử lý Ngày tháng & Năm nhuận (Python):**
  ```python
  import datetime
  from datetime import date

  def add_year(date_obj):
      try:
          return date_obj.replace(year=date_obj.year + 1)
      except ValueError: # Xử lý ngày 29/02 năm nhuận
          return date_obj.replace(year=date_obj.year + 4)

  def next_date(date_string):
      # .date() để chuyển datetime object sang date object
      date_obj = datetime.datetime.strptime(date_string, "%Y-%m-%d").date()
      next_obj = add_year(date_obj)
      # Định dạng đúng: %Y (viết hoa) cho năm 4 chữ số
      return next_obj.strftime("%Y-%m-%d")
  ```

  **4. Tích hợp Bash & Python (CLI):**
  *Bash (`meeting_reminder.sh`):*
  ```bash
  #!/bin/bash
  meeting_info=$(zenity --forms --title 'Meeting' \
      --add-calendar 'Date' --add-entry 'Title' --add-entry 'Emails' \
      --forms-date-format='%Y-%m-%d' 2>/dev/null)
  [[ -n "$meeting_info" ]] && python3 send_reminders.py "$meeting_info"
  ```

- **Ghi chú quan trọng:** 
  - Luôn ưu tiên kiểm tra nguyên nhân đơn giản nhất (nguồn điện, kết nối mạng) trước khi đi sâu vào debug phức tạp.
  - Trong regex, ký tự `-` trong `[]` phải được đặt ở đầu/cuối hoặc escape (`\-`) để tránh hiểu nhầm là khoảng ký tự (character range).
  - `datetime.strptime()` trả về `datetime`, cần `.date()` nếu hàm xử lý chỉ nhận `date`.
  - Định dạng năm trong `strftime` là `%Y` (viết hoa), không phải `yyyy`.
  - Lệnh `./script.sh` thực thi file trong thư mục hiện tại, yêu cầu file có quyền thực thi (`chmod +x script.sh`).
