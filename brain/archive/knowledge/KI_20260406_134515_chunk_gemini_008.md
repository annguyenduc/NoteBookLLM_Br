# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Tự động hóa tạo slide từ Markdown, Kiến trúc Multi-Agent cho Engineering Design Process (EDP), và Framework Prompt Engineering (TCREI).
- **Vấn đề / Câu hỏi:** Làm sao để chuyển đổi nội dung Markdown sang `.pptx` tự động, thiết kế pipeline AI agents cho quy trình thiết kế kỹ thuật, và tối ưu prompt cho tài liệu/giao tiếp IT?
- **Cách giải quyết / Logic lõi:** 
  1. **Chuyển Markdown → PPTX:** Dùng `python-pptx` kết hợp Regex để phân tách khối slide, ánh xạ dòng đầu làm Title, phần còn lại làm Body vào layout tiêu chuẩn.
     ```python
     slides_data = re.split(r'## SLIDE \d+.*?\n', content)
     for slide_text in slides_data[1:]:
         lines = [l.strip() for l in slide_text.split('\n') if l.strip()]
         slide = prs.slides.add_slide(prs.slide_layouts[1])
         slide.shapes.title.text = lines[0]
         if len(lines) > 1:
             slide.shapes.placeholders[1].text_frame.text = '\n'.join(lines[1:])
     ```
  2. **Multi-Agent EDP Pipeline:** Thiết kế luồng tuần tự 7 agents giao tiếp qua file artifacts (`.md`, `.csv`) để dễ debug và kiểm soát state. Luồng: `Problem → Research → Concept → Decision → Engineering (CAD/Code/Fab) → Test → Iteration`.
  3. **Framework TCREI:** Quy trình 5 bước tối ưu prompt: **T**houghtfully (Task/Persona/Format) → **C**reate (Context) → **R**eally (References) → **E**valuate (Đánh giá output) → **I**terate (Tinh chỉnh prompt dựa trên kết quả).
- **Ghi chú quan trọng:** 
  - `python-pptx` chỉ xử lý text thô. Để giữ layout/màu sắc phức tạp, ưu tiên dùng Gamma.app hoặc Marp.
  - LLM không thể trực tiếp xuất file CAD 3D (.step/.f3d). CAD Agent nên đóng vai trò "Advisor" hoặc sinh code OpenSCAD/Python script.
  - Pipeline EDP bắt buộc có **Human-in-the-loop** ở bước Test: học sinh phải nhập dữ liệu thực tế vào `test_results.csv` trước khi kích hoạt Test/Iteration Agent.
  - TCREI giúp chuẩn hóa giao tiếp IT, chuyển ngữ thuật ngữ kỹ thuật sang ngôn ngữ người dùng cuối, giảm thiểu hiểu lầm và tăng hiệu suất hỗ trợ.
