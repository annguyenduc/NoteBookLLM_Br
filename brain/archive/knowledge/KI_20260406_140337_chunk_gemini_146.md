# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Quản lý mã nguồn (Git/GitHub), CI/CD Pipeline & Thực hành Code Review
- **Vấn đề / Câu hỏi:** 
  1. Xử lý lỗi `fatal: repository not found` khi clone.
  2. Cú pháp tham chiếu Issue tự động trong commit message.
  3. Phân biệt Artifact, CI và CD.
  4. Quy tắc phản hồi, thuật ngữ & mục tiêu trong Code Review.
- **Cách giải quyết / Logic lõi:** 
  - **Lỗi Git Clone:** 
    ```bash
    git clone https://github.com/[username]/[repo-name].git
    ```
    *Nguyên nhân:* Sai chính tả username/repo, repo không tồn tại, repo ở chế độ Private (thiếu quyền truy cập), hoặc URL sai.
    *Xử lý:* Kiểm tra kỹ URL trên GitHub, xác minh quyền truy cập hoặc tạo repo nếu chưa có.
  - **Tham chiếu Issue:** Dùng cú pháp `[Keyword] #<số_issue>` (VD: `Fixes #123`, `Closes #45`, `Resolves #10`). Nền tảng sẽ tự động tạo hyperlink đến issue.
  - **CI/CD Core Concepts:**
    - `Artifact`: Bất kỳ file nào được sinh ra trong pipeline (file thực thi, package, báo cáo test, config).
    - `CI (Continuous Integration)`: Tự động hóa việc build & chạy test mỗi khi có commit/PR mới để đảm bảo tính ổn định của codebase.
    - `CD (Continuous Delivery)`: Đảm bảo code luôn sẵn sàng triển khai, thường áp dụng `incremental rollouts` (triển khai gia tăng) để giảm rủi ro khi đưa lên production.
  - **Code Review Best Practices:**
    - Phản hồi: `Always` (Luôn phản hồi mọi comment để duy trì giao tiếp, giải thích quyết định & tôn trọng reviewer).
    - `Nit`: Bình luận/góp ý nhỏ nhặt về style, format, cách đặt tên; không ảnh hưởng đến logic/chức năng chính.
    - Lỗi thường gặp: Tên biến/hàm không rõ nghĩa, quên xử lý điều kiện biên/exception, thiếu unit test.
    - Trạng thái comment: Nếu code đã được push phiên bản mới, comment cũ sẽ bị hệ thống đánh dấu `Outdated`.
    - Mục tiêu: Đảm bảo code dễ hiểu, nhất quán style, phát hiện case thiếu. *Không* nhằm mục tiêu "xây dựng code hoàn hảo" (điều này bất khả thi).
- **Ghi chú quan trọng:** 
  - Người duy trì dự án (Maintainer) nên: Phản hồi PR nhanh chóng, hiểu rõ thay đổi trước khi merge, và sử dụng Issue Tracker để quản lý công việc. Tránh phối hợp dự án chỉ qua email.
  - CI tập trung vào **kiểm thử & tích hợp liên tục**, CD tập trung vào **triển khai & phân phối an toàn**. Hai khái niệm bổ trợ nhau nhưng có trọng tâm vận hành khác nhau.
