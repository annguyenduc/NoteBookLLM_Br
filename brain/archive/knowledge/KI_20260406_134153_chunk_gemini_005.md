# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Kiến trúc & Phương pháp tích hợp kho lưu trữ CLI bên ngoài vào Google Antigravity IDE dưới dạng "Skill-Workflow".
- **Vấn đề / Câu hỏi:** Làm thế nào chuyển đổi tài liệu/kịch bản CLI tĩnh thành năng lực tự trị cho AI Agent trong Antigravity mà không gây bão hòa ngữ cảnh (Context Saturation) hoặc lãng phí tài nguyên tính toán?
- **Cách giải quyết / Logic lõi:** 
  1. **Phân tách kiến trúc 3 thành phần cốt lõi:**
     - `Rules` (`.agents/rules/`): Ràng buộc thụ động, luôn được tiêm vào system prompt.
     - `Skills` (`.agents/skills/`): Năng lực mô-đun, tự kích hoạt bởi Agent thông qua cơ chế **Tiết lộ Lũy tiến (Progressive Disclosure)**.
     - `Workflows` (`.agents/workflows/`): Chuỗi thao tác vĩ mô, kích hoạt thủ công qua lệnh `/`.
  2. **Cấu trúc chuẩn của Agent Skill:**
     ```yaml
     # .agents/skills/<tên-skill>/SKILL.md
     ---
     name: ten-skill-du-nhat
     description: Mô tả ngắn gọn, chứa từ khóa kích hoạt ngữ nghĩa chính xác (viết ngôi thứ 3).
     ---
     # Mục tiêu / Hướng dẫn / Ràng buộc / Ví dụ
     ```
  3. **Quy trình chuyển đổi 5 bước:**
     - Khởi tạo workspace: `ag init <tên-dự-án>` (tự động tạo `.agents/`).
     - Đóng gói logic CLI vào `SKILL.md` & thư mục `scripts/` (xử lý lỗi mạnh, output chuẩn STDOUT, tránh prompt tương tác gây treo agent).
     - Tạo Workflow điều phối (`.md` trong `.agents/workflows/`) kết nối tuần tự: Audit môi trường -> Gọi Skill -> Chờ phê duyệt Artifact -> Thực thi script.
     - Thực thi qua Agent Manager bằng lệnh `/tên-workflow`.
  4. **So sánh Repo:** `tody-agent/codymaster` vượt trội do hệ sinh thái mở, >60 skills tích hợp sẵn. Repo `for-learn-claude-cli` hiện không truy cập được.
- **Ghi chú quan trọng:** 
  - **Quản lý Context:** Chỉ tải `SKILL.md` vào context khi Agent nhận diện đúng ý định, giải phóng ngay sau khi hoàn thành để tránh "context rot" và hallucination.
  - **MCP vs Skills:** Ưu tiên dùng Agent Skills cho tác vụ CLI cục bộ (serverless, nhẹ). Dùng MCP khi cần kết nối stateful đến hệ thống ngoài (DB, API, GitHub).
  - **An ninh & FSM:** Áp dụng mô hình Máy Trạng Thái Hữu Hạn (FSM) & Bộ Ba Tác Nhân (Architect/Executor/Tech Lead). Bắt buộc dùng công cụ `ask_user` và kiểm tra `state.md` trước khi thực thi lệnh rủi ro.
  - **Tương thích chéo:** Chuẩn `SKILL.md` hoạt động đồng nhất trên Antigravity, Claude Code, Cursor, Gemini CLI. Có thể symlink từ `~/.claude/skills/` sang `~/.gemini/antigravity/skills/`.
