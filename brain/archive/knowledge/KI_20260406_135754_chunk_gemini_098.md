# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Lập trình JavaScript cho Minecraft Education Edition (Code Builder) - Điều khiển Agent xây dựng cấu trúc kim tự tháp.
- **Vấn đề / Câu hỏi:** 
  - Làm sao để Agent xây kim tự tháp với số tầng linh hoạt (thay đổi biến đầu vào)?
  - Khắc phục lỗi Agent bị lệch tọa độ, không quay lại đầu hàng/tầng sau khi xây.
  - Tối ưu hiệu suất xây dựng và xử lý sai khác API giữa các phiên bản Code Builder.
- **Cách giải quyết / Logic lõi:** 
  - **Thuật toán động:** Tính kích thước đáy `baseSize = (soTang * 2) - 1`. Kích thước tầng hiện tại `currentLayerSize = baseSize - (height * 2)`.
  - **Khắc phục lỗi drift (lệch vị trí):** Thay lệnh di chuyển dài `agent.move(BACK, n)` bằng vòng lặp `for` di chuyển từng bước `agent.move(BACK, 1)` để tránh lỗi lag/va chạm. Sau mỗi tầng, bắt buộc reset Agent về góc xuất phát bằng `agent.move(LEFT, n)` và `agent.move(BACK, n)`.
  - **Tối ưu hiệu suất:** Dùng `blocks.fill()` thay vì vòng lặp `agent.place()`. Phương pháp này xây nguyên tầng ngay lập tức, không tiêu tốn vật liệu trong túi Agent.
  ```javascript
  // Logic cốt lõi xây kim tự tháp động
  function buildPyramid(soTang) {
      if (soTang < 1) return;
      let baseSize = (soTang * 2) - 1;
      
      for (let height = 0; height < soTang; height++) {
          let size = baseSize - (height * 2);
          if (height > 0) {
              agent.move(UP, 1); agent.move(FORWARD, 1); agent.move(RIGHT, 1);
          }
          
          // Lấy tọa độ Agent hiện tại
          let pos = agent.getPosition();
          let x = positions.x(pos), y = positions.y(pos), z = positions.z(pos);
          
          // Tối ưu: Lấp đầy nguyên tầng ngay lập tức
          blocks.fill(SANDSTONE, world(x, y, z), world(x + size - 1, y, z + size - 1));
          
          // Reset vị trí Agent về góc xuất phát tầng (tránh lỗi tích tụ tọa độ)
          if (height < soTang - 1) {
              for(let i=0; i<size-1; i++) agent.move(BACK, 1);
              for(let i=0; i<size-1; i++) agent.move(LEFT, 1);
          }
      }
      agent.teleportToPlayer();
  }
  ```
- **Ghi chú quan trọng:** 
  - **API Code Builder:** Dùng `player.say()` thay cho `agent.say()` để tránh lỗi `Property does not exist`. Lấy tọa độ chuẩn bằng `agent.getPosition()` kết hợp `positions.x/y/z()`.
  - **Độ ổn định di chuyển:** Lệnh `agent.move(DIR, n)` với `n` lớn dễ thất bại trong môi trường game. Luôn chia nhỏ thành vòng lặp di chuyển `1` khối/lần.
  - **Hiệu suất:** `blocks.fill()` là phương pháp tối ưu nhất cho cấu trúc đặc, bỏ qua hoàn toàn cơ chế inventory của Agent.
  - **Validate đầu vào:** Đảm bảo `soTang > 0` và `baseSize` luôn là số lẻ để kim tự tháp có đỉnh nhọn chuẩn 1x1.
