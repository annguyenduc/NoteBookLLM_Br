# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Khắc phục lỗi in 3D (Overhang), Cấu hình kết nối WiFi trên Armbian Linux & Tích hợp OrcaSlicer với hệ sinh thái Klipper/Moonraker cho Elegoo Neptune 4.
- **Vấn đề / Câu hỏi:** 
  1. Lỗi nhựa chảy xệ, rối sợi khi in phần nhô ra 90° (Overhang) không có vật đỡ.
  2. Neptune 4 bản Standard thiếu module WiFi tích hợp, cần giải pháp kết nối không dây ổn định.
  3. Xung đột cổng mạng và lỗi cấu hình khi đồng bộ OrcaSlicer với máy in qua API.
- **Cách giải quyết / Logic lõi:** 
  **1. Khắc phục Overhang:**
  - Bật `Tree Support` trong Slicer (tiết kiệm nhựa, dễ gỡ, bề mặt tiếp xúc tốt hơn Normal Support).
  - Tối ưu tham số in: Bật quạt phụ (Auxiliary Fan 100%), giảm `Overhang Speed` xuống 20-30mm/s, giảm `Layer Height` còn 0.12-0.16mm.
  - Đổi hướng in (lật ngược mẫu để mặt phẳng xuống dưới) hoặc cắt đôi mẫu nếu thiết kế cho phép.

  **2. Cấu hình WiFi (Linux Armbian):**
  - *Phương án tối ưu:* Dùng Ethernet Bridge (Repeater/Router cũ có cổng LAN) → Ổn định, không cần driver, giảm tải CPU máy in.
  - *Phương án USB Dongle:* Bắt buộc dùng chipset `RTL8188EUS` (VD: TP-Link WN725N V1/V2). Cấu hình file `wpa_supplicant`:
    ```bash
    sudo nano /etc/wpa_supplicant/wpa_supplicant-wlan0.conf
    ```
    ```ini
    ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
    update_config=1
    country=VN
    network={
        ssid="Tên_Wifi"
        psk="Mật_Khẩu"
        key_mgmt=WPA-PSK
        priority=1
    }
    ```
  - *Cập nhật không cần SSH:* Tạo thư mục `ELEGOO_UPDATE_DIR` trên USB FAT32 → đặt file `wpa_supplicant-wlan0.conf` vào → cắm USB & reboot máy.
  - *Khắc phục lỗi mất WiFi sau reboot:* Xóa file rác `wpa_supplicant.conf` trong cùng thư mục, chỉ giữ lại file `-wlan0.conf`. Restart dịch vụ: `sudo systemctl restart networking`.

  **3. Tích hợp OrcaSlicer:**
  - Host Type: Chọn `Octo/Klipper`.
  - Xử lý xung đột cổng: Nhập `IP:7125` cho kết nối API (gửi G-code/Upload), nhập `http://IP` (cổng 80) cho trường `Device UI` (hiển thị giao diện Fluidd).
  - Gán IP tĩnh qua Router (DHCP Reservation) để tránh thay đổi IP sau mỗi lần khởi động lại mạng.
- **Ghi chú quan trọng:** 
  - Neptune 4 dùng kiến trúc kép: Linux Host (chạy Klipper/Moonraker/Fluidd) + MCU riêng điều khiển động cơ. Moonraker API mặc định chạy cổng `7125`, Web UI (Fluidd) chạy cổng `80`.
  - TP-Link Archer T2U Plus dùng chipset `RTL8821AU` (chuẩn AC), **không tương thích mặc định** với kernel Armbian của Neptune 4. Cần kiểm tra bằng lệnh `lsusb` hoặc đổi sang dongle chuẩn `RTL8188EUS`.
  - Tránh cắm chung Webcam và WiFi Dongle vào cùng hub USB để tránh nghẽn băng thông USB 2.0 và sụt áp gây drop kết nối.
  - Nếu OrcaSlicer crash khi mở tab `Device`, để trống trường `Device UI` và giám sát qua trình duyệt web ngoài.
