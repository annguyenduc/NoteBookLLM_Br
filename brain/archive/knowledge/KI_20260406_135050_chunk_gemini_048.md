# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Tích hợp Wi-Fi, khắc phục lỗi Firmware & Tối ưu kiến trúc hệ thống nhúng trên máy in 3D Elegoo Neptune 4 (Klipper/Armbian).
- **Vấn đề / Câu hỏi:** 
  1. Cập nhật firmware gốc (`_factory.deb`) gây ghi đè cấu hình, kích hoạt lỗi xung đột mạng do file cấu hình thử nghiệm (`znpkj-668`) chiếm dụng `wpa_supplicant`.
  2. Adapter Wi-Fi USB hiện đại (TP-Link Archer T2U Plus - chipset RTL8821AU) không hoạt động do thiếu driver trong kernel Rockchip cũ, đồng thời việc biên dịch driver out-of-tree gặp rào cản thiếu `linux-headers` và sai lệch GCC.
- **Cách giải quyết / Logic lõi:** 
  - **Kiến trúc hệ thống:** Neptune 4 dùng SoC Rockchip RK3328 (ARM64) chạy Armbian Linux + Klipper Host. MCU (STM32) xử lý thời gian thực cho động cơ/cảm biến, SBC xử lý mạng & giao diện web.
  - **Khắc phục lỗi cấu hình mạng (RCA):** Xóa file cấu hình rác của nhà sản xuất để giải phóng daemon mạng.
    ```bash
    sudo mv /etc/wpa_supplicant/wpa_supplicant-wlan0.conf /etc/wpa_supplicant/wpa_supplicant-wlan0.conf.bak
    sudo systemctl restart networking
    ```
  - **Lựa chọn phần cứng tương thích:** Ưu tiên **TP-Link TL-WN725N (V2/V3)** dùng chipset `RTL8188EUS` (driver `8188eu.ko` đã tích hợp sẵn `in-tree`). Tránh dòng 5GHz/Archer do yêu cầu biên dịch phức tạp.
  - **Quy trình cấu hình:** Kết nối Ethernet/USB Tethering → SSH (`ssh mks@<IP>`, pass: `makerbase`) → Dùng `sudo nmtui` để quét & kết nối Wi-Fi.
  - **Giải pháp thay thế tối ưu:** Dùng **Wi-Fi Bridge** (thiết bị chuyển đổi WiFi sang Ethernet). Máy in nhận diện là mạng dây, bypass hoàn toàn vấn đề driver Linux, giảm tải CPU RK3328.
  - **Tối ưu ổn định kết nối:** Tắt chế độ tiết kiệm năng lượng Wi-Fi để tránh ngắt kết nối khi in lâu:
    ```bash
    sudo iw dev wlan0 set power_save off
    ```
- **Ghi chú quan trọng:** 
  - Firmware `_factory.deb` có cơ chế `destructive update`, sẽ reset `printer.cfg` (mất Z-Offset, PID, Input Shaper, Bed Mesh). Bắt buộc hiệu chuẩn lại sau khi cập nhật.
  - Cổng USB trên máy in có giới hạn nguồn & băng thông. Nên dùng dây nối dài USB hoặc Hub có nguồn phụ để tránh sụt áp gây lỗi `Timer too close` trong Klipper.
  - Nếu bắt buộc dùng chipset hiện đại, cân nhắc cài firmware cộng đồng **OpenNept4une** (kernel mới hơn, kho driver phong phú) thay vì cố biên dịch trên Armbian gốc bị cắt giảm công cụ.
