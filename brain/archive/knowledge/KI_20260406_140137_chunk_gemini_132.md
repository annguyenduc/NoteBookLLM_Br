# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Triển khai Model Context Protocol (MCP) Server bằng Python & Tích hợp Cursor IDE
- **Vấn đề / Câu hỏi:** Làm thế nào để chuẩn hóa giao tiếp giữa LLM và công cụ/dữ liệu bên ngoài, xây dựng MCP Server bằng Python, và cấu hình để Cursor AI Agent tự động khám phá, gọi công cụ an toàn?
- **Cách giải quyết / Logic lõi:** 
  - **Kiến trúc MCP:** Mô hình `Host` (Cursor/Claude) → `Client` (quản lý phiên 1:1) → `Server` (cung cấp Tools/Resources/Prompts). Giao tiếp qua `stdio` (cục bộ), `SSE` hoặc `Streamable HTTP` (từ xa).
  - **Thiết lập môi trường & SDK:**
    ```bash
    uv init my-mcp-server && cd my-mcp-server
    uv venv && source .venv/bin/activate
    uv add "mcp[cli]"  # Cài SDK + công cụ debug
    ```
  - **Định nghĩa Primitives với FastMCP:**
    ```python
    from mcp.server.fastmcp import FastMCP, Context
    mcp = FastMCP("MyServer")

    @mcp.tool()
    def process_data(query: str, ctx: Context) -> str:
        """Mô tả rõ ràng để LLM hiểu mục đích & tham số."""
        ctx.info(f"Bắt đầu xử lý: {query}")
        # Logic nghiệp vụ...
        return "Kết quả xử lý"

    @mcp.resource("config://app")
    def get_app_config() -> dict:
        return {"version": "1.0", "env": "prod"}

    if __name__ == "__main__":
        mcp.run()  # Mặc định chạy qua stdio
    ```
  - **Cấu hình Cursor (`mcp.json`):** Đặt tại `.cursor/mcp.json` (dự án) hoặc `~/.cursor/mcp.json` (toàn cục).
    ```json
    {
      "mcpServers": {
        "myPythonServer": {
          "command": "/path/to/.venv/bin/python",
          "args": ["/path/to/server.py"],
          "env": { "API_KEY": "${SECRET_KEY}" },
          "description": "Cung cấp công cụ xử lý dữ liệu nội bộ"
        }
      }
    }
    ```
  - **Debug & Test:** Chạy `mcp dev server.py` để mở **MCP Inspector** (giao diện web), kiểm tra schema tool, gọi thử và xem phản hồi trước khi tích hợp vào Cursor.
- **Ghi chú quan trọng:**
  - **Bảo mật:** Không hardcode secret. Luôn truyền qua trường `env` trong `mcp.json` hoặc `.env`. Validate & sanitize input trong tool để tránh command/SQL injection.
  - **Logging đúng chuẩn:** Ghi log debug vào `stderr` (`print("msg", file=sys.stderr)`) để Cursor bắt được. Tránh ghi vào `stdout` vì sẽ phá vỡ luồng giao thức `stdio`.
  - **Đối tượng `Context`:** Tham số `ctx: Context` cho phép ghi log có cấu trúc (`ctx.info/warn/error`), báo cáo tiến trình (`await ctx.report_progress()`), và đọc resource khác.
  - **Giới hạn Cursor:** Hiện tại chỉ gửi ~40 tools đầu tiên cho Agent. Nên nhóm chức năng liên quan vào ít server hơn hoặc tạm tắt server không dùng.
  - **Transport:** `stdio` tối ưu cho tool cục bộ/IDE. `SSE/Streamable HTTP` dùng cho server từ xa, cấu hình Cursor thay `command/args` bằng `url`.
