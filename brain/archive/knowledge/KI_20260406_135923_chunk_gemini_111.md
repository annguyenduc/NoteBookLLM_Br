# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Tối ưu hóa & Tự động hóa quy trình phát triển phần mềm với AI Agent trong Cursor IDE.
- **Vấn đề / Câu hỏi:** Làm thế nào để cấu hình, mở rộng và kiểm soát AI Agent trong Cursor IDE nhằm tự động hóa tác vụ đa bước, tích hợp công cụ ngoài, đồng thời duy trì chất lượng mã & kiểm soát ngữ cảnh?
- **Cách giải quyết / Logic lõi:** 
  1. **Cấu hình hành vi AI phân cấp với `.cursorrules`:**
     - `User Rules`: Áp dụng toàn cục (cài đặt Cursor).
     - `Project Rules` (`.cursor/index.mdc`): Gắn với repo, luôn được đưa vào ngữ cảnh.
     - `Dynamic Rules` (`.cursor/rules/*.mdc`): Chỉ kích hoạt khi tác vụ khớp mô tả, tiết kiệm token.
     ```yaml
     # Ví dụ .cursor/index.mdc
     type: "always"
     description: "Chuẩn code dự án"
     content: |
       - Luôn dùng TypeScript strict mode & ESLint.
       - Bao gồm try/catch hoặc error handling trong mọi hàm async.
       - Quy ước đặt tên: camelCase (biến/hàm), PascalCase (class/interface).
     ```
  2. **Mở rộng không gian hành động với Model Context Protocol (MCP):**
     - MCP đóng vai trò chuẩn kết nối "plug-and-play" giữa Cursor và hệ thống ngoài (Jira, Figma, DB, API nội bộ).
     - Cấu hình: `Cursor Settings > MCP` → Nhập `Name`, `Transport` (`stdio`/`sse`), `Command/URL`.
     - Cho phép Agent gọi công cụ tùy chỉnh để tự động hóa quy trình liên ứng dụng mà không cần rời IDE.
  3. **Kỹ thuật Nhắc lệnh & Gắn ngữ cảnh động (`@` symbols):**
     - `@files` / `@folders`: Đẩy nội dung file/thư mục vào prompt.
     - `@code`: Tham chiếu hàm/class cụ thể để refactor.
     - `@docs` / `@web` / `@git` / `@terminal` / `@rules`: Mở rộng ngữ cảnh theo thời gian thực.
     - Prompt phải chi tiết: nêu rõ yêu cầu, đường dẫn file, số dòng, và ràng buộc đầu ra.
  4. **Thiết kế luồng Agent Mode & Tách biệt Planning/Execution:**
     - Luồng chuẩn: Hiểu yêu cầu → Quét codebase → Lập kế hoạch → Thực thi → Xác minh (fix linter) → Báo cáo.
     - **Chiến lược tối ưu:** Tách thành 2 custom mode:
       - `Planner Mode`: Chỉ phân tích & tạo plan chi tiết (không sinh code).
       - `Executor Mode`: Đọc plan → code → test → commit atomic.
     - Giúp Git history sạch, dễ rollback và giảm lỗi do AI "sáng tạo" ngoài luồng.
  5. **Quản lý Mô hình & API Key tùy chỉnh:**
     - Hỗ trợ key từ OpenAI, Anthropic, Google, Azure, AWS Bedrock, OpenRouter.
     - Cấu hình: `Cursor Settings > Models` → Dán key → Verify.
     - Chọn mô hình theo độ phức tạp: `Gemini 1.5 Flash`/`GPT-3.5` cho tác vụ đơn giản; `Claude 4 Sonnet`/`Gemini 2.5 Pro`/`GPT-4o` cho kiến trúc & logic phức tạp.
- **Ghi chú quan trọng:**
  - ⚠️ **Hạn chế hiện tại:** Chưa có API mở rộng công khai như VS Code, giới hạn 5 custom modes, dễ "quên ngữ cảnh" hoặc overcomplicate code đơn giản. Phụ thuộc cloud → cần lưu ý bảo mật dữ liệu nhạy cảm.
  - 🔒 **Tối ưu hiệu suất & Token:** Dùng `.cursorignore` để loại file nặng/không liên quan khỏi indexing. Gom nhóm prompt tương tự để giảm số lần gọi API. Luôn review code AI trước khi commit.
  - 🔄 **Triết lý vận hành:** Coi AI là "pair programmer" hoặc "trợ lý điều phối", không phải "autopilot". Áp dụng vòng lặp: `Prompt chi tiết → Review → Refine → Test → Commit`.
