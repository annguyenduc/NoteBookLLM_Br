# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Lập trình Minecraft Education Edition (Python) & Tối ưu hóa mã Arduino điều khiển Robot.
- **Vấn đề / Câu hỏi:** 
  1. Cách đặt khối có thuộc tính đặc biệt (cầu thang ngược) bằng Agent, xử lý lỗi tham số ẩn khi dùng `player.on_chat`, và giới hạn API của đối tượng Player.
  2. Tối ưu mã Arduino bị lỗi cấu trúc (logic nằm trong `while(1)` ở `setup()`), chứa "magic numbers", và các câu lệnh `if-else` lồng nhau phức tạp.
- **Cách giải quyết / Logic lõi:** 
  **1. Minecraft Education (Python):**
  - *Đặt khối có thuộc tính:* Dùng chuỗi định dạng `"tên_khối[thuộc_tính=giá_trị]"`.
    ```python
    agent.place("stone_brick_stairs[upside_down_bit=true, facing_direction=2]")
    ```
  - *Sửa lỗi `player.on_chat`:* Hệ thống tự truyền đối số ẩn. Khai báo hàm với `*args` để tránh lỗi `types not compatible`.
    ```python
    def dat_cau_thang_nguoc(*args):
        pos = player.position()
        target = pos(pos.get_value(Axis.X), pos.get_value(Axis.Y) + 2, pos.get_value(Axis.Z))
        blocks.place("stone_brick_stairs[upside_down_bit=true, facing_direction=2]", target)
    player.on_chat("datcau", dat_cau_thang_nguoc)
    ```
  **2. Arduino (C++):**
  - *Cấu trúc chuẩn:* Chuyển toàn bộ logic lặp từ `setup()` sang `loop()`. `setup()` chỉ dùng cho `pinMode`, `attach`, khởi tạo biến.
  - *Tách hàm & Làm phẳng logic:* 
    ```cpp
    void loop() {
      controlServos();
      controlMotors();
    }
    void controlMotors() {
      int x = analogRead(JOYSTICK_X_PIN);
      int y = analogRead(JOYSTICK_Y_PIN);
      if (y > DEADZONE_HIGH) { /* Lùi */ }
      else if (y < DEADZONE_LOW) { /* Tiến */ }
      else if (x > DEADZONE_HIGH) { /* Phải */ }
      else if (x < DEADZONE_LOW) { /* Trái */ }
      else { stopMotors(); }
    }
    ```
- **Ghi chú quan trọng:** 
  - **API Minecraft:** `Agent` được thiết kế để thao tác thế giới (đặt/phá khối, di chuyển). `Player` chỉ hỗ trợ lấy thông tin (`position()`) hoặc kích hoạt sự kiện (`on_chat`). Không thể dùng code mô phỏng hành động vật lý của Player (như Shift/ngồi).
  - **Cú pháp khối:** Thuộc tính `facing_direction` nhận giá trị 2-5 tương ứng Bắc-Nam-Tây-Đông. `upside_down_bit=true` để lật ngược khối.
  - **Arduino Best Practices:** Luôn dùng `const int` thay cho số trực tiếp (magic numbers). Tách logic thành các hàm con (`controlX`, `stopX`) giúp code dễ đọc, dễ bảo trì và tránh xung đột tài nguyên. Loại bỏ biến không dùng để giảm bộ nhớ.
