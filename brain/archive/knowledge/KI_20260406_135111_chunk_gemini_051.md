# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Hiệu chỉnh thông số in 3D (nhựa PETG), Cấu hình mạng & SSH cho máy in Klipper (Elegoo Neptune 4), Thiết lập phần mềm cắt lớp (OrcaSlicer/Bambu Studio).
- **Vấn đề / Câu hỏi:** Cách calibrate chuẩn xác các thông số in PETG, khắc phục lỗi mất menu WiFi/thiếu driver trên Neptune 4, thiết lập kết nối mạng qua LAN/SSH, và cấu hình hiển thị thumbnail trên màn hình máy in Klipper.
- **Cách giải quyết / Logic lõi:** 
  **1. Quy trình Calibrate PETG (Bambu Studio/OrcaSlicer)**
  - `Temperature`: Kiểm tra độ bóng (lì = quá lạnh), cầu nối/bridging (xệ = quá nóng), tơ nhện (quá nóng) và độ bền lớp (bẻ thử: gãy giòn = lạnh, dai/xơ = tốt). Ưu tiên khoảng `240-250°C`. Điền vào `Filament > Temperature > Nozzle`.
  - `Flow Rate`: In tháp test (-20 đến +20), chọn thẻ có bề mặt mịn nhất, không hở/sần. Điền vào `Filament > Flow Ratio` (mặc định PETG ~`0.95`).
  - `Pressure Advance (PA)`: Dùng test line, chọn dòng có độ rộng đều nhất từ đầu đến cuối. Điền vào `Filament > Enable Pressure Advance` (khoảng `0.02-0.06`).
  - `Max Volumetric Speed`: Test tốc độ tối đa trước khi nhựa không kịp chảy (mất lớp/sần). Điền vào `Filament > Max volumetric speed` (thường `10-15 mm³/s`).

  **2. Cấu hình Mạng & SSH cho Neptune 4 (Armbian/Klipper)**
  - *Windows LAN Sharing (Cấp IP tạm thời):* `Win + R` → `ncpa.cpl` → Chuột phải `Wi-Fi` → `Properties` → Tab `Sharing` → Tích `Allow other network users...` → Chọn `Ethernet`. Máy in nhận IP dạng `192.168.137.x`.
  - *Truy cập SSH:* 
    ```bash
    ssh mks@<IP_MAY_IN>  # Password mặc định: makerbase (không hiện ký tự khi gõ)
    ```
  - *Cấu hình WiFi thủ công (khi thiếu giao diện `nmtui`/`nmcli`):*
    ```bash
    sudo nano /etc/wpa_supplicant/wpa_supplicant.conf
    ```
    Nội dung file chuẩn:
    ```ini
    ctrl_interface=/var/run/wpa_supplicant
    update_config=1
    network={
        ssid="TÊN_WIFI"
        psk="MAT_KHAU"
        key_mgmt=WPA-PSK
        scan_ssid=1
    }
    ```
    Lưu: `Ctrl+O` → `Enter` → `Ctrl+X`. Áp dụng: `sudo reboot`.
  - *Kiểm tra trạng thái:* `ip a` (xem interface `wlan0`), `sudo iwlist wlan0 scan | grep ESSID` (quét mạng).

  **3. Thiết lập Slicer & Thumbnail**
  - Tải OrcaSlicer từ GitHub, chọn profile `Elegoo Neptune 4`, Host Type: `OctoPrint/Klipper`, nhập IP máy in.
  - Bật Thumbnail: `Printer Settings` → `G-code thumbnails` điền `300x300, 32x32`, định dạng `PNG`. OrcaSlicer hỗ trợ native cho Klipper tốt hơn Bambu Studio.

- **Ghi chú quan trọng:** 
  - USB WiFi TP-Link (chip RTL8188EUS/GU) thường **không tương thích mặc định** với Armbian Lite trên Neptune 4. Nếu `ip a` không hiện `wlan0`, hệ thống thiếu driver. Giải pháp thay thế: dùng WiFi Repeater chuyển tín hiệu sang LAN hoặc duy trì kết nối LAN trực tiếp.
  - IP `169.254.x.x` là IP tự gán (APIPA), báo hiệu DHCP thất bại. Nguyên nhân thường do sai mật khẩu hoặc router phát sóng `5GHz` trong khi adapter chỉ hỗ trợ `2.4GHz`.
  - Firmware Neptune 4 bản thường (`v1.3.3`) ẩn hoàn toàn menu WiFi trên màn hình cảm ứng. Việc cấu hình phải thực hiện qua giao diện Web (Fluidd) hoặc SSH.
  - Ưu tiên dùng **OrcaSlicer** cho máy Klipper hãng khác để tránh lỗi kết nối, thumbnail và tận dụng profile tối ưu có sẵn.
