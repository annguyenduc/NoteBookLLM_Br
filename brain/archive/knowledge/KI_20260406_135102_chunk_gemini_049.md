# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Tích hợp Adapter Wi-Fi USB cho máy in 3D Elegoo Neptune 4 & Cân chỉnh thông số in (Calibration) trên Bambu Studio/OrcaSlicer.
- **Vấn đề / Câu hỏi:** Neptune 4 không có Wi-Fi tích hợp, làm sao kết nối không dây ổn định với laptop qua USB TP-Link? Đồng thời, cách khắc phục lỗi bề mặt sần và cân chỉnh thông số đùn nhựa trên slicer cho máy hãng thứ 3.
- **Cách giải quyết / Logic lõi:** 
  1. **Tương thích phần cứng:** Kernel Armbian cũ trên Neptune 4 chỉ hỗ trợ sẵn driver cho chipset `Realtek RTL8188EUS` (TP-Link TL-WN725N V2/V3). Các dòng 5GHz/Wi-Fi 6 thiếu driver và không hoạt động.
  2. **Khắc phục lỗi Firmware gốc:** Elegoo để thừa file cấu hình thử nghiệm gây xung đột tiến trình mạng, khiến Wi-Fi mất kết nối sau mỗi lần reboot.
  3. **Cấu hình hệ thống & Mạng:** Truy cập SSH, xóa file lỗi, dùng `nmtui` thiết lập kết nối. Tắt chế độ tiết kiệm năng lượng của card mạng để tránh drop kết nối khi in.
  4. **Cân chỉnh Slicer:** Dùng Manual Calibration cho Flow Rate & Pressure Advance. Nhiệt độ nozzle phải chuẩn trước khi test Flow Rate để tránh sai lệch kết quả.
  ```bash
  # 1. Truy cập SSH vào máy in
  ssh mks@<IP_MAY_IN>
  
  # 2. Kiểm tra nhận diện phần cứng & giao diện
  lsusb | grep -i realtek  # Tìm Vendor ID 0bda:8179
  ip link show             # Xác nhận interface wlan0 đã hiện
  
  # 3. Fix lỗi xung đột firmware Elegoo (Bắt buộc)
  cd /etc/wpa_supplicant/
  sudo rm wpa_supplicant-wlan0.conf
  
  # 4. Cấu hình Wi-Fi & Tối ưu kết nối
  sudo nmtui                 # Giao diện text chọn SSID & nhập mật khẩu
  sudo iw dev wlan0 set power_save off  # Tắt power saving gây mất kết nối giữa chừng
  ```
- **Ghi chú quan trọng:** 
  - Neptune 4 dùng kiến trúc lai: SBC RK3328 (chạy Armbian/Klipper) + MCU. Cổng USB chia sẻ băng thông, tránh cắm chung với Webcam để tránh lỗi `Timer too close` làm dừng in đột ngột.
  - Giải pháp thay thế ổn định hơn USB Dongle: Dùng **Wi-Fi Bridge/Extender** có cổng LAN nối vào máy in. Máy in nhận diện là mạng dây, loại bỏ hoàn toàn rủi ro driver Linux và sụt áp cổng USB.
  - **Bambu Studio** bị giới hạn tính năng Calibration cho máy hãng thứ 3. Khuyến nghị chuyển sang **OrcaSlicer** (fork của Bambu Studio) để có đầy đủ công cụ tự động tạo Temp Tower, Flow Dynamics.
  - Khi cân chỉnh Flow Rate, phải đảm bảo nhiệt độ in chính xác. Sai nhiệt độ sẽ làm kết quả Flow Ratio bị lệch, gây sần hoặc hở lớp in.
