# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Xây dựng & Vận hành Nền tảng E-learning Toán học + Khung chương trình Đào tạo Robotics THPT (3 tháng)
- **Vấn đề / Câu hỏi:** Lộ trình kỹ thuật, chiến lược nội dung, mô hình kinh doanh cho startup ed-tech solo founder và phương pháp triển khai chương trình robotics thực chiến cho học sinh.
- **Cách giải quyết / Logic lõi:** 
  **1. Kiến trúc & Vận hành E-learning:**
  - *Tech Stack:* 
    - `MVP/Tiết kiệm:` WordPress + LMS Plugin (LearnPress, Tutor LMS, LifterLMS). Ưu: chi phí thấp, cộng đồng lớn. Nhược: giới hạn scale.
    - `Scale/Tùy biến:` Tự phát triển (Laravel, Django, Node.js). Ưu: kiểm soát hoàn toàn, bảo mật & hiệu năng cao. Nhược: tốn thời gian/kỹ thuật.
  - *Tính năng cốt lõi:* Quản lý user, Course builder, Auto-grading (trắc nghiệm), Progress tracking, Community/Q&A.
  - *Business Model (Freemium):* Miễn phí nội dung nền tảng -> Thu phí tính năng cao cấp (Lớp live, AI Tutor, Tài liệu độc quyền, Chứng chỉ, Tắt ads). Bổ trợ: AdSense, Donation, Affiliate.
  
  **2. Khung chương trình Robotics (OhStem ORC K3 - 12 tuần):**
  - *Phương pháp sư phạm:* Project-Based Learning (PBL), cấu trúc `4 tuần học -> 2 tuần thi đấu nội bộ`.
  - *Lộ trình kỹ thuật:* 
    - `Tháng 1:` Lắp ráp khung -> Lập trình chuyển động cơ bản -> Điều khiển Gamepad -> Tối ưu chiến thuật -> Thi đấu Robot Sumo.
    - `Tháng 2-3:` Tích hợp cảm biến IR (dò line) & Siêu âm (né vật cản) -> Lập trình hỗn hợp (chuyển đổi chế độ tay/tự động) -> Thi đấu nhiệm vụ phức hợp.
  - *Logic lập trình điều khiển Gamepad (Tuần 3):*
    ```javascript
    // Pseudo-code logic map Joystick sang động cơ
    const axisX = getJoystick('X'); // -100 đến 100
    const axisY = getJoystick('Y'); // -100 đến 100
    
    // Tính tốc độ vi sai cho 2 động cơ
    let speedLeft  = clamp(axisY + axisX, -100, 100);
    let speedRight = clamp(axisY - axisX, -100, 100);
    
    // Xử lý trạng thái dừng & xuất lệnh
    if (axisX === 0 && axisY === 0) {
      stopMotors(); 
    } else {
      setMotor('left', speedLeft);
      setMotor('right', speedRight);
    }
    ```
- **Ghi chú quan trọng:** 
  - E-learning: Nội dung chất lượng & trải nghiệm người dùng là yếu tố sống còn. Nên bắt đầu với MVP phạm vi hẹp, mở rộng dần. Cần tuân thủ pháp lý ed-tech & bảo mật dữ liệu tại VN.
  - Robotics: Ưu tiên hệ sinh thái đồng bộ (OhStem) để giảm thời gian debug phần cứng, tập trung vào tư duy thuật toán & cơ khí. Duy trì CLB bền vững bằng mô hình `Mentorship` (khóa trước hướng dẫn khóa sau) và định hướng thi đấu chuyên nghiệp (WRO, FPT AI & Robotics).
