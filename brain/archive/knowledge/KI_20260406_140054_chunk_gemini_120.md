# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Khắc phục lỗi xác thực GitHub MCP, gỡ lỗi ứng dụng GUI Python (Exit Code 1), và kiến trúc luồng tích hợp MCP Server trong SDLC.
- **Vấn đề / Câu hỏi:** 
  1. Tool `search_code` của GitHub MCP Hub báo lỗi `Authentication Failed` dù token hợp lệ, scopes đầy đủ và các tool GitHub khác hoạt động bình thường.
  2. `interactive-feedback-mcp` lỗi khởi chạy giao diện "exit code 1" sau khi đã đồng bộ môi trường (`uv sync`).
  3. Cách orchestrate hiệu quả nhiều công cụ MCP trong một quy trình phát triển phần mềm thực tế.
- **Cách giải quyết / Logic lõi:** 
  - **Bypass MCP cho `search_code`:** Lỗi nằm ở cách GitHub MCP Server xử lý endpoint search, không phải do token. Giải pháp tối ưu là gọi trực tiếp qua REST API:
    ```bash
    python -c "import requests; r = requests.get('https://api.github.com/search/code?q=def+main', headers={'Authorization': 'token <YOUR_TOKEN>'}); print(r.json())"
    ```
  - **Gỡ lỗi `interactive-feedback-mcp` (Exit Code 1):**
    1. Cài đặt `Microsoft Visual C++ Redistributable` (x86 & x64) để cung cấp DLL hệ thống bắt buộc cho Qt/PyQt.
    2. Chạy trực tiếp `server.py` để bắt stack trace chi tiết:
       ```bash
       cd D:/Python/interactive-feedback-mcp
       .venv/Scripts/python.exe server.py
       ```
    3. Thêm logging vào `server.py` để xác định chính xác điểm crash:
       ```python
       import logging
       logging.basicConfig(level=logging.DEBUG, format='%(asctime)s - %(levelname)s - %(message)s')
       try:
           # Code khởi tạo QApplication/UI
           logging.debug("UI khởi tạo thành công.")
       except Exception as e:
           logging.error(f"Lỗi khởi tạo giao diện: {e}")
       ```
    4. Nếu vẫn lỗi, ưu tiên chuyển sang giải pháp web-based hoặc tích hợp API feedback bên thứ ba.
  - **Luồng tích hợp MCP Server (Pipeline chuẩn):** 
    `GitHub` (Quản lý repo/PR) → `Filesystem` (Đồng bộ code cục bộ) → `Code-Quality` (Kiểm tra type hints, docstring, PEP8) → `Performance-Optimization` (Phân tích CPU/RAM, tối ưu DB) → `Automated-Deployment` (Gen Dockerfile/docker-compose, deploy) → `Interactive-Feedback` (Thu thập phản hồi).
- **Ghi chú quan trọng:** 
  - Token GitHub cần đủ scopes (`repo`, `user`, `workflow`, `admin:org`...) nhưng MCP layer có thể tồn tại bug xử lý riêng với một số endpoint như `search/code`.
  - Lỗi GUI Python "exit code 1" thường do thiếu runtime OS hoặc xung đột biến môi trường `PATH`, hiếm khi do thiếu gói Python. Luôn kiểm tra log console trước khi debug code.
  - Khi tích hợp nhiều MCP server, nên thiết kế luồng dữ liệu tuần tự (Input → Process → Output) và xử lý lỗi từng bước để tránh xung đột context, rate-limit hoặc trạng thái treo tool.
