# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Lộ trình & Kiến thức cốt lõi cho Python Developer (Hệ thống nội bộ ERP/CRM/OMS & eCommerce)
- **Vấn đề / Câu hỏi:** Làm thế nào để chuyển từ Python cơ bản sang phát triển hệ thống doanh nghiệp phức tạp, xử lý dữ liệu lớn, tự động hóa quy trình và tích hợp API đa nền tảng một cách hiệu quả & có khả năng mở rộng?
- **Cách giải quyết / Logic lõi:** 
  **1. Python Nâng cao & Concurrency**
  - `Decorators`: Sửa đổi hành vi hàm mà không đổi mã nguồn gốc. Luôn dùng `functools.wraps` để bảo toàn metadata.
  - `Generators/Iterators`: Dùng `yield` để lazy evaluation, giảm tải bộ nhớ khi xử lý luồng dữ liệu lớn.
  - `Concurrency`: `asyncio` cho I/O-bound (gọi API, DB), `multiprocessing` cho CPU-bound (tính toán nặng), tránh GIL.
  ```python
  # Decorator chuẩn
  import functools
  def log_time(func):
      @functools.wraps(func)
      def wrapper(*args, **kwargs):
          return func(*args, **kwargs)
      return wrapper

  # Generator xử lý file lớn
  def stream_data(file_path):
      with open(file_path) as f:
          for line in f: yield line.strip()
  ```

  **2. Cấu trúc dữ liệu & DSA**
  - `List`: Dynamic array. Truy cập O(1), chèn/xóa cuối O(1), chèn/xóa giữa O(n).
  - `Dict`/`Set`: Hash table. Tra cứu/thêm/xóa trung bình O(1). Key phải hashable & duy nhất.
  - `Tuple`: Immutable, dùng làm key dict hoặc trả về nhiều giá trị từ hàm.
  - Áp dụng Big O notation để chọn cấu trúc dữ liệu & thuật toán phù hợp với quy mô dữ liệu.

  **3. Web Frameworks & REST API**
  - `Django/DRF`: "Batteries-included", ORM tích hợp, Admin UI, Serializers cho validation & serialization.
  - `FastAPI`: Async native, Pydantic validation, tự động sinh Swagger/ReDoc, phù hợp API hiệu suất cao.
  - `Flask`: Microframework, linh hoạt, thích hợp microservices hoặc công cụ nội bộ nhỏ.
  ```python
  # FastAPI + Pydantic Validation
  from fastapi import FastAPI
  from pydantic import BaseModel
  app = FastAPI()
  class Product(BaseModel):
      sku: str
      price: float
  @app.post("/products/")
  def create_product(p: Product):
      return {"status": "created", "sku": p.sku}
  ```

  **4. Database & ORM**
  - SQL: PostgreSQL/MySQL. Nắm vững DDL/DML, Joins, Indexing, ACID.
  - ORM: `SQLAlchemy` (Engine, Session, Declarative Models) cho FastAPI/Flask; `Django ORM` cho Django. Giảm SQL injection, tăng tốc dev.
  - NoSQL: MongoDB (Document model, BSON) cho dữ liệu phi cấu trúc hoặc thay đổi schema nhanh.

  **5. Task Queues & Message Brokers**
  - `Celery`: Xử lý tác vụ nền bất đồng bộ, tách rời khỏi luồng request/response.
  - `Redis`: Caching, broker đơn giản (`LPUSH`/`BRPOP`), Pub/Sub. Nhanh, dễ setup.
  - `RabbitMQ`: Broker enterprise, định tuyến phức tạp (Exchanges, Bindings), durable queues cho tác vụ quan trọng.
  ```python
  # Celery Task Definition
  from celery import Celery
  app = Celery('worker', broker='redis://localhost:6379/0')
  @app.task
  def sync_inventory(product_id):
      # Logic đồng bộ tồn kho tốn thời gian
      return f"Synced {product_id}"
  ```

  **6. Selenium Automation**
  - Định vị: Ưu tiên `By.ID`, `By.CSS_SELECTOR`. Tránh XPath dễ vỡ khi DOM thay đổi.
  - Chờ động: Bắt buộc dùng `Explicit Waits` thay vì `time.sleep()` để xử lý trang web động.
  ```python
  from selenium.webdriver.common.by import By
  from selenium.webdriver.support.ui import WebDriverWait
  from selenium.webdriver.support import expected_conditions as EC
  wait = WebDriverWait(driver, 10)
  btn = wait.until(EC.element_to_be_clickable((By.CSS_SELECTOR, ".submit-btn")))
  btn.click()
  ```

  **7. Tích hợp API (Shopify, Amazon, TikTok)**
  - Xác thực: OAuth 2.0, API Keys/Secrets, LWA (Amazon). Quản lý token & refresh token an toàn.
  - Xử lý: Rate limiting (exponential backoff), pagination, retry logic, error handling.
  - Ưu tiên SDK chính thức. Fallback sang `requests` khi cần tùy chỉnh endpoint hoặc SDK thiếu tính năng.

- **Ghi chú quan trọng:**

  - Kiến trúc thực tế luôn là sự kết hợp: Web Framework + DB/ORM + Task Queue + API Integration + Automation Pipeline.

  - Luôn tuân thủ bảo mật (quản lý credentials, xác thực OAuth), xử lý lỗi gracefully, và ưu tiên viết code có khả năng mở rộng (scalable) & dễ bảo trì.
