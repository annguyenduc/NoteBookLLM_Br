# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Tích hợp Python vào Minecraft Education Edition (M:EE) để tự động hóa xây dựng & Phát triển game giáo dục 2D độc lập bằng Pygame.
- **Vấn đề / Câu hỏi:** Làm thế nào để lập trình Agent trong M:EE, thiết kế khung game giáo dục tương tác, và xây dựng lộ trình giảng dạy chuẩn hóa cho học sinh (có/không có nền tảng code)?
- **Cách giải quyết / Logic lõi:** 
  1. **Điều khiển Agent trong M:EE (Python):** Sử dụng `Code Builder` để viết script điều khiển Agent xây dựng cấu trúc hình học thông qua vòng lặp và lệnh di chuyển/đặt khối.
     ```python
     def build_house():
         agent.set_item(OAK_PLANKS, 1, 1)
         agent.set_active_slot(1)
         agent.move(UP, 1)
         for i in range(4):
             length = 4 if i % 2 != 0 else 5
             for j in range(length):
                 agent.place(DOWN)
                 agent.move(FORWARD, 1)
             agent.turn(RIGHT_TURN)

     player.on_chat("build", build_house) # Kích hoạt qua lệnh chat
     ```
  2. **Khung Game Giáo dục 2D (Pygame):** Xây dựng thế giới mở dạng top-down, sử dụng `Sprite` và `Group` để quản lý va chạm giữa nhân vật và các "Điểm tri thức". Khi va chạm, game chuyển sang trạng thái hiển thị UI câu hỏi trắc nghiệm.
     ```python
     class KnowledgePoint(pygame.sprite.Sprite):
         def __init__(self, x, y, question, answers, correct_idx):
             # Khởi tạo sprite & lưu dữ liệu quiz
             self.question, self.answers, self.correct_answer_index = question, answers, correct_idx

     # Vòng lặp chính: Xử lý va chạm & kích hoạt quiz
     collided = pygame.sprite.spritecollideany(player, knowledge_points)
     if collided:
         display_quiz(collided) # Xử lý input phím UP/DOWN/ENTER, so sánh đáp án, hiển thị feedback
     ```
  3. **Kiến trúc Công cụ Sư phạm M:EE:** 
     - `Classroom Mode`: Dashboard quản lý real-time (teleport, toggle thời tiết/quái vật, vô hiệu hóa TNT).
     - `NPC & Dialogue`: Nhúng URL ngoài hoặc thực thi lệnh game (`/give`, `/tp`), hỗ trợ JSON cho hội thoại phân nhánh.
     - `Không gian có kiểm soát`: Dùng `Allow/Deny/Border Blocks` để tạo "scaffolded sandbox", giới hạn vùng hoạt động học tập.
- **Ghi chú quan trọng:** 
  - **Lộ trình 8 tuần (1.5h/tuần):** Phân nhánh rõ ràng: *Nhóm chưa biết code* tập trung xây dựng thủ công, hợp tác nhóm, ghi chép qua Camera/Portfolio/Book & Quill. *Nhóm đã biết code* chuyển sang Code Builder/Python, tự động hóa tác vụ, lập trình hướng sự kiện (`on_chat`, `on_block_broken`).
  - **Tư duy tính toán:** M:EE hiện thực hóa 4 trụ cột CS: Phân rã (Decomposition), Nhận diện mẫu (Pattern Recognition), Trừu tượng hóa (Abstraction), Thuật toán (Algorithm) thông qua phản hồi trực quan của Agent.
  - **Đánh giá:** Portfolio xuất PDF và nhật ký dự án (Book & Quill) giúp chuyển đổi đánh giá từ kết quả cuối kỳ sang đánh giá quá trình (formative assessment).
  - **Triển khai:** Cần xây dựng "Cộng đồng thực hành" (Community of Practice) cho giáo viên, chuẩn hóa tài nguyên dùng chung và liên kết dự án với chuẩn chương trình học hiện hành để đảm bảo tính bền vững.
