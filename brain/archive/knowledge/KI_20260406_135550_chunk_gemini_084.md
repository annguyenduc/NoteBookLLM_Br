# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Lập trình xây dựng hình học đa lớp (bát giác, lục giác, kim cương) trong Minecraft Education Edition sử dụng JavaScript và Agent API.
- **Vấn đề / Câu hỏi:** 
  - Xây dựng thuật toán tổng quát để vẽ hình bát giác/lục giác đồng tâm nhiều lớp với kích thước tùy chỉnh.
  - Khắc phục hạn chế của giao diện Block Coding so với JavaScript (thiếu hàm `Math.abs()`, khó tạo vòng lặp ngược).
  - Quản lý vị trí, hành trang và luồng di chuyển của Agent khi xây dựng cấu trúc phức tạp.
- **Cách giải quyết / Logic lõi:** 
  1. **Thuật toán Bát giác đồng tâm (Grid-based):**
     - Công thức lưới tổng thể: `D = n + 2 * k` (`n`: kích thước khoảng trống trung tâm, `k`: số lớp).
     - Duyệt lớp từ ngoài vào trong (`l` từ `k` về `1`). Tính tham số lớp: `D_l = n + 2*l`, `offset = k - l`, `cut = l`.
     - Điều kiện điểm `(i, j)` thuộc bát giác lớp `l`:
     ```javascript
     const i_rel = i - offset, j_rel = j - offset;
     const cornerSum = cut - 1;
     if (i_rel >= 0 && i_rel < D_l && j_rel >= 0 && j_rel < D_l) {
         if ((i_rel + j_rel >= cornerSum) && 
             (i_rel + j_rel <= (D_l - 1) + cornerSum) && 
             (Math.abs(i_rel - j_rel) <= (D_l - 1) - cornerSum)) {
             blocks.place(currentBlock, world(startX, startY + i, startZ + j));
         }
     }
     ```
  2. **Xử lý hạn chế Block Coding:**
     - Thay `Math.abs(a - b)` bằng logic khối: `if (a > b) set hieu_so to (a - b) else set hieu_so to (b - a)`.
     - Vòng lặp ngược `for (let l = k; l >= 1; l--)` không có khối mặc định -> Khuyến nghị chuyển sang tab JavaScript cho logic phức tạp.
  3. **Quản lý Agent & Xây dựng theo lớp:**
     - Cấp phát & chọn khối: `agent.setItem(block, 64, slotIndex)` → `agent.setSlot(slotIndex)`.
     - Di chuyển & đặt khối: `agent.teleport(targetPos, EAST)` → `agent.place(DOWN)`.
     - Logic vẽ viền giảm dần cạnh chéo: Dùng 8 chuỗi `agent.move()` kết hợp `agent.place(FORWARD)`, tính toán offset vị trí bắt đầu mỗi lớp theo biến `lop`.
  4. **Thiết lập điểm gốc cố định cho Agent:**
     ```javascript
     let startPos: Position = null;
     player.onChat("start", function () {
         startPos = player.position();
         agent.teleport(positions.add(startPos, pos(1, 0, 1)), WEST);
     });
     ```
- **Ghi chú quan trọng:**
  - `blocks.place()` (Grid-based) thực thi nhanh nhưng tốn bộ nhớ tính toán cho lưới lớn. `agent.teleport()` + `agent.place()` trực quan hơn nhưng chậm do độ trễ di chuyển thực tế của Agent.
  - Khi dùng Agent, bắt buộc phải cấp đủ khối vào hành trang (`agent.setItem`) trước khi gọi hàm xây dựng.
  - Hình lục giác đứng được tối ưu bằng cách duyệt chiều rộng và thay đổi chiều cao cột theo quy luật `+2` (nửa đầu) và `-2` (nửa sau) để tạo độ dốc cạnh chính xác.
