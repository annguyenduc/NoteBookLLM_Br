# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Quy trình phát triển phần mềm hỗ trợ AI với Cursor IDE & Cơ chế quản lý tài nguyên tính toán (Compute-based Pricing).
- **Vấn đề / Câu hỏi:** Làm thế nào để tối ưu hóa workflow lập trình với AI trong Cursor, đồng thời quản lý hiệu quả giới hạn tài nguyên và chi phí của gói Pro mới?
- **Cách giải quyết / Logic lõi:** 
  - **Bộ công cụ AI & Phím tắt cốt lõi:**
    - `Ctrl+L` / `Cmd+L` (Chat): Đối thoại, giải thích code, lên kế hoạch kiến trúc.
    - `Ctrl+I` / `Cmd+I` (Agent/Composer): Thực thi tác vụ đa tệp, tạo cấu trúc dự án, chạy lệnh terminal.
    - `Ctrl+K` / `Cmd+K` (Inline Edit): Chỉnh sửa/tạo code nhanh tại vị trí con trỏ.
    - `Tab`: Chấp nhận gợi ý autocomplete.
    - `@Context`: `@tệp`, `@thư_mục`, `@Web`, `@Docs`, `@Git` để neo ngữ cảnh chính xác cho AI.
  - **Workflow phát triển & Debug:**
    1. *Khởi tạo:* Dùng Agent (`Ctrl+I`) sinh cấu trúc thư mục chuẩn (`index.html`, `css/`, `js/`).
    2. *Triển khai:* Chia nhỏ prompt. Dùng `Ctrl+K` cho logic đơn lẻ, `Ctrl+I` cho tính năng phức tạp (CRUD, `localStorage`).
    3. *Debug RCA:* Copy lỗi từ Browser Console → dán vào Chat (`Ctrl+L`). Nếu lỗi ẩn, yêu cầu AI chèn `console.log` tại các điểm then chốt để trace luồng dữ liệu.
    4. *Quản lý phiên bản:* 
       ```bash
       git init
       git add .
       git commit -m "feat: mô tả thay đổi"
       ```
       Dùng `@Commit` hoặc `@Branch` trong prompt để AI hiểu ngữ cảnh thay đổi chưa lưu.
  - **Cơ chế định giá & Quản lý Compute (Gói Pro $20):**
    - Chuyển từ *Request-based* (500 req/tháng) sang *Compute-based* (ngân sách $20 tiêu thụ trực tiếp theo giá API mô hình).
    - Ước lượng tiêu thụ: ~225 req (Claude Sonnet 4) hoặc ~650 req (GPT-4.1).
    - Hết credit: Hệ thống chuyển sang chế độ `Auto` (chất lượng không ổn định), yêu cầu bật pay-as-you-go, hoặc nâng cấp gói.
- **Ghi chú quan trọng:** 
  - **Tư duy kiến trúc:** Người dùng đóng vai trò "kiến trúc sư", AI là "thợ xây". Luôn chia nhỏ yêu cầu, review code kỹ trước khi merge.
  - **Tối ưu chi phí:** Ưu tiên mô hình nhẹ (GPT-4.1, Gemini) cho tác vụ boilerplate/UI. Chỉ dùng mô hình đắt (Sonnet 4/Opus) cho logic phức tạp hoặc debugging khó.
  - **Chuẩn hóa dự án:** Tạo file `.cursorrules` ở root để định nghĩa quy chuẩn code (naming, linting, framework preference), giúp AI sinh code nhất quán.
  - **Rủi ro kỹ thuật:** Chế độ `Auto` sau khi hết credit không phù hợp cho môi trường production. Luôn commit Git trước khi để AI refactor lớn để dễ dàng rollback.
