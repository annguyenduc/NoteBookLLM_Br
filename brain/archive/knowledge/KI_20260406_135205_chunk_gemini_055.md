# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Tích hợp API IoT cho Yolo UNO (ESP32), Logic dự báo sạt lở, Quy trình làm việc Bambu Studio, Chiến lược kiếm điểm MakerWorld cho máy Non-Bambu, và Phân tích kỹ thuật Bambu Lab A1 Mini.
- **Vấn đề / Câu hỏi:** 
  1. Yolo UNO (OhStem Block) không hỗ trợ/kết nối HTTPS ổn định với API bên ngoài.
  2. Cần mô phỏng logic cảnh báo sạt lở đất dựa trên dữ liệu thời tiết thực tế.
  3. Cách xếp chồng vật thể trong Bambu Studio & chuẩn hóa quản lý file in 3D.
  4. Kiếm điểm/tiền trên MakerWorld khi dùng máy in khác hệ (Elegoo Neptune 4).
  5. Khác biệt cốt lõi về phần cứng, workflow & kinh tế giữa A1 Mini Standard và Combo.
- **Cách giải quyết / Logic lõi:** 
  **1. Cầu nối HTTP Bridge (ThingSpeak) cho Yolo UNO:**
  - Cơ chế: `Yolo UNO (HTTP) → ThingHTTP (ThingSpeak) → API Đích (HTTPS) → Trả kết quả về`.
  - Cấu hình: Tạo ThingHTTP trên ThingSpeak, điền URL HTTPS đích, Headers (API Key). Lấy link `http://api.thingspeak.com/apps/thinghttp/send_request?api_key=...` (bắt buộc xóa chữ `s`).
  - Code MicroPython (OhStem):
  ```python
  import urequests
  url_bridge = "http://api.thingspeak.com/apps/thinghttp/send_request?api_key=YOUR_KEY"
  try:
      res = urequests.get(url_bridge)
      print(res.text)
      res.close() # Bắt buộc giải phóng RAM trên ESP32
  except Exception as e:
      print("Lỗi:", e)
  ```
  **2. Logic Dự Báo Sạt Lở (Open-Meteo API):**
  - Endpoint: `https://api.open-meteo.com/v1/forecast?latitude={LAT}&longitude={LON}&current=rain,soil_moisture_0_to_1cm`
  - Ngưỡng cảnh báo (Threshold) trên Yolo UNO:
    - `Cấp 1 (An toàn)`: Mưa = 0mm & Độ ẩm < 30% → Đèn Xanh
    - `Cấp 2 (Nguy cơ)`: Mưa > 50mm HOẶC Độ ẩm > 60% → Đèn Vàng + Còi ngắn
    - `Cấp 3 (Báo động)`: Mưa > 100mm VÀ Độ ẩm > 80% → Đèn Đỏ + Còi hú liên tục
  **3. Bambu Studio & Quản lý File 3D:**
  - Xếp chồng vật thể: Chọn 2 vật → `Ctrl + G` (Assemble) → Chọn vật con trong danh sách → `Move (M)` kéo trục Z. Tránh tính năng auto-drop.
  - Cấu trúc thư mục chuẩn: `01_My_Projects` (CAD/STL/Sliced), `02_Education` (Năm/Lớp/Học sinh), `03_Collection` (Calibration/Functional/Decor).
  - Quy tắc đặt tên: `[YYYYMMDD]_[TenDuAn]_[vX]_[TacGia].stl`
  - Xử lý GCODE: Sau khi in thành công, chỉ giữ `.3mf` (chứa setting) hoặc `.stl`. Xóa `.gcode` cũ để tránh in nhầm thông số gây hỏng bản in.
  **4. Chiến lược MakerWorld cho máy Non-Bambu:**
  - Chuyển hướng: Từ "Profile Farmer" sang "Creator" (Upload thiết kế gốc STL).
  - Tạo `.3mf` giả lập: Mở Bambu Studio → Chọn máy ảo (P1S/A1) → Import STL → Slice cơ bản (0.2mm, 15% infill) → Export 3MF.
  - Upload: Kèm STL gốc + 3MF + Ảnh thực tế in từ Neptune 4 + Mô tả trung thực: `"Tested on Neptune 4. Seeking community profile optimization for Bambu Lab printers."`
  **5. So sánh A1 Mini Standard vs Combo:**
  - **Combo:** Tích hợp AMS Lite (4 màu, tự động đổi nhựa, backup cuộn dự phòng, hỗ trợ vật liệu support chuyên dụng). Nhược điểm: Thiết kế mở dễ hút ẩm, chi phí nhựa purge (poop) cao khi đổi màu.
  - **Standard:** Giá rẻ hơn ~3-4 triệu, phù hợp in đơn sắc, chi tiết kỹ thuật, mô hình print farm số lượng lớn.
  - **Lựa chọn:** Standard cho kỹ thuật/sinh viên/ngân sách thấp. Combo cho giáo dục STEM, quà tặng đa sắc, nội dung sáng tạo.
- **Ghi chú quan trọng:** 
  - `urequests` trên ESP32 tự bỏ qua xác thực SSL để tiết kiệm RAM, nhưng dễ treo nếu payload JSON quá dài. Luôn kiểm tra độ dài dữ liệu và dùng `res.close()`.
  - ThingHTTP Free giới hạn ~15s/lần request. Không phù hợp cho ứng dụng real-time tốc độ cao.
  - Khi in đa màu trên A1 Mini, bật `Flush into Infill/Support` trong Bambu Studio để xả nhựa thừa vào phần không nhìn thấy, giảm lãng phí.
  - AMS Lite không kín, bắt buộc quản lý độ ẩm nghiêm ngặt (hộp sấy/chống ẩm) tại khí hậu Việt Nam để tránh nhựa nổ/stringing.
  - MakerWorld tính điểm dựa trên lượt tải thành công & Boost. Tuyệt đối không dùng ảnh render giả hoặc spam profile ảo (sẽ bị khóa tài khoản).
