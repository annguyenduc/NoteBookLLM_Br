# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Máy in 3D (Phần cứng, Firmware, Slicer, Khắc phục lỗi) & Phần mềm CAD cho giáo dục STEM.
- **Vấn đề / Câu hỏi:** So sánh kiến trúc máy in 3D, xử lý lỗi import profile giữa các slicer, chẩn đoán lỗi đùn thiếu nhựa (under-extrusion), và lựa chọn công cụ CAD phù hợp cho môi trường dạy học có/không internet.
- **Cách giải quyết / Logic lõi:** 
  1. **Kiến trúc & Phần cứng máy in 3D:**
     - Elegoo Neptune 4 & Bambu Lab A1 đều dùng hệ **Bed Slinger (i3 Cartesian)** khung Gantry. Khác biệt cốt lõi nằm ở hệ dẫn hướng: Neptune 4 dùng `V-Wheels` (nhựa POM, dễ mòn, cần bảo trì), A1 dùng `Linear Rails` (kim loại, bền, chính xác cao).
     - **Firmware:** 
       - `Marlin`: Xử lý toàn bộ trên MCU. Cấu hình phức tạp (sửa code C++ → Compile → Flash). Ổn định, plug-and-play.
       - `Klipper`: Kiến trúc chia tải. Máy chủ Linux (Raspberry Pi) tính toán động học/G-code, MCU chỉ thực thi lệnh. Cấu hình qua file text `printer.cfg`, hỗ trợ `Macro`, `Input Shaping`, `Pressure Advance`. Tốc độ cao, dễ tùy biến nhưng yêu cầu kiến thức Linux/SSH.
  2. **Quản lý & Import Profile Slicer (OrcaSlicer → Bambu Studio):**
     - Lỗi `0 configs imported` xảy ra do Bambu Studio chặn `System Profiles`. Chỉ chấp nhận `User Presets`.
     - *Quy trình chuẩn:* 
       ```text
       OrcaSlicer: Chọn profile → Nhấn icon Save (đĩa mềm) → Đặt tên mới (User Preset) 
       → File > Export > Export Presets... → Chọn file vừa lưu.
       Bambu Studio: File > Import > Import Configs... → Chọn file .json đã export.
       ```
     - *Can thiệp thủ công (nếu lỗi):* Copy file `.json` từ thư mục cấu hình:
       ```text
       Nguồn: %APPDATA%\OrcaSlicer\user\default\{machine|filament|process}\
       Đích:  %APPDATA%\BambuStudio\user\default\{machine|filament|process}\
       ```
  3. **Chẩn đoán & Khắc phục Under-Extrusion (Đùn thiếu nhựa):**
     - *Quy trình kiểm tra vật lý (Ngoài → Trong):*
       1. `Cuộn nhựa`: Kiểm tra rối dây, chồng chéo, lực kéo quá căng.
       2. `Ống PTFE`: Kiểm tra đoạn gập nhọn, ma sát lớn (thử đẩy tay thấy nặng/kẹt).
       3. `Extruder`: Nghe tiếng `tạch tạch` → bánh răng trượt do lực cản quá lớn hoặc nhiệt độ thấp.
       4. `Nozzle`: Đùn thủ công ở nhiệt độ in. Nhựa chảy thẳng = OK. Cong/vón cục/mảnh = tắc một phần.
     - *Xử lý phần mềm:* Nếu vật lý ổn, kiểm tra: Tăng nhiệt độ nozzle (~10°C nếu in tốc độ cao), giảm tốc độ in, tăng `Flow Ratio`, hoặc hiệu chuẩn lại `Pressure Advance (PA)` / `Flow Dynamics Calibration`.
  4. **Lựa chọn & Quy trình CAD cho STEM:**
     - `Onshape`: 100% Cloud, bắt buộc có Internet. Ưu điểm: Collaboration real-time, chạy trên mọi cấu hình, quản lý bài tập qua Link & History.
     - `FreeCAD`: Offline, Parametric Modeling. Phù hợp môi trường không mạng. Mẹo chuyển đổi giao diện: `Chuột phải vùng vẽ → Navigation Styles → TinkerCAD`.
     - *Quy trình dạy học tối ưu:* Tinkercad (Phác thảo ý tưởng/Direct Modeling) → FreeCAD/Onshape (Thiết kế chính xác/Tham số hóa & Assembly).
- **Ghi chú quan trọng:** 
  - Klipper mạnh về tốc độ & tùy biến nhưng dễ lỗi nếu sửa sai `printer.cfg`. Marlin phù hợp cho người mới hoặc môi trường cần độ ổn định cao.
  - Khi import profile máy bên thứ 3 vào Bambu Studio, nếu thiếu `Machine Definition`, một số tính năng đặc thù sẽ không hoạt động. Ưu tiên dùng OrcaSlicer cho máy non-Bambu.
  - Under-extrusion cần ưu tiên排查 (kiểm tra) nguyên nhân vật lý trước khi can thiệp thông số slicer.
  - Onshape không có chế độ Offline. FreeCAD là lựa chọn bắt buộc cho môi trường giáo dục thiếu kết nối mạng ổn định.
