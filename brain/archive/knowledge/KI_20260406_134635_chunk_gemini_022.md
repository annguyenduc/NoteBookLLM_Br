# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Tích hợp phần cứng IoT (Yolo Uno/ESP32-S3) vào quy trình Data Science, cấu hình AI IDE (Google Antigravity) để học tập có kiểm soát, và đánh giá thiết bị di động (iPad) cho công việc STEM/3D/Robotics.
- **Vấn đề / Câu hỏi:** Làm thế nào để biến Yolo Uno thành công cụ học Data, cấu hình Rules/Skills trong Antigravity để tránh "học vẹt", phân biệt Data Scientist vs Analyst, và chọn iPad thay thế laptop cho thiết kế 3D & Robotics?
- **Cách giải quyết / Logic lõi:** 
  1. **Yolo Uno trong Data Pipeline:** Đóng vai trò `Data Collection & Edge Computing`. Thu thập dữ liệu cảm biến thời gian thực, xử lý nhiễu (Kalman Filter, Moving Average), gửi qua MQTT/HTTP. Hỗ trợ TinyML (Keyword Spotting, Gesture Recognition) nhờ chip ESP32-S3.
  2. **Cấu hình Rules cho AI IDE:** Tạo file `.antigravityrules` (hoặc `.cursorrules`) tại thư mục gốc dự án thay vì tìm trong menu Settings. AI sẽ đọc file này đầu tiên để tuân thủ ngữ cảnh dự án.
  3. **Quản lý Agent Skills & Permissions:** Đặt `Artifact` và `Terminal Command` ở chế độ `Request Review` để buộc người dùng đọc code, hiểu thư viện và quy trình cài đặt môi trường. Tắt `Strict Mode` để linh hoạt khi làm việc với phần cứng/URL ngoài. Skill được kích hoạt qua Context/Prompt.
  4. **Framework Prompt học tập:** `Vai trò + Mục tiêu + Ràng buộc + Chế độ giảng dạy`. Áp dụng quy trình `Deconstruct & Reconstruct`: Lập kế hoạch -> Viết code thủ công từ gốc (NumPy) -> Đối chiếu thư viện chuẩn (Scikit-learn) -> Trực quan hóa & Kiểm chứng metrics.
  5. **So sánh thiết bị cho STEM:** iPad Air 5 (M1, 8GB RAM, Stage Manager, USB-C 10Gbps) vượt trội so với iPad tiêu chuẩn (A14/A16, 4GB RAM, Mirror only) cho CAD 3D, đa nhiệm và kết nối Hub phần cứng.
- **Ghi chú quan trọng:** 
  - **Mẫu file `.antigravityrules` tối ưu:**
    ```markdown
    # ROLE
    Expert Data Scientist & Mathematician.
    # CORE BEHAVIORS
    1. Math-First: Giải thích công thức toán học (LaTeX) trước khi viết code.
    2. Stack: Ưu tiên pandas/polars, seaborn/plotly, scikit-learn/pytorch.
    3. Code Quality: Bắt buộc Type Hinting, Docstring giải thích "Why", không dùng Magic Numbers.
    # TEACHING MODE
    Cấu trúc: Hypothesis -> Methodology -> Implementation -> Verification (Metrics).
    ```
  - **Cảnh báo phần cứng:** iPadOS không chạy được Slicer chuyên nghiệp (Cura/Prusa) và hạn chế driver USB-Serial để nạp code Arduino/ESP32. Nên dùng iPad Air 5 làm thiết bị bổ trợ, vẫn cần PC/Laptop cho khâu Slicing & Flashing.
  - **Giới hạn Yolo Uno:** Không chạy được Pandas/Scikit-learn trực tiếp do giới hạn RAM. Chỉ dùng làm trạm thu thập/xử lý sơ bộ dữ liệu trước khi đẩy lên Cloud/PC để phân tích.
