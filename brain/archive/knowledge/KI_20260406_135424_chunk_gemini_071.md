# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Kiến trúc & Lộ trình phát triển công cụ chuyển mã (Transpiler) MicroPython sang định dạng XML của Blockly cho nền tảng OhStem.
- **Vấn đề / Câu hỏi:** Làm thế nào để xây dựng trình chuyển mã đáng tin cậy, chuyển đổi mã MicroPython thành khối lệnh Blockly trực quan, hỗ trợ API phần cứng tùy chỉnh và đảm bảo không mất dữ liệu với các cấu trúc Python phức tạp?
- **Cách giải quyết / Logic lõi:** 
  - **Luồng xử lý cốt lõi:** `Mã nguồn` → `ast.parse()` (tạo AST) → `ast.NodeVisitor` (duyệt cây có trạng thái) → `xml.etree.ElementTree` (xuất XML Blockly).
  - **Tại sao dùng AST & NodeVisitor:** AST chuẩn hóa cấu trúc mã, loại bỏ chi tiết cú pháp thừa. `ast.NodeVisitor` (thay vì `ast.walk()`) cho phép duy trì stack ngữ cảnh cha-con, bắt buộc để tạo XML phân cấp chính xác.
  - **Bảng ánh xạ AST sang Blockly (Logic chuyển đổi):**
    ```python
    # ast.Assign  -> <block type="variables_set"> (target -> VAR, value -> VALUE)
    # ast.If      -> <block type="controls_if"> (test -> IF0, body -> DO0, orelse -> ELSE)
    # ast.For     -> <block type="controls_for"> (target -> VAR, iter -> FROM/TO/BY, body -> DO)
    # ast.Call    -> <block type="custom_api_type"> (args -> <value name="ARGx">)
    # ast.BinOp   -> <block type="math_arithmetic"> (op -> OP, left/right -> A/B)
    ```
  - **Cơ chế "Grey Block" (Xử lý mã không hỗ trợ):** Dùng `ast.unparse()` (Python 3.9+) để chuyển các nút AST phức tạp (lambda, try/except, class, list comprehension) ngược lại thành chuỗi mã gốc, đóng gói vào khối `raw_micropython` (màu xám, read-only). Đảm bảo chuyển đổi lossless và có thể đảo ngược.
  - **Reverse Engineering API OhStem:** Do thiếu tài liệu ánh xạ công khai, cần tạo khối trên `app.ohstem.vn`, kiểm tra mã Python sinh ra và DOM XML để xác định chính xác `type` và `name` của các khối tùy chỉnh (ví dụ: `rover_move_speed_time`, `yolobit_display_scroll`).
- **Ghi chú quan trọng:** 
  - Bắt buộc bắt `SyntaxError` từ `ast.parse()` và xác thực cấu trúc XML đầu ra trước khi trả về.
  - Ưu tiên ánh xạ hàm cấp cao nhất trước khi fallback về biểu diễn chung để tối ưu trải nghiệm người dùng.
  - Áp dụng Test-Driven Development (TDD), kiểm thử chuyển đổi 2 chiều (MicroPython ↔ XML) để đảm bảo tính trung thực của mã.
