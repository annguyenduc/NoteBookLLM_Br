# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Chuyển đổi kho lưu trữ CLI/tài liệu học tập thành `Skill-Workflow` tự trị trên Google Antigravity IDE.
- **Vấn đề / Câu hỏi:** Làm thế nào để tích hợp các kịch bản CLI tĩnh và hướng dẫn vào IDE tác nhân AI mà không gây "bão hòa ngữ cảnh" (context saturation), đồng thời đảm bảo tính tự chủ, an toàn và khả năng điều phối đa bước?
- **Cách giải quyết / Logic lõi:** 
  - **Ma trận điều khiển tác nhân:** Tách biệt 3 nguyên thủy tính toán:
    - `Rules` (`.agents/rules/`): Ràng buộc thụ động, luôn được tiêm vào system prompt (guardrails).
    - `Workflows` (`.agents/workflows/`): Chuỗi thao tác vĩ mô, kích hoạt thủ công qua lệnh `/tên-workflow`.
    - `Skills` (`.agents/skills/`): Năng lực tự kích hoạt bởi tác nhân, áp dụng **Tiết lộ lũy tiến (Progressive Disclosure)** để chỉ tải hướng dẫn nặng khi cần.
  - **Cấu trúc chuẩn `SKILL.md`:**
    ```yaml
    ---
    name: ten-ky-nang
    description: Mô tả ngắn gọn, chứa từ khóa kích hoạt ngữ nghĩa (viết ngôi thứ 3, quyết định độ chính xác định tuyến).
    ---
    # Mục tiêu
    # Hướng dẫn
    # Ràng buộc
    # Ví dụ
    ```
    Kèm thư mục hỗ trợ: `scripts/` (bash/python tất định), `references/` (tài liệu tĩnh), `examples/` (few-shot).
  - **Quy trình triển khai:**
    1. `ag init my-project` → sinh `.agents/`.
    2. Đóng gói Skill: Viết `SKILL.md` + script "hộp đen" (xử lý lỗi rõ ràng, trả STDOUT chuẩn, tránh prompt tương tác).
    3. Định nghĩa Workflow: File `.md` mô tả chuỗi bước (Audit môi trường → Gọi Skill → Chờ phê duyệt Artifact → Thực thi script).
    4. Kích hoạt: Gõ `/tên-workflow` trong Agent Manager.
  - **Kiến trúc FSM & Đa tác nhân:** Áp dụng mô hình Bộ Ba (Architect lập kế hoạch → Executor thực thi → Tech Lead đánh giá) kết hợp Cổng an toàn (`state.md`) để hard-stop nếu trạng thái không phải `USER_APPROVAL_PENDING`.
  - **Tương thích chéo:** Chuẩn `SKILL.md` dùng chung cho Antigravity, Claude Code, Cursor. Có thể symlink `~/.claude/skills/` → `~/.gemini/antigravity/skills/`.
- **Ghi chú quan trọng:** 
  - **Bảo mật:** Bắt buộc dùng công cụ `ask_user` trước lệnh rủi ro cao. Cấu hình Allow/Deny Lists cho thư mục. Browser Agent chạy sandbox.
  - **Tối ưu ngữ cảnh:** Không nhồi toàn bộ repo vào prompt. Chỉ tải `SKILL.md` khi khớp `description`. Script phải là black-box để tiết kiệm token.
  - **MCP vs Skills:** Dùng MCP cho kết nối stateful bên ngoài (DB, API). Dùng Skills cho logic cục bộ, serverless, ưu tiên nguyên tắc "less infrastructure, more intelligence".
