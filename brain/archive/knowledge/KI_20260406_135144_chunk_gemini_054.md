# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Lập trình IoT & AI nhúng trên Yolo UNO (ESP32-S3) với MicroPython: Khắc phục lỗi extension, xử lý HTTP/JSON, chuẩn hóa thư viện và tiêu chí chọn phần cứng TinyML.
- **Vấn đề / Câu hỏi:** 
  1. Lỗi xung đột tên thư viện khi port extension từ Yolo:Bit sang Yolo UNO.
  2. Lỗi `ImportError: can't import name Neopixel` và `AttributeError: 'Response' object has no attribute 'on_successed'`.
  3. Cơ chế xử lý HTTP request trong MicroPython và nguy cơ tràn RAM khi parse JSON.
  4. Tiêu chí lựa chọn board ESP32 để phát triển Embedded AI/TinyML.
- **Cách giải quyết / Logic lõi:** 
  **1. Chuẩn hóa Import & Extension:**
  - `generator.js`: Đổi namespace import thành `from yolouno_extension_https import *` và gọi đúng hàm `http_get()`/`http_post()`.
  - `__init__.py`: Bắt buộc dùng `try-except` khi import `urequests`, xử lý ngoại lệ mạng và **luôn gọi `response.close()`** để giải phóng RAM.
  - **Neopixel:** MicroPython tách riêng module này. Không dùng `from machine import Neopixel`.
    ```python
    import neopixel
    from machine import Pin
    np = neopixel.NeoPixel(Pin(48), 2) # Yolo UNO ESP32-S3 thường dùng Pin 48
    np[0] = (r, g, b); np.write()
    ```
  **2. Xử lý HTTP Request (Đồng bộ):**
  - `urequests` chạy **đồng bộ (Synchronous)**, không hỗ trợ callback `.on_successed()` hay `.on_failed()`.
  - Cấu trúc chuẩn thay thế:
    ```python
    import urequests, json
    try:
        res = urequests.get(url)
        if res.status_code == 200:
            data = json.loads(res.text) # Parse JSON
            # Xử lý data...
        res.close() # Giải phóng bộ nhớ ngay sau khi dùng
    except Exception as e:
        print("Lỗi kết nối/xử lý:", e)
    ```
  **3. Tối ưu bộ nhớ & Dữ liệu:**
  - ESP32 có RAM hạn chế. Tránh parse chuỗi JSON dài trực tiếp trên board.
  - **Mẹo:** Dùng tính năng `Parse String` trên ThingSpeak/ThingHTTP để lọc sẵn giá trị cần thiết (VD: `current.soil_moisture`) trước khi gửi về vi điều khiển.
- **Ghi chú quan trọng:** 
  - **Phần cứng AI:** Bắt buộc chọn chip **ESP32-S3** có tích hợp **PSRAM** (tối thiểu 2MB, khuyến nghị 8MB) để chứa mô hình AI. Board gợi ý: Seeed XIAO ESP32S3 Sense, Yolo UNO S3, ESP32-S3 DevKitC-1 (N16R8).
  - **Công cụ AI:** Ưu tiên **Edge Impulse** để thu thập dữ liệu, huấn luyện mô hình và xuất thư viện C++/Arduino tối ưu cho ESP32.
  - **Xử lý HTTPS:** ESP32 thường thiếu RAM/RTC để xác thực chứng chỉ SSL. Nên dùng HTTP nội bộ hoặc bridge trung gian. Nếu bắt buộc dùng HTTPS, đảm bảo firmware đã patch `urequests` hỗ trợ `verify=False` hoặc dùng socket trực tiếp.
