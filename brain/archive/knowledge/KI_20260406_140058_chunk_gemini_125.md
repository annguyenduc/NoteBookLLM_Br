# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Quản lý môi trường Python & Thiết kế kiến trúc phần mềm (Clean Architecture + CQRS)
- **Vấn đề / Câu hỏi:** 
  1. Tại sao phải cài lại thư viện trong `.venv` dù đã có ở môi trường toàn cục? Cách kiểm tra đường dẫn Python.
  2. Giải pháp tối ưu dung lượng ổ đĩa khi nhiều dự án dùng chung thư viện.
  3. Tổ chức code, luồng dữ liệu và nguyên tắc thiết kế cho ứng dụng phức tạp theo chuẩn công nghiệp.
- **Cách giải quyết / Logic lõi:** 
  - **Tính độc lập của `.venv`:** Môi trường ảo có `PATH` và thư viện riêng biệt. Thư viện global **không** tự động khả dụng trong `.venv`. Luôn phải `pip install` lại sau khi kích hoạt môi trường ảo.
  - **Kiểm tra đường dẫn cài đặt:**
    ```bash
    where python   # Windows
    which python   # macOS/Linux
    ```
    Hoặc kiểm tra trực tiếp trong Python:
    ```python
    import sys
    print(sys.executable)
    ```
  - **Tối ưu dung lượng & Quản lý Dependency:** 
    - ❌ Không dùng shared `.venv` (gây xung đột version nghiêm trọng).
    - ✅ `pip` tự động cache gói đã tải (`%LOCALAPPDATA%\pip\Cache`), giảm tải lại từ internet.
    - ✅ Dùng `pipenv` hoặc `Poetry`: tự quản lý `.venv`, dùng cache toàn cục, khóa version chặt chẽ (`Pipfile.lock`/`poetry.lock`), tránh trùng lặp tải xuống.
  - **Kiến trúc Clean Architecture & CQRS:**
    - Phân tầng & hướng phụ thuộc: `Presentation` → `Application` → `Domain` ← `Infrastructure`
    - **Dependency Inversion:** Domain định nghĩa interface, Infrastructure cài đặt cụ thể.
    - **CQRS Pattern:** Tách biệt `Command` (Write/Thay đổi state) và `Query` (Read/Không đổi state).
    - Cấu trúc thư mục chuẩn:
      ```text
      src/
      ├── domain/          # Entities, Events, Business Rules (Pure logic, không phụ thuộc ngoài)
      ├── application/     # Commands, Queries, Handlers, Interfaces (Abstractions)
      ├── infrastructure/  # DB, Files, Sync (Implement interfaces của Domain)
      └── api/             # Routes, Middleware, Models (Presentation Layer)
      ```
    - Luồng xử lý mẫu: `UI` → `API Route` → `Command Handler` → `Domain Entity` (validate & raise event) → `Repository` (Infrastructure) → `Publish Event` → `Response`.
- **Ghi chú quan trọng:** 
  - Luôn tạo `.venv` riêng cho mỗi dự án để đảm bảo tính tái lập, dễ triển khai và tránh "Dependency Hell".
  - `pip cache purge` chỉ dùng khi cần giải phóng dung lượng gấp, sẽ buộc tải lại từ PyPI ở lần cài đặt sau.
  - Clean Architecture giúp dễ dàng thay đổi công nghệ (DB, UI) mà không ảnh hưởng logic nghiệp vụ, đồng thời tăng khả năng test unit độc lập từng tầng.
  - Tích hợp Event Sourcing & Audit Trail ở tầng Domain/Infrastructure giúp theo dõi lịch sử thay đổi, cảnh báo và khôi phục dữ liệu an toàn.
