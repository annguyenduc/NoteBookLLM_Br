# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Tải khóa học Coursera qua CLI & Khắc phục lỗi tương thích công cụ `coursera-dl`
- **Vấn đề / Câu hỏi:** Công cụ `coursera-dl` (bản gốc & các fork cũ) không còn tương thích với API Coursera hiện tại, gây ra chuỗi lỗi: `JSONDecodeError` (API thay đổi/trả về HTML), `unrecognized arguments: --cauth` (thay đổi cơ chế xác thực), và `ValueError: invalid mode: 'rU'` (xung đột cú pháp Python 3.11+).
- **Cách giải quyết / Logic lõi:** 
  - **Giải pháp tối ưu (Khuyến nghị):** Chuyển sang `yt-dlp`. Công cụ được cập nhật hàng tuần, hỗ trợ Coursera ổn định và tự động lấy cookie từ trình duyệt.
    ```bash
    pip install --upgrade yt-dlp
    yt-dlp --cookies-from-browser chrome "https://www.coursera.org/learn/<course-slug>"
    ```
    *(Thay `chrome` bằng `firefox` hoặc `edge` tùy trình duyệt đang đăng nhập Coursera)*
  - **Xử lý lỗi nếu bắt buộc dùng `coursera-dl`:**
    1. Lỗi `unrecognized arguments: --cauth`: Phiên bản mới dùng file cookie. Cài extension "Get cookies.txt LOCALLY" → xuất `cookies.txt` → chạy: `coursera-dl -c cookies.txt <course-slug>`
    2. Lỗi `ValueError: invalid mode: 'rU'`: Mở file `.../site-packages/coursera/cookies.py`, tìm `open(cookies_file, 'rU')` và sửa thành `open(cookies_file, 'r')`.
- **Ghi chú quan trọng:** 
  - Lỗi `JSONDecodeError: Expecting value` thường do Coursera chặn request cũ hoặc cookie hết hạn, khiến tool nhận về trang HTML thay vì JSON.
  - Luôn đảm bảo đã đăng nhập Coursera trên trình duyệt trước khi chạy lệnh trích xuất cookie.
  - `yt-dlp` là giải pháp bền vững dài hạn, tránh được tình trạng "chết repo" hoặc lỗi API như các fork `coursera-dl` cũ.
