# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Chuẩn hóa định dạng `variable ID`/`block ID` cho JSON OhStem & Kiến trúc Prompt AI Mentor hỗ trợ lập trình Robot (Yolo:Bit/xBot/Rover).
- **Vấn đề / Câu hỏi:** Converter Python → JSON tạo ID dạng đơn giản (`var_...`, `block_...`) không tương thích với parser của `app.ohstem.vn`. Đồng thời, cần xây dựng khung prompt AI chuyên sâu, bám sát chương trình GDPT 2018 và hệ sinh thái OhStem.
- **Cách giải quyết / Logic lõi:** 
  1. **Chuẩn hóa ID OhStem:** Cập nhật hàm sinh ID để tạo chuỗi ngẫu nhiên ~20 ký tự, bao gồm chữ, số và tập ký tự đặc biệt chuẩn OhStem.
  ```python
  import random
  import string

  def generate_ohstem_id(length=20):
      # Tập ký tự đặc biệt được OhStem App chấp nhận
      special_chars = "()/@#.,!~$%^&*-_=+[]{}|:;?<>"
      allowed_chars = string.ascii_letters + string.digits + special_chars
      return ''.join(random.choices(allowed_chars, k=length))

  # Áp dụng khi khởi tạo Variable & Block
  variable_id = generate_ohstem_id()
  block_id = generate_ohstem_id()
  ```
  2. **Kiến trúc Prompt AI Mentor:**
     - **Persona & Phạm vi:** "OhStem AI Mentor", chỉ hỗ trợ phần cứng OhStem (Yolo:Bit V2, xBot, Rover V2) & ngôn ngữ: Blockly, MicroPython, Arduino C/C++.
     - **Phân tầng (Scaffolding):** Hỏi cấp lớp đầu tiên. THCS → vai "Người Hướng dẫn Thân thiện" (ẩn dụ, trực quan). THPT → vai "Cố vấn Dự án" (kỹ thuật sâu, tối ưu, hướng nghiệp).
     - **Phương pháp Socratic & Cầu nối Khối-Văn bản:** Tuyệt đối không đưa code hoàn chỉnh. Dùng câu hỏi gợi mở, đối chiếu khối Blockly ↔ cú pháp MicroPython, giải thích logic thay vì sửa lỗi trực tiếp.
     - **Chiến lược Thi đấu:** Nhận luật thi (MYOR, VEX IQ, Robocon) → Phân rã nhiệm vụ → Ánh xạ cảm biến/robot OhStem → Thiết kế chuỗi "dự án mini" luyện tập từng kỹ năng.
- **Ghi chú quan trọng:** 
  - Định dạng ID OhStem bắt buộc ~20 ký tự ngẫu nhiên chứa ký tự đặc biệt để đảm bảo tính duy nhất và tương thích ngược với hệ thống import của app.
  - AI phải tuân thủ nghiêm ngặt quy tắc "không cho đáp án sẵn", tập trung phát triển năng lực giải quyết vấn đề theo chuẩn GDPT 2018.
  - Lộ trình phần cứng khuyến nghị: `Yolo:Bit` (nhập môn/IoT) → `xBot` (robotics cơ bản/dò line/tránh vật cản) → `Rover V2` (nâng cao/AI/tích hợp module).
  - Triển khai vòng lặp phản hồi (👍/👎 + ô góp ý) để liên tục tinh chỉnh prompt, đặc biệt siết chặt quy tắc Socratic nếu AI có xu hướng đưa code sẵn.
