### 1. Cấu hình Mạng & Kết nối SSH cho Máy in 3D (Neptune 4)
- **Chủ đề chính:** Thiết lập kết nối Point-to-Point/Network Bridge trên Windows và truy cập SSH vào hệ điều hành Klipper.
- **Vấn đề / Câu hỏi:** Mạng bị ngắt khi tạo Bridge, xung đột ICS, không tìm thấy IP máy in khi cắm trực tiếp LAN vào laptop.
- **Cách giải quyết / Logic lõi:** 
  1. **Tắt ICS (Bắt buộc):** `Properties` (Wi-Fi) → Tab `Sharing` → Bỏ tích `"Allow other network users..."` → `OK`.
  2. **Tạo Bridge:** Giữ `Ctrl` chọn cả `Wi-Fi` & `Ethernet` → Chuột phải → `Bridge Connections`. Chờ 10-30s để Windows tạo card ảo.
  3. **Dò IP:** Dùng CMD `arp -a` hoặc `Advanced IP Scanner` (quét `192.168.1.1-254`). Nếu không thấy, đặt IP tĩnh cho Ethernet: `192.168.1.10`, Subnet `255.255.255.0`.
  4. **Kết nối Putty:** Host: IP máy in, Port: `22`, Type: `SSH`. Login: `mks` / Pass: `makerbase`.
  ```bash
  # Lệnh kiểm tra & xử lý lỗi sau khi SSH thành công
  systemctl status klipper
  sudo service klipper restart
  tail -f ~/printer_data/logs/klippy.log
  ```
- **Ghi chú quan trọng:** IPv4 của `Network Bridge` phải để `Obtain an IP address automatically`. Nếu Putty báo `Connection refused`, kiểm tra lại lớp mạng hoặc khởi động lại dịch vụ SSH/máy in.

### 2. Khác biệt Firmware Neptune 4 vs Neptune 4 Pro
- **Chủ đề chính:** Tương thích phần mềm & phần cứng giữa hai phiên bản máy in 3D cùng dòng.
- **Vấn đề / Câu hỏi:** Có thể dùng chéo firmware giữa bản thường và bản Pro không?
- **Cách giải quyết / Logic lõi:** **KHÔNG THỂ dùng chéo** do khác biệt phần cứng cốt lõi:
  - **Bàn nhiệt:** Pro có 2 vùng gia nhiệt độc lập (`heater_bed1`), bản thường chỉ 1. Nạp nhầm gây lỗi `ADC out of range` hoặc `Heater not found`.
  - **Cơ khí & Input Shaping:** Pro dùng thanh ray kim loại, bản thường dùng bánh xe POM. Thông số bù rung (Input Shaping) được tinh chỉnh riêng. Dùng sai firmware gây lỗi `ghosting` hoặc rung lắc mạnh.
- **Ghi chú quan trọng:** Khi nạp lại Image cho module eMMC, bắt buộc chọn đúng file firmware tương thích với model máy.

### 3. Ứng dụng Data Analytics vào Quản lý Đào tạo E-learning
- **Chủ đề chính:** Chuyển đổi từ đào tạo hành chính sang Learning Analytics Specialist.
- **Vấn đề / Câu hỏi:** Làm sao dùng dữ liệu để đánh giá phản hồi, đảm bảo công bằng chấm điểm và theo dõi tiến độ chi nhánh?
- **Cách giải quyết / Logic lõi:** 
  - **Phản hồi:** Tính chỉ số NPS, dùng Scatter Plot phân tích tương quan giữa `Thời lượng xem video` và `Drop-off rate`.
  - **Công bằng đánh giá:** Áp dụng Độ lệch chuẩn ($\sigma$) & Phân phối chuẩn để phát hiện đề thi quá dễ/gian lận. Dùng SQL (`Item Analysis`) lọc câu hỏi có tỷ lệ sai bất thường.
  - **Theo dõi tiến độ:** Xây dựng Dashboard tự động (Looker Studio/Power BI) kết nối trực tiếp dữ liệu E-learning.
  - **Dự án thực chiến:** Tạo Mock Data (CSV) → Xử lý bằng SQL (`JOIN`, `GROUP BY` tính điểm trung bình/tỷ lệ hoàn thành theo chi nhánh) → Trực quan hóa Dashboard.
- **Ghi chú quan trọng:** Dữ liệu giúp tự động hóa báo cáo, phát hiện sớm điểm bất thường trong ngân hàng đề và nâng cao tính chiến lược khi phỏng vấn/quản lý.

### 4. Xử lý nhãn lặp trong Python (Log/Data Parsing)
- **Chủ đề chính:** Định vị chính xác & cắt dữ liệu khi nhãn (label) xuất hiện nhiều lần.
- **Vấn đề / Câu hỏi:** Hàm `.index()` chỉ trả về vị trí đầu tiên, không phân biệt được các `"Test case 1"` lặp lại.
- **Cách giải quyết / Logic lõi:** 
  1. **Dùng tham số `start`:** `data.index("Test case 1", first_idx + 1)`
  2. **Lấy tất cả vị trí (List Comprehension):**
     ```python
     all_indices = [i for i, x in enumerate(data) if x == "Test case 1"]
     last_test_case = data[all_indices[-1]:]
     ```
  3. **Tìm theo ngữ cảnh (Contextual Search):**
     ```python
     target_idx = next(i for i, line in enumerate(data) if "Test case 1" in line and "09:00" in line)
     ```
- **Ghi chú quan trọng:** Kết hợp `enumerate` và điều kiện phụ (`next` + `in`) giúp xử lý log/dữ liệu STEM chính xác hơn so với `.index()` đơn thuần, đặc biệt khi dữ liệu có timestamp hoặc ID đi kèm.
