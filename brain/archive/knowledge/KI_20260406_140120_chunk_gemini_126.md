# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Thiết kế & Triển khai Hệ thống Quản lý Lớp học Offline theo Clean Architecture + DDD + CQRS
- **Vấn đề / Câu hỏi:** Làm thế nào để xây dựng ứng dụng quản lý lớp học hoạt động ổn định offline, tối ưu cho máy cấu hình thấp (2GB RAM), đảm bảo tính bảo trì, mở rộng và tuân thủ nghiệp vụ chặt chẽ?
- **Cách giải quyết / Logic lõi:** 
  Áp dụng kiến trúc phân tầng rõ ràng với Dependency Inversion, tách biệt luồng đọc/ghi (CQRS) và xử lý bất đồng bộ qua Event Sourcing.
  ```python
  # 1. DOMAIN LAYER: Chứa business rules thuần túy, không phụ thuộc bên ngoài
  class HocSinh:
      @classmethod
      def create(cls, ma_hs: str, ten: str, ...):
          if not cls._is_valid_ma_hs(ma_hs): # Validate format HS-YYYY-MM-NNN
              raise DomainException("Mã học sinh không hợp lệ")
          student = cls(ma_hs=ma_hs, ten=ten, ...)
          student.raise_event(StudentCreatedEvent(student)) # Event Sourcing
          return student

  # 2. APPLICATION LAYER: Orchestrate use case, xử lý Command/Query
  class CreateStudentHandler:
      async def handle(self, command: CreateStudentCommand):
          student = HocSinh.create(ma_hs=command.ma_hoc_sinh, ten=command.ho_ten, ...)
          saved = await self.repository.save(student) # Gọi interface
          await self.event_publisher.publish(StudentCreatedEvent(saved.ma_hs))
          return HandlerResult.success(saved.to_dict())

  # 3. INFRASTRUCTURE LAYER: Implement interface, xử lý kỹ thuật (DB, Sync)
  class SQLiteHocSinhRepository(IHocSinhRepository):
      async def save(self, student: HocSinh) -> HocSinh:

          student.mark_as_persisted()
          return student
  ```
  **Luồng dữ liệu chuẩn:** UI → API (chuyển thành Command) → Handler → Domain Entity (validate & raise event) → Repository (lưu DB) → Event Handler (xử lý audit/notification) → Trả kết quả về UI.
- **Ghi chú quan trọng:** 
  - **Tối ưu tài nguyên:** Giới hạn RAM ~1800MB, dùng DB connection pool, LRU cache nén dữ liệu, thiết kế Offline-first (sync qua LAN/USB khi có mạng).
  - **Testing Strategy:** 95+ unit tests pass. Chạy riêng theo tầng: `pytest tests/unit/domain/`, `application/`, `integration/`. Coverage report dạng HTML.
  - **Triển khai nhanh:** `pip install -r requirements.txt` → Console: `python main.py` | Web: `uvicorn app:app --host 0.0.0.0 --port 8000`.
  - **Nguyên tắc cốt lõi:** Domain là lõi bất biến, không import framework/DB. Infrastructure chỉ implement interface do Domain/Application định nghĩa → Dễ dàng swap DB (SQLite ↔ PostgreSQL) hoặc thay đổi UI mà không ảnh hưởng logic nghiệp vụ.
