# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Triển khai Model Context Protocol (MCP) để AI Agent tự động lập trình & gỡ lỗi ứng dụng Python trong IDE (Cursor/VS Code).
- **Vấn đề / Câu hỏi:** Làm thế nào chuẩn hóa giao tiếp giữa AI Agent và các công cụ phát triển (hệ thống tệp, terminal, Git, debugger) để tự động hóa quy trình viết mã, kiểm thử và sửa lỗi một cách an toàn, có kiểm soát?
- **Cách giải quyết / Logic lõi:** 
  - **Kiến trúc MCP:** Mô hình `Client-Host-Server`. Host (IDE như Cursor) quản lý bảo mật & vòng đời; Client (AI Agent) kết nối 1-1 với Server; Server chuẩn hóa việc phơi bày `Tools` (hành động), `Resources` (ngữ cảnh/dữ liệu), `Prompts` (mẫu lệnh).
  - **Công cụ cốt lõi cho Python Dev:** 
    - Hệ thống tệp: `read_file`, `write_file`, `list_directory`
    - Thực thi lệnh: `execute_command` (bắt buộc sandbox)
    - Git: `git_commit`, `git_push`, `git_diff`
    - Phân tích & Gỡ lỗi: Tích hợp LSP (tìm định nghĩa, diagnostics) và DAP (đặt breakpoint, inspect variables, step execution).
  - **Triển khai Server bằng Python SDK (`fastmcp`):**
    ```python
    from mcp.server.fastmcp import FastMCP, Context
    mcp = FastMCP("PythonDevServer")

    @mcp.tool()
    async def run_sandboxed_cmd(command: str, allowed: list[str], ctx: Context) -> str:
        if command.split()[0] not in allowed:
            return "Error: Command not in allowlist."
        ctx.info(f"Executing: {command}")
        # Logic thực thi trong sandbox (Docker/WASM) & trả kết quả
        return "Command output..."
    ```
  - **Triển khai Server bằng TypeScript SDK (Zod validation):**
    ```typescript
    import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
    import { z } from "zod";
    const server = new McpServer({ name: "SafeFileServer", version: "1.0.0" });

    server.tool("read_file", {
        path: z.string().describe("Relative file path"),
        root: z.string().describe("Project root for path safety")
    }, async ({ path, root }) => {
        // Kiểm tra path traversal & đọc file
        return { content: [{ type: "text", text: "File content..." }] };
    });
    ```
  - **Quản lý trạng thái (Stateful Tools):** Giao thức MCP mặc định không trạng thái. Với công cụ có trạng thái như Debugger (DAP), Server phải tự duy trì `session_id`, cache trạng thái (breakpoints, call stack) và ánh xạ các lệnh gọi MCP liên tiếp vào cùng một phiên gỡ lỗi.
- **Ghi chú quan trọng:** 
  - 🔒 **Bảo mật bắt buộc:** Áp dụng sandbox cho thực thi lệnh, xác thực/làm sạch đầu vào (chống path traversal & prompt injection), nguyên tắc đặc quyền tối thiểu, và luồng phê duyệt người dùng (Human-in-the-loop) cho hành động ghi/xóa.
  - 🛠 **Thiết kế công cụ:** Giữ tính nguyên tử (atomic), mô tả tham số & giá trị trả về rõ ràng (docstring/Zod schema), xử lý lỗi có cấu trúc để LLM dễ hiểu và retry.
  - 🔄 **Tích hợp IDE:** Cursor & VS Code Agent Mode hỗ trợ MCP native. Cấu hình server trong `mcp.json` (global/project) để AI tự động khám phá capability.
  - 🧠 **Vai trò của Framework Agentic:** MCP chỉ là lớp kết nối công cụ. Cần kết hợp với LangChain/AutoGen/LangGraph để xử lý logic lập luận, lập kế hoạch đa bước, quản lý bộ nhớ dài hạn và điều phối luồng công việc phức tạp.
