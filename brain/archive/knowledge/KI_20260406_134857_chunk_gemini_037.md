# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Cấu hình mạng, hiệu chuẩn & khắc phục sự cố máy in 3D FDM (Elegoo Neptune 4/Klipper) trên OrcaSlicer/Bambu Studio & Nguyên tắc quản lý kho vật tư STEM.
- **Vấn đề / Câu hỏi:** 
  1. Nhiệt độ lớp đầu tiên (First Layer) nên cao hay thấp hơn các lớp sau?
  2. Khắc phục lỗi mất kết nối giữa Slicer và máy in 3D.
  3. Hiệu chuẩn Flow Rate thủ công cho máy chạy Klipper.
  4. Cấu hình WiFi qua SSH khi thiếu `nmtui` & xử lý tương thích Wi-Fi 6.
- **Cách giải quyết / Logic lõi:** 
  **1. Tối ưu First Layer (FDM):**
  - Nhiệt độ Nozzle: Cao hơn `5°C~10°C` so với lớp thường.
  - Nhiệt độ Bed: Cao hơn `0°C~5°C`.
  - Quạt tản nhiệt (Part Cooling Fan): `0%` ở lớp đầu, tăng dần từ lớp 2-3.
  - *Lý do:* Tăng độ lỏng nhựa, kéo dài thời gian kết dính, giảm ứng suất co ngót (warping).

  **2. Khắc phục mất kết nối Slicer:**
  - Đảm bảo PC & máy in cùng mạng LAN/Wi-Fi, tắt VPN.
  - Cập nhật IP mới vào Slicer: `Host Type` chọn `Octo/Klipper` hoặc `OctoPrint`.
  - Nếu lỗi `Upload & Print`: Thêm port API trực tiếp vào IP → `192.168.1.X:7125`.
  - Kiểm tra Windows Firewall cho phép OrcaSlicer qua mạng Private/Public.

  **3. Hiệu chuẩn Flow Rate (Klipper - Bán thủ công):**
  - Slicer tạo mẫu in (Pass 1/Pass 2) → In ra → Quan sát bề mặt bằng mắt → Chọn miếng mịn nhất.
  - Công thức tính: `Flow Ratio Mới = Flow Ratio Cũ × (100 + Số_trên_miếng_đẹp_nhất) / 100`
  - *Ví dụ:* Cũ `0.98`, chọn miếng `+5` → Mới = `0.98 × 1.05 = 1.029`. Cập nhật vào Filament Settings.

  **4. Cấu hình WiFi qua SSH (Linux/Klipper):**
  - Khi `nmtui: command not found`, chỉnh sửa trực tiếp `wpa_supplicant`.
  - Mở file: `sudo nano /etc/wpa_supplicant/wpa_supplicant-wlan0.conf`
  - Cấu hình tối ưu cho Wi-Fi 6 (bỏ `key_mgmt` để driver tự đàm phán bảo mật):
    ```text
    country=VN
    ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
    update_config=1
    network={
        ssid="Tên_WiFi_Thực_Tế"
        psk="Mật_Khẩu_Thực_Tế"
        scan_ssid=1
    }
    ```
  - Lưu & thoát: `Ctrl+O` → `Enter` → `Ctrl+X`
  - Xử lý xung đột file: `sudo rm /etc/wpa_supplicant/wpa_supplicant.conf` (chỉ giữ file `-wlan0.conf`).
  - Áp dụng: `sudo reboot` → Rút dây LAN → Kiểm tra IP mới trên màn hình máy.

- **Ghi chú quan trọng:** 
  - Máy Klipper không có cảm biến LiDAR như Bambu Lab, mọi hiệu chuẩn (Flow, Pressure Advance) đều phải in mẫu và nhập thủ công.
  - Bambu Studio kết nối máy bên thứ 3 qua giao thức giả lập `OctoPrint`. Tab `Device` thường không hiện camera, nên mở trình duyệt web truy cập IP máy để giám sát tiến độ.
  - Luôn rút dây LAN sau khi cấu hình WiFi thành công để tránh xung đột giao diện mạng.
  - Quản lý kho STEM: Nên phân loại theo nhóm chức năng (cắt/gọt, điện tử, trang trí), theo dõi cột `Trạng thái` & `Số lượng tối thiểu` trên Excel/Sheets để chủ động mua bổ sung.
