- **Cách giải quyết / Logic lõi:** 
  - **Python List vs Numpy Slicing:** 
    - List dùng truy cập lồng `[i][j]` và List Comprehension để tách cột. Numpy dùng slicing đa chiều `[i, j]` cho tốc độ xử lý ma trận tối ưu.
    ```python
    # Tách cột 0 từ List of Lists
    names = [m[0] for m in movie_entries]
    # Ép kiểu sang Numpy Array
    names_np = np.array(names)
    
    # Slicing trực tiếp trên Numpy (yêu cầu dữ liệu đã đồng nhất kiểu)
    col_0 = data_np[:, 0]
    ```
  - **Chuẩn hóa Vector & Tính Norm:** 
    - `np.linalg.norm(arr, axis=1)` tính độ dài (chuẩn L2) cho từng hàng. `axis=1` bắt buộc để áp dụng theo hàng, tránh gom toàn ma trận thành 1 giá trị.
    - Công thức chuẩn hóa: `u = v / np.linalg.norm(v)` (đưa vector về độ dài 1, giữ nguyên hướng).
  - **Xử lý dữ liệu hỗn hợp:** Tách phần tử bằng List trước, sau đó ép kiểu sang Numpy. Với cột số, dùng `.astype(float)` để tính toán.
  - **File `.ipynb` & `antigravity`:** `.ipynb` thực chất là file JSON chứa cells, metadata, outputs. Cần Kernel (IPython) để thực thi. `import antigravity` chỉ là Easter Egg mở trình duyệt, không phải IDE hay công cụ xử lý file.
  - **Khôi phục Firmware Neptune 4:** Gói cập nhật USB (`.zip`) chỉ là bản vá, yêu cầu OS phải hoạt động. Khi OS sập, bắt buộc dùng file `eMMC Image` (`.img`) từ hãng + mạch chuyển đổi `MKS eMMC to USB` để nạp lại hệ điều hành trực tiếp từ máy tính (dùng BalenaEtcher).
- **Ghi chú quan trọng:** 
  - Numpy yêu cầu đồng nhất kiểu dữ liệu trong một mảng. Dữ liệu hỗn hợp (string + float) phải tách riêng trước khi convert.
  - `axis=1` trong Numpy là thao tác theo hàng (ngang), `axis=0` là theo cột (dọc). Hiểu sai sẽ gây lỗi `shape mismatch`.
  - Không thể "cứu" máy in 3D bị brick OS bằng USB thông thường. Bắt buộc can thiệp phần cứng (đọc/ghi eMMC trực tiếp).
