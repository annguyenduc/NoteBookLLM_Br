# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Triển khai Model Context Protocol (MCP) trên Cursor, tích hợp quy trình TDD, tối ưu chi phí vận hành, và kỹ thuật bơm ngữ cảnh (Context Injection) cho dự án cụ thể.
- **Vấn đề / Câu hỏi:** Làm thế nào để xây dựng MCP Server, tích hợp vào Cursor, áp dụng TDD để phát triển, tối ưu chi phí hạ tầng, và cấu hình ngữ cảnh dự án để AI hỗ trợ lập trình chính xác?
- **Cách giải quyết / Logic lõi:** 
  1. **Bản chất MCP vs AI Agent:** AI Agent là thực thể tự chủ thực thi tác vụ. MCP là giao thức chuẩn hóa (mô hình Client-Server) đóng vai trò "cầu nối", giúp Agent truy cập công cụ/dữ liệu ngoài qua `stdio` (cục bộ) hoặc `SSE/HTTP` (từ xa).
  2. **Triển khai MCP Server cơ bản (Python - Stdio):**
     ```python
     import sys, json

     # 1. Khai báo công cụ (Cursor đọc khi khởi động)
     print(json.dumps({
         "type": "tool_declarations",
         "tools": [{"name": "summarize_url", "description": "...", "parameters": {"type": "object", "properties": {"url": {"type": "string"}}, "required": ["url"]}}]
     })); sys.stdout.flush()

     # 2. Vòng lặp nhận yêu cầu & trả kết quả
     for line in sys.stdin:
         req = json.loads(line)
         if req.get("type") == "tool_call":
             # Xử lý nghiệp vụ...
             res = {"type": "tool_result", "id": req["id"], "output": "kết quả xử lý"}
             print(json.dumps(res)); sys.stdout.flush() # Bắt buộc flush
     ```
  3. **Cấu hình trong Cursor:** Vào Settings → MCP/Agents → Thêm server mới. Chọn Transport: `stdio`, Command: `python path/to/server.py`. Cursor tự động gọi công cụ qua stdin/stdout.
  4. **Áp dụng TDD & Tối ưu chi phí:**
     - *TDD:* Viết Test (Đỏ) → Code tối thiểu vượt test (Xanh) → Refactor (Sạch). Mock tầng DB/Service để test unit độc lập.
     - *Chi phí:* Ưu tiên `stdio` cho local (miễn phí). Nếu cần remote, dùng Serverless (Lambda/Cloud Functions) trả tiền theo request. Áp dụng caching, rate-limiting API ngoài và modular hóa code.
  5. **Context Injection cho Dự án (Ví dụ: Vietnamese Class Manager):** Tạo file `cursor.json` hoặc tài liệu cấu trúc để Cursor hiểu sâu codebase:
     ```json
     {
       "entry_points": ["main.py", "app/main.py"],
       "architecture": {"type": "Clean Architecture", "pattern": "Service Layer", "db": "SQLite (1 DB/giáo viên)"},
       "data_management": {"schema_new": "app/database/models.py", "migration": "app/migrations/"},
       "testing_strategy": {"methodology": "TDD", "dir": "tests/"},
       "bug_history": [{"id": "BUG 1", "desc": "Double Hashing", "fix": "Kiểm tra hex64 trước khi hash lại"}]
     }
     ```
- **Ghi chú quan trọng:** 
  - Luôn gọi `sys.stdout.flush()` sau mỗi `print()` trong MCP Server, nếu không Cursor sẽ treo chờ phản hồi.
  - MCP không thay thế AI Agent mà "trao quyền" cho Agent bằng cách chuẩn hóa giao tiếp với hệ thống ngoài.
  - Cursor đọc ngữ cảnh từ file cấu hình, `README.md` và cấu trúc thư mục. Cung cấp rõ ràng kiến trúc, vai trò user, schema DB và lịch sử bug giúp AI gợi ý code, refactor và viết test chính xác hơn.
  - TDD giảm đáng kể chi phí sửa lỗi dài hạn, đặc biệt hiệu quả khi phát triển các module MCP có logic nghiệp vụ phức tạp.
