# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Git Version Control & Ứng dụng AI trong học lập trình Python
- **Vấn đề / Câu hỏi:** 
  1. Phân biệt cơ chế hoạt động của các tùy chọn trong `git rebase -i`, khái niệm `pull request`, `fork` và lệnh force push.
  2. Khả năng hỗ trợ học & thực hành Python của mô hình AI (Gemini Advanced).
- **Cách giải quyết / Logic lõi:** 
  - **Git Interactive Rebase (`git rebase -i`):**
    - `pick`: Mặc định. Giữ nguyên commit và áp dụng tuần tự lên nhánh đích.
    - `squash`: Gộp commit hiện tại với commit ngay trước nó, mở trình soạn thảo để chỉnh sửa thông điệp commit gộp.
    - `fixup`: Gộp commit nhưng tự động loại bỏ thông điệp của commit mới, giữ nguyên thông điệp của commit gốc.
  - **Quy trình & Lệnh Git cốt lõi:**
    - `Pull Request (PR)`: Đề xuất hợp nhất (merge) thay đổi từ nhánh cá nhân vào nhánh chính, tạo không gian review & thảo luận trước khi tích hợp.
    - `Fork`: Tạo bản sao độc lập của repo trên nền tảng remote (GitHub/GitLab) để thử nghiệm mà không ảnh hưởng repo gốc.
    - Force Push: `git push -f` (hoặc `--force`) ghi đè lịch sử remote, bỏ qua cơ chế kiểm tra an toàn của Git.
  - **Hỗ trợ học Python với AI:**
    - AI hỗ trợ: Giải thích cú pháp/logic, debug lỗi, đề xuất thư viện, sinh code mẫu, giải thích khái niệm trừu tượng và tạo bài tập thực hành theo cấp độ.
- **Ghi chú quan trọng:** 
  - `git push -f` có nguy cơ gây mất dữ liệu vĩnh viễn nếu đồng nghiệp đã pull nhánh đó. Chỉ dùng khi đã xác nhận tuyệt đối an toàn.
  - Code sinh bởi AI có thể chưa tối ưu hoặc chứa lỗi nhỏ. Luôn kiểm tra, chạy thử và hiểu rõ luồng xử lý trước khi áp dụng vào dự án thực tế.
  - Giấy xác nhận nhập học (`Enrollment Verification`) không chứa dữ liệu tài chính. Biên lai học phí (`Tuition Receipt`) cần được cấp chính thức từ phòng kế toán nhà trường; AI chỉ có thể tạo mẫu giả định dựa trên template.
