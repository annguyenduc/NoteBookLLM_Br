# TÓM TẮT TRI THỨC KỸ THUẬT
- **Chủ đề chính:** Tối ưu phần cứng ESP32-S3 (N16R8), xác định tính tương thích mạch mở rộng, chiến lược Firmware (OhStem vs Standard) và tích hợp AI Agent (Google Antigravity) cho phát triển nhúng & STEM.
- **Vấn đề / Câu hỏi:** Làm sao chọn đúng cấu hình ESP32-S3, tránh sai lệch pinout khi dùng mạch mở rộng, và quyết định giữ firmware gốc hay cài firmware chuẩn để cân bằng giữa giáo dục đại trà và phát triển AI chuyên sâu?
- **Cách giải quyết / Logic lõi:** 
  - **Chuẩn phần cứng & Tương thích:** 
    - ESP32-S3 N16R8 (16MB Flash, 8MB PSRAM) là cấu hình tối đa cho Edge AI. Chip hoạt động ở logic **3.3V** (cảnh báo chập nếu ghép trực tiếp với shield/cảm biến 5V cũ).
    - Mạch mở rộng phải khớp chuẩn **44 chân** (YD-ESP32-S3). Loại 38P (ESP32 thường) sẽ lệch sơ đồ chân, gây treo hoặc chập cháy.
  - **Cấu hình Arduino IDE chuẩn:**
    ```ini
    Board: ESP32S3 Dev Module
    USB CDC On Boot: Enabled
    Flash Size: 16MB (128Mb)
    Partition Scheme: 16M Flash (3MB APP/9.9MB FATFS)
    PSRAM: OPI PSRAM
    ```
  - **Chiến lược Firmware:**
    - *Firmware hãng (OhStem):* Ưu tiên cho dạy học STEM, giao diện kéo thả, ổn định, tương thích App. Không tương thích tốt với AI Agent IDE.
    - *Firmware chuẩn (Arduino/ESP-IDF/MicroPython):* Bắt buộc để dùng Google Antigravity, khai thác tối đa 16MB Flash/8MB PSRAM, chạy TensorFlow Lite, hiệu năng biên dịch native.
  - **Tích hợp Google Antigravity (AI Agent IDE):**
    - Cung cấp ngữ cảnh chính xác (`CONTEXT.md`): Chip ESP32-S3 N16R8, bản đồ chân Grove, tính năng onboard.
    - AI tự sinh code C++/Python, quản lý `platformio.ini`, tự đọc log lỗi terminal để sửa code (Self-Healing).
    - Lệnh nạp: `pio run --target upload`. Cần thao tác vật lý (giữ BOOT → nhấn RESET → thả RESET → thả BOOT) nếu auto-flash thất bại.
  - **DIY Form Arduino:** Dùng Perfboard + Female Header 44 chân, map GPIO linh hoạt qua code (`#define LED_BUILTIN 2`). Ưu tiên giữ nguyên logic 3.3V hoặc dùng IC chuyển mức nếu bắt buộc tương thích shield 5V.
- **Ghi chú quan trọng:** 
  - Luôn sao lưu firmware gốc trước khi flash: `esptool.py --chip esp32s3 --port <COM> read_flash 0x0 0x1000000 backup.bin`
  - Yolo UNO = Giải pháp giáo dục "cắm là chạy", tối ưu thời lượng tiết học. ESP32-S3 DevKit = Phần cứng thô giá rẻ, tối ưu cho dự án AI/IoT chuyên sâu & nghiên cứu.
  - Antigravity hoạt động hiệu quả nhất với môi trường mở (PlatformIO/ESP-IDF). AI không thể tự nhấn nút BOOT/RESET, vòng lặp phần cứng vẫn cần can thiệp vật lý.
