### Xử lý `FileNotFoundError` khi dùng `os.rename()` trong Python

Lỗi này thường xuất hiện khi đường dẫn file nguồn không tồn tại hoặc thư mục đích chưa được tạo. Lưu ý quan trọng: `os.rename()` **không tự động tạo thư mục đích**.

**Quy trình xử lý an toàn:**
1. **Chuẩn hóa đường dẫn:** Luôn dùng `os.path.join()` để ghép path, đảm bảo tương thích đa nền tảng.
2. **Tạo thư mục đích trước:** Dùng `os.makedirs(dest_dir, exist_ok=True)` để tự động tạo thư mục nếu chưa có, tránh lỗi khi di chuyển.
3. **Xác minh file nguồn:** Kiểm tra bằng `os.path.exists(src_file)` trước khi thực thi lệnh rename.

**Mẫu triển khai:**
```python
import os

src_file = os.path.join(os.getcwd(), "sample_data", "README.md")
dest_dir = os.path.join(os.getcwd(), "test1")
dest_file = os.path.join(dest_dir, "README.md")

# 1. Đảm bảo thư mục đích tồn tại
os.makedirs(dest_dir, exist_ok=True)

# 2. Kiểm tra & di chuyển file
if os.path.exists(src_file):
    os.rename(src_file, dest_file)
    print("Di chuyển thành công.")
else:
    print("Lỗi: File nguồn không tồn tại.")
```

💡 **Nguyên tắc:** Tách biệt rõ ràng bước chuẩn bị môi trường (tạo thư mục) và bước thao tác file (rename/move). Luôn kiểm tra sự tồn tại của đường dẫn trước khi gọi các hàm I/O để tránh exception không mong muốn.
