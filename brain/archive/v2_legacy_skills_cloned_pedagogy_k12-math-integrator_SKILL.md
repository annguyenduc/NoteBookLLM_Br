# k12-math-integrator — Math-STEAM Real-World Integrator (PRO)

> **Goal:** Transform Math from "solving textbook exercises" into "a tool to overcome engineering constraints." Use mathematical modeling to help students understand the real-world application of formulas within project contexts.

## When to Activate

- Designing a STEAM lesson that requires integrating mathematics as a core engineering tool
- Helping students apply mathematical modeling to a design challenge
- Aligning a math unit to NCTM standards within a project-based learning context
- Designing open-ended math problems that connect to STEAM project constraints

## Instructions

### Mathematical Modeling Framework (6 Steps)

Use this process to guide students through "open" math problems in projects:

1. **🚩 Identify Problem:** Translate a "real-world problem" into a "mathematical question."
2. **📉 Make Assumptions:** Simplify the problem with reasonable assumptions (e.g., ignore air resistance).
3. **✍️ Build Model:** Set up equations, functions, or diagrams to represent the variables.
4. **⚙️ Compute:** Solve mathematically by hand or using technology (Thunkable, Python, Excel).
5. **🧪 Evaluate:** Check — does the mathematical result match reality?
6. **📝 Report:** Explain the results in "non-mathematical" language for all stakeholders.

### Integration Strategies

| Strategy | Application |
|---------|-------------|
| **Geometry in Design** | Use geometry to optimize space and materials in EDP (Engineering Design Process). |
| **Data & Statistics** | Analyze data from test iterations to find patterns and optimize design. |
| **Functions & Algebra** | Build control logic for Robots or Chatbots based on mathematical variables. |

### NCTM Standard Alignment

- **Problem Solving:** Use mathematics to confront genuinely unknown situations.
- **Reasoning & Proof:** Explain "Why is this formula correct in this context?"
- **Communication:** Present mathematical reasoning clearly through project reports.

## Quality Gate (Red Flags)

- ❌ **Plug-and-Chug:** Students only execute calculations without understanding what the result means for the project.
- ❌ **Isolated Math:** Math problems have no direct connection to an engineering constraint in the project.
- ❌ **Over-simplicity:** Problem is too simple and doesn't reflect the complexity of the real world.
- ⚠️ Always cross-reference with `k12-shared-references` for correct GDPT 2018 math terminology.
