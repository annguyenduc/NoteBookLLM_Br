# 📚 Ứng dụng Nộp Bài Trực tuyến

> [!summary] Tổng quan
> Dự án xây dựng ứng dụng web đơn giản giúp học sinh nộp bài tập khi có kết nối internet. Giáo viên có thể xem, lọc và tải bài về. Ưu tiên: dễ dùng, triển khai nhanh, mở rộng được sau này.

## 👥 Đối tượng & Phân quyền
| Vai trò | Chức năng chính |
|---|---|
| **Học sinh** | Đăng nhập/đăng ký, chọn file, điền thông tin môn học, nộp bài |
| **Giáo viên** | Đăng nhập, xem danh sách bài nộp, lọc theo môn/ngày, tải file về |

## 🛠️ Công nghệ đề xuất
| Thành phần | Công nghệ | Ghi chú |
|---|---|---|
| **Backend** | Python + `Flask` | Ưu tiên cho người mới, nhẹ, dễ debug |
| **Frontend** | HTML, CSS, JS + `Bootstrap` | Giao diện nhanh, responsive, không cần build tool |
| **Database** | `SQLite` (ban đầu) → PostgreSQL/MySQL (sau) | Không cần cài server DB, file `.db` duy nhất |
| **Lưu trữ file** | Thư mục `uploads/` trên server → AWS S3 / Firebase Storage | Chuyển sang cloud khi cần scale |

## 📋 Lộ trình phát triển
### 1. Backend & API
- [ ] API xác thực: đăng ký, đăng nhập, phân quyền (HS/GV)
- [ ] API upload file: nhận file, kiểm tra định dạng/dung lượng, lưu kèm metadata (tên HS, môn, ngày nộp)
- [ ] API quản lý bài: lấy danh sách, lọc theo môn/ngày, endpoint tải file

### 2. Frontend
- [ ] Trang đăng nhập/đăng ký (form cơ bản + validation)
- [ ] Form nộp bài (chọn file, nhập thông tin, nút gửi)
- [ ] Bảng quản lý bài nộp (dành cho GV, có bộ lọc & nút tải)

### 3. Tích hợp & Kiểm thử
- [ ] Kết nối Frontend ↔ Backend qua `fetch()` hoặc `axios`
- [ ] Test luồng: đăng nhập → nộp bài → giáo viên xem/tải
- [ ] Xử lý lỗi cơ bản: file sai định dạng, mất kết nối, quyền truy cập

## 🚀 Bước khởi đầu (Ưu tiên)
> [!tip] Gợi ý triển khai
> 1. **Bắt đầu từ Backend**: Tạo project Flask → cài `Flask`, `Flask-SQLAlchemy`, `Flask-Login` → viết 2 endpoint cơ bản: `/register`, `/login`, `/upload`.
> 2. **Frontend sau**: Dùng HTML + Bootstrap tạo form → gọi API bằng JavaScript thuần.
> 3. **Lưu file tạm**: Dùng `werkzeug.utils.secure_filename` + thư mục `uploads/` để tránh lỗi bảo mật.

## 💡 Ghi chú kỹ thuật & Mở rộng
- **Bảo mật file**: Giới hạn extension (`.pdf`, `.docx`, `.zip`), kiểm tra dung lượng (`MAX_CONTENT_LENGTH`), dùng `secure_filename`.
- **Database**: Bảng `users` (id, username, password_hash, role), bảng `submissions` (id, user_id, subject, file_path, submitted_at).
- **Triển khai sau này**: 
  - Thêm thông báo email khi nộp bài thành công
  - Hỗ trợ nộp nhiều file / nén zip
  - Dashboard thống kê tỷ lệ nộp bài
  - Chuyển storage sang AWS S3 hoặc Firebase
