## 📝 Phong cách Code (Coding Style)
- **Comment có chọn lọc:** Chỉ giải thích thuật toán phức tạp, quyết định thiết kế hoặc logic không hiển nhiên. Tránh comment mô tả lại cú pháp code.
- **Self-documenting code:** Ưu tiên đặt tên biến/hàm rõ nghĩa, cấu trúc mạch lạc để code tự giải thích, giảm phụ thuộc vào comment.
- **Tái cấu trúc code trùng lặp (DRY):** Gom các đoạn code lặp thành hàm/module tái sử dụng. Giảm chi phí bảo trì, hạn chế lỗi không nhất quán.
- **Quy tắc đặt tên biến:** Tránh dùng 1 ký tự (trừ biến đếm vòng lặp chuẩn `i`, `j`, `k`). Tên biến phải phản ánh đúng kiểu dữ liệu hoặc mục đích sử dụng.

## 🔧 Tham số Hàm (Parameters)
- Giá trị được truyền vào hàm làm đầu vào tại thời điểm gọi hàm được gọi là **Parameters** (Tham số).

## 🔀 Cấu trúc Điều kiện `if-elif-else`
- Dùng để kiểm tra tuần tự nhiều điều kiện, thay thế hiệu quả cho `if` lồng nhau.
- Trình thông dịch dừng ngay khi gặp điều kiện `True` đầu tiên và thực thi khối lệnh tương ứng.
- Khối `else` (tùy chọn) chỉ chạy khi tất cả điều kiện `if`/`elif` đều `False`.
```python
if điều_kiện_1:
    # Xử lý 1
elif điều_kiện_2:
    # Xử lý 2
else:
    # Xử lý mặc định
```

## ⚙️ Toán tử & Thứ tự Ưu tiên
- `+` với `str`: Thực hiện nối chuỗi (String concatenation).
- `+`, `*` với `int`/`float`: Tuân thủ độ ưu tiên toán học chuẩn (nhân/chia trước, cộng/trừ sau).
- Ví dụ: `9999 + 8888 + 100 * 100` → `100 * 100` tính trước (`10000`), sau đó cộng dồn → kết quả `28887`.
