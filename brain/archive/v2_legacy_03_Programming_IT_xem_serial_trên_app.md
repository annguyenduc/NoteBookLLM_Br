# Hướng dẫn Giám sát Dữ liệu Serial trên Yolo UNO qua app.ohstem.vn

## 1. Nguyên lý Hoạt động & Yêu cầu Hệ thống
- **Giao thức truyền thông:** UART/Serial (thường được ánh xạ qua USB-CDC hoặc Bluetooth tùy cấu hình phần cứng).
- **Cơ chế Web:** `app.ohstem.vn` sử dụng **Web Serial API** để trình duyệt giao tiếp trực tiếp với cổng COM của thiết bị mà không cần cài driver trung gian.
- **Yêu cầu trình duyệt:** Bắt buộc sử dụng Chromium-based (Google Chrome, Microsoft Edge, Opera). Firefox/Safari chưa hỗ trợ đầy đủ Web Serial API.
- **Tốc độ Baudrate:** Mặc định khuyến nghị `9600` hoặc `115200`. Giá trị này phải khớp tuyệt đối giữa code firmware và cấu hình trên giao diện monitor.

## 2. Cấu hình Mã nguồn (Firmware & Block)
Thiết bị phải được lập trình để chủ động đẩy dữ liệu ra cổng Serial trước khi bật chế độ giám sát.

### 🖥️ Lập trình Arduino C++
```cpp
void setup() {
  Serial.begin(9600); // Khởi tạo cổng Serial với tốc độ 9600 bps
}
void loop() {
  Serial.println("Yolo UNO: Sensor Ready"); // Gửi dữ liệu + ký tự xuống dòng
  delay(1000);
}
```

### 🧩 Lập trình Khối (Block-based)
Trong thư viện OhStem, sử dụng các khối thuộc nhóm **Serial/Console**:
- `In chuỗi` / `In số`: Gửi dữ liệu thô, không tự động xuống dòng.
- `In chuỗi (dòng mới)` / `In số (dòng mới)`: Tự động thêm `\r\n`, phù hợp để phân tách log.
- `In cặp giá trị`: Tối ưu cho việc truyền đồng thời 2 tham số (ví dụ: nhiệt độ, độ ẩm) để xuất CSV hoặc vẽ đồ thị.

## 3. Quy trình Bật Serial Monitor trên app.ohstem.vn
Do giao diện web có thể cập nhật theo phiên bản, thực hiện theo luồng kỹ thuật chuẩn:

1. **Truy cập & Mở dự án:** Đăng nhập `https://app.ohstem.vn`, chọn workspace chứa mã nguồn Yolo UNO.
2. **Kết nối Thiết bị:** Nhấn nút **Connect** / **Kết nối** (biểu tượng 🔌 hoặc 📡). Trình duyệt sẽ hiện hộp thoại yêu cầu chọn cổng COM → Chọn cổng tương ứng với Yolo UNO và nhấn **Allow**.
3. **Mở Bảng Giám sát:** Tìm tab/menu có nhãn `Serial Monitor`, `Console`, `Terminal` hoặc `Output`. Thường nằm ở panel dưới cùng hoặc cạnh cửa sổ code.
4. **Đồng bộ Baudrate:** Chọn tốc độ truyền khớp với `Serial.begin()` trong code (ví dụ: `9600`).
5. **Chạy & Quan sát:** Nhấn **Run** hoặc **Upload**. Dữ liệu từ `Serial.print()` sẽ được stream realtime lên bảng console.

## 4. Bảng Tham chiếu Lệnh Serial & Khối lệnh
| Chức năng | Arduino C++ | Khối lệnh OhStem | Ứng dụng thực tế |
|:---|:---|:---|:---|
| Gửi chuỗi (không xuống dòng) | `Serial.print("text")` | `In chuỗi` | Ghép tiêu đề + giá trị cảm biến |
| Gửi chuỗi (có xuống dòng) | `Serial.println("text")` | `In chuỗi (dòng mới)` | Phân tách log, dễ đọc |
| Gửi số nguyên/thực | `Serial.print(123.45)` | `In số` | Truyền giá trị đo lường |
| Gửi cặp giá trị | `Serial.print(x); Serial.print(","); Serial.println(y)` | `In cặp giá trị` | Chuẩn bị dữ liệu cho CSV/Chart |

## 5. Xử lý Sự cố & Best Practice (IoT/STEAM)
- **⚠️ Không hiển thị dữ liệu:** 
  - Kiểm tra lại Baudrate trên app và trong code.
  - Đảm bảo không có ứng dụng khác (Arduino IDE, PuTTY, Serial Plotter) đang chiếm dụng cổng COM.
  - Thử rút/cắm lại cáp USB và cấp lại quyền Web Serial API.
- **⚠️ Ký tự rác (Garbage characters):** Sai Baudrate hoặc nhiễu đường truyền. Ưu tiên chuyển sang `115200` và dùng cáp USB chất lượng cao.
- **🔧 Best Practice cho STEAM/IoT:**
  - Luôn dùng `println` hoặc khối `dòng mới` để tránh dữ liệu bị dính chuỗi.
  - Khi truyền nhiều cảm biến, định dạng dữ liệu theo chuẩn `key:value` hoặc `CSV` ngay từ khối lệnh để dễ dàng tích hợp với các công cụ phân tích dữ liệu (Excel, Grafana, hoặc dashboard OhStem).
  - Tắt hoặc comment các lệnh `Serial.print()` không cần thiết trước khi triển khai thực tế để giảm tải CPU và tiết kiệm năng lượng cho thiết bị IoT.
