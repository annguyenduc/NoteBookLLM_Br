# Kỹ thuật Prompt & Metadata AI cho Thẻ Định Danh Sinh Viên

**Nguyên lý cấu trúc Prompt**
- Phân lớp thông tin rõ ràng: (1) Chất liệu & hình thái (PVC dẻo, bền), (2) Dữ liệu định danh (tên trường, mã SFID, ngày sinh/cấp/hết hạn, trạng thái, chữ ký), (3) Thành phần hình ảnh (chân dung đối tượng), (4) Định hướng phong cách (thiết kế học thuật, chính thức, thực tế).
- AI sinh ảnh (DALL-E) có độ chính xác văn bản thấp. Prompt cần nhấn mạnh "official/realistic" và chỉ định rõ vị trí hiển thị text để giảm hiện tượng ký tự sai hoặc méo.

**Phân tích Metadata Đầu Ra**
- Tool trả về JSON chứa `asset_pointer` (định vị file nội bộ), kích thước pixel (`width`/`height`), và khối `metadata.dalle`/`metadata.generation` (lưu `gen_id`, `gen_size`, `seed`).
- Trường `sanitized` xác nhận ảnh đã qua bộ lọc nội dung. `watermarked_asset_pointer` dùng để truy xuất phiên bản có watermark bản quyền nếu cần.

**Lưu ý Kỹ thuật & Tối ưu**
- Tỷ lệ mặc định (1024x1024 hoặc 1536x1024) lệch chuẩn thẻ vật lý CR80 (85.6×53.98mm). Cần crop hoặc điều chỉnh tham số `size` trong prompt.
- Việc lặp prompt trong hội thoại phản ánh giới hạn của AI trong việc render text chính xác. Giải pháp bền vững: dùng AI sinh nền/layout, sau đó chèn text định danh bằng script (Python/PIL) hoặc template vector để đảm bảo tính pháp lý và độ chính xác dữ liệu.
