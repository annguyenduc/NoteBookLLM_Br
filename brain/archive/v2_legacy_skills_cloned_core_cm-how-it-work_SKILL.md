# cm-how-it-work — The Complete Vibe Coding Guide (LITE)

> **Goal:** Transform ideas into real products through the Antigravity/CodyMaster skill ecosystem. This guide ensures maximum quality, absolute security, and fastest execution speed.

## When to Activate

- New user asking how the skill system works
- Team member needs an orientation to the full workflow
- Debugging a workflow that skipped critical phases
- Understanding how skills integrate with each other

## Instructions

### The 5-Phase Master Workflow

1. **Phase 0: Safety & Analysis** — Authenticate identity (`cm-identity-guard`) + Strategic analysis (`cm-brainstorm-idea`).
2. **Phase 1: Planning** — Understand JTBD + Create detailed plan (`cm-planning`) → User approves.
3. **Phase 2: Implementation** — Execute TDD (Red → Green → Refactor) + Batch/parallel processing (`cm-execution`).
4. **Phase 3: Quality Control** — Gate quality via `cm-quality-gate` & systematic `cm-debugging`.
5. **Phase 4: Safe Deploy** — Deploy through 8 safety gates + multi-language sync (`cm-safe-i18n`).

### 9 Golden Rules

1. **Identity First:** Verify account before pushing or deploying.
2. **Design Before Code:** Have an approved plan before touching the keyboard.
3. **Test Before Code:** No failing test = no new code (`cm-tdd`).
4. **Evidence Over Claims:** Trust only terminal/test output, not AI promises.
5. **Deploy via Gates:** Pass 8 sequential gates. Fail one gate = STOP.
6. **Safe Secrets:** Never commit secrets (use secret-shield).
7. **Working Memory:** Read `CONTINUITY.md` at start, update at end.
8. **Grep Before Edit:** Always search before modifying any file.
9. **Atomic Commits:** One logical change per commit.

## Quality Gate (Red Flags)

- ❌ Jumping to Phase 2 (Code) without completing Phase 1 (Planning).
- ❌ Skipping unit tests because "this change is too small."
- ❌ Operating without working memory (skipping `cm-continuity`).
- ❌ Deploying to production without passing `test:gate`.
