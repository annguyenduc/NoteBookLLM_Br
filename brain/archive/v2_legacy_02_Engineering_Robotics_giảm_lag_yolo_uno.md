# Giảm độ trễ AI trên Yolo UNO (ESP32-S3)

**Vấn đề:** Chạy nhận diện qua server OhStem (Teachable Machine) gây trễ do phụ thuộc đường truyền mạng và tải server chia sẻ. Độ trễ có thể loại bỏ bằng cách dịch chuyển vị trí suy luận (inference) hoặc tối ưu hóa luồng dữ liệu.

## 1. Chạy AI trực tiếp trên thiết bị (On-Device)
- **Công cụ:** Edge Impulse, TensorFlow Lite Micro, hoặc thuật toán ML nhẹ (K-NN, Decision Tree, ngưỡng động/EMA).
- **Tối ưu tài nguyên:** 
  - Dùng model `int8 quantization`, input 1D (cảm biến/audio) hoặc ảnh `96×96 grayscale`.
  - Giảm tần suất lấy mẫu: `20–62.5Hz` (cảm biến), `16kHz` (audio).
  - Tắt Bluetooth, giảm log serial, tận dụng PSRAM nếu có.
- **Luồng dữ liệu:** Suy luận cục bộ → chỉ publish lệnh điều khiển qua MQTT. Băng thông cực thấp, loại bỏ độ trễ mạng.

## 2. Chạy AI cục bộ trên PC/Điện thoại
- **Cách triển khai:** Export Teachable Machine sang TensorFlow.js → chạy offline trên trình duyệt. Suy luận bằng webcam/mic tại máy cục bộ → gửi lệnh qua MQTT/WebSocket (Mosquitto/EMQX trong LAN) đến Yolo UNO.
- **Ưu điểm:** Giữ nguyên workflow TM, inference gần như tức thì. UNO chỉ nhận event và điều khiển phần cứng. Phù hợp bài toán thị giác máy tính đơn giản.

## 3. Tối ưu kết nối Server OhStem (bắt buộc dùng)
- **Giảm tần suất publish:** Chỉ gửi khi trạng thái thay đổi (edge-trigger + hysteresis/EMA), chu kỳ `200–500ms`.
- **Tối ưu MQTT:** `QoS 0`, không `retain`, payload JSON gọn/gộp nhiều trường, `keepalive=30–60s`.
- **Ổn định mạng:** Tắt Wi-Fi power-save, dùng static IP, cơ chế reconnect exponential backoff.
- **Kiến trúc:** Triển khai broker nội bộ (Mosquitto) để giảm tải, app.ohstem chỉ đóng vai trò dashboard phụ.

## Ma trận lựa chọn giải pháp
| Loại dữ liệu / Yêu cầu | Hướng tối ưu |
|---|---|
| Âm thanh, cảm biến rời rạc, cử chỉ IMU | On-Device (Edge Impulse / TFLM) |
| Nhận diện hình ảnh, ưu tiên đơn giản | AI cục bộ PC/ĐT + MQTT LAN |
| Bắt buộc đồng bộ dashboard OhStem | Tối ưu server + giảm tần suất sự kiện |

## Nguyên tắc kỹ thuật cốt lõi
- Tránh truyền raw data/ảnh qua mạng. Chỉ truyền `event/command`.
- Cân bằng độ chính xác model với giới hạn RAM/Flash của ESP32-S3.
- Kiến trúc LAN MQTT giúp cách ly độ trễ internet khỏi vòng điều khiển thời gian thực.
