# Khắc phục widget hiển thị chuỗi bị ghi log liên tục (MQTT)

## Vấn đề & Nguyên nhân
- **Hiện tượng:** Widget dạng ô cam trên dashboard nhận dữ liệu liên tục và tự động xuống dòng, tạo thành nhiều hàng log thay vì cập nhật tại chỗ.
- **Nguyên nhân:** Code đang gọi `publish(topic, msg)` định kỳ (ví dụ mỗi 300ms qua Timer) mà không kiểm tra trạng thái thay đổi của dữ liệu, khiến dashboard nhận trùng lặp liên tục.

## Giải pháp khắc phục
1. **Điều chỉnh UI (Nhanh):** Chuyển widget từ chế độ `Text Log/Console` sang `Label/Value Box` hoặc tắt tùy chọn `Keep history/Append` để kích hoạt chế độ ghi đè.
2. **Tối ưu Logic (Khuyến nghị):** Chỉ `publish` khi giá trị thực sự thay đổi.
   - Khai báo biến lưu trạng thái cũ: `last_msg = ""`
   - So sánh trước khi gửi: `if msg != last_msg:` → thực hiện `publish` → cập nhật `last_msg = msg`
   - **Xóa** khối `publish` khỏi vòng lặp Timer định kỳ.
3. **Giới hạn tần suất (Throttle):** Tăng chu kỳ Timer lên 2–5s hoặc dùng `ticks_ms()` để debounce, tránh spam message.

## Triển khai thực tế
- Di chuyển lệnh `publish` ra khỏi Timer 0.3s.
- Chỉ kích hoạt `publish` tại các điểm thay đổi trạng thái thực tế: khi nhận lệnh điều khiển (V1/V2) hoặc khi hệ thống Auto vượt ngưỡng bật/tắt thiết bị.
- Luôn kết hợp kiểm tra `last_msg` để đảm bảo tính ổn định và tiết kiệm băng thông MQTT.
