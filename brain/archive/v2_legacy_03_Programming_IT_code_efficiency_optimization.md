# 🚀 Tối Ưu Hiệu Suất Code (Code Efficiency Optimization)

> [!summary] Tổng quan
> Cải thiện hiệu suất code là quá trình cân bằng giữa **thời gian thực thi** (Time Complexity) và **bộ nhớ sử dụng** (Space Complexity). Ghi chú này tổng hợp quy trình 6 bước giúp bạn phân tích, tối ưu và kiểm chứng code một cách hệ thống.

## 🔍 6 Bước Tối Ưu Hệ Thống

### 1️⃣ Hiểu rõ code hiện tại
- Xác định rõ chức năng & luồng hoạt động của code.
- Đo lường hiệu suất thực tế (thời gian chạy, bộ nhớ) với các kích thước input khác nhau.
> [!question] Câu hỏi phản tư
> - Phần nào của code đang tốn nhiều thời gian/bộ nhớ nhất?
> - Bạn đã thử chạy với input lớn để kiểm tra giới hạn chưa?

### 2️⃣ Phân tích độ phức tạp (Complexity Analysis)
- **Time Complexity**: Tốc độ tăng thời gian chạy khi input tăng (`O(1)`, `O(log n)`, `O(n)`, `O(n²)`,...).
- **Space Complexity**: Lượng bộ nhớ phụ thuộc vào kích thước input.
> [!tip] Gợi ý
> Đánh giá độ phức tạp hiện tại. Liệu có thuật toán nào cùng giải quyết bài toán nhưng có độ phức tạp thấp hơn không?

### 3️⃣ Tối ưu thuật toán & cấu trúc dữ liệu
- **Chọn cấu trúc dữ liệu phù hợp**: Hash table (tra cứu nhanh), Heap (lấy min/max nhanh), v.v.
- **Tránh tính toán thừa**: Dùng memoization/cache để lưu kết quả trung gian.
- **Chia để trị (Divide & Conquer)**: Tách bài toán lớn thành các bài toán con (VD: QuickSort, MergeSort).
> [!challenge] Thử thách
> Code của bạn có đang lặp lại tính toán hoặc dùng cấu trúc dữ liệu chưa tối ưu? Làm thế nào để đơn giản hóa luồng xử lý?

### 4️⃣ Tận dụng hàm & thư viện có sẵn
- Các ngôn ngữ đều có thư viện chuẩn được tối ưu sẵn cho sorting, searching, xử lý dữ liệu.
- Code thủ công thường chậm hơn và dễ lỗi hơn so với built-in functions.
> [!question] Kiểm tra
> Bạn đã dùng đủ các hàm built-in chưa? Có đoạn code nào có thể thay thế bằng 1 dòng gọi thư viện không?

### 5️⃣ Tối ưu vòng lặp & điều kiện
- Giảm thiểu vòng lặp lồng nhau (`nested loops`), ưu tiên thuật toán một lượt (`single-pass`).
- Tận dụng **short-circuit evaluation** trong điều kiện để tránh tính toán không cần thiết.
> [!experiment] Thực hành
> Tìm một vòng lặp lồng hoặc điều kiện phức tạp trong code. Thử "làm phẳng" (flatten) hoặc tối ưu logic để giảm số lần duyệt.

### 6️⃣ Đo lường & Kiểm thử (Profiling & Testing)
- Dùng công cụ profiling để xác định chính xác **bottleneck** (nút thắt cổ chai).
- Test kỹ với edge cases và dataset lớn.
> [!suggestion] Công cụ gợi ý
> Python: `timeit`, `cProfile` | JS: `console.time()`, Chrome DevTools | C++: `gprof`, `Valgrind`

## 💡 Lưu ý quan trọng
> [!warning] Đừng tối ưu sớm (Premature Optimization)
> *"Premature optimization is the root of all evil."* – Donald Knuth. Hãy đảm bảo code **đúng** và **dễ đọc/bảo trì** trước, chỉ tối ưu khi profiling chỉ ra vấn đề thực sự.
