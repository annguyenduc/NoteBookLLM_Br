# 🔨 Skill Forge (LITE)

> **Goal:** Duy trì sự tiến hóa của hệ thống thông qua việc phát hiện khoảng trống kiến thức (Skill gaps) và tự động hóa quy trình tạo mới/nâng cấp kỹ năng (Skill-Driven Development - SDD).

## 🚀 3 Operation Modes
| Mode | Trigger | Action |
|---|---|---|
| **CREATE** | `skill-gaps.md` ≥ 2 entries. | Research → Draft SKILL.md (Lite) → Register Index. |
| **UPGRADE**| User feedback / Corrections. | Đọc `learnings.md` → Refactor template → Bump version. |
| **AUDIT** | Mỗi 20 phiên hoặc thủ công. | Review toàn bộ 51 skills → Check overlap & hỏng link. |

## 🏗️ New Skill Structure (Lite Standard)
1. **YAML Header:** metadata (name, description, version).
2. **Goal Section:** Mục đích cốt lõi (1-2 câu).
3. **Instruction Table:** Quy trình hoặc quy tắc dạng bảng.
4. **Quality Gate:** Red Flags & Anti-patterns.
5. **Triggers:** Tiếng Việt & Tiếng Anh.

## 📐 System Registration (Final Step)
- Cập nhật `SKILLS_INDEX.md`.
- Đăng ký vào `gemini-extension.json`.
- Ghi nhận thay đổi vào `CONTINUITY.md`.

## 🚨 Quality Gate (Red Flags)
- ❌ Tạo skill mới bị trùng lặp >30% với skill hiện có.
- ❌ Thiếu framework sư phạm nền tảng (5E, PBL, Bloom).
- ❌ Output template quá mơ hồ, không tạo được kết quả nhất quán.
- ❌ Quên cập nhật Index sau khi tạo/nâng cấp.

- "Nâng cấp skill module-architect thêm phần phân hóa HS khuyết tật."
