## 📝 Content
# Tích hợp Teachable Machine với Yolo UNO qua Web Serial và IoT

## Nguyên lý kiến trúc
Mô hình phân tách tác vụ: Trình duyệt đảm nhận suy luận AI (Teachable Machine + WebCam), Yolo UNO đóng vai trò gateway nhận dữ liệu qua UART và đẩy lên nền tảng IoT. Giải pháp này tối ưu hiệu năng do vi điều khiển không phải gánh tải tính toán ML.

## Luồng dữ liệu
`WebCam → JS (TM Model) → Web Serial API → Yolo UNO (UART 115200) → IoT Platform (app.ohstem.vn)`

## Cấu hình IoT (app.ohstem.vn)
- Tạo thiết bị (Device) cho Yolo UNO.
- Định nghĩa 2 khóa dữ liệu:
  - `tm_label` (kiểu Text)
  - `tm_score` (kiểu Number)
- Gắn widget tương ứng lên dashboard để giám sát thời gian thực.

## Xử lý phía Trình duyệt (JavaScript)
- **Thư viện:** `@teachablemachine/image` + Web Serial API.
- **Cơ chế gửi:** Chạy vòng lặp dự đoán (~6–7 Hz). Chỉ truyền dữ liệu khi nhãn thay đổi hoặc độ tin cậy (`score`) chênh lệch > `0.05` so với lần gửi trước, tránh spam cổng Serial.
- **Định dạng payload:** `TM:<label>:<score>\n`
- **Yêu cầu môi trường:** Chạy trên HTTPS hoặc `file://` (cần bật flag Web Serial trong trình duyệt). Người dùng phải chọn cổng COM thủ công khi kích hoạt kết nối.

## Xử lý phía Yolo UNO
- Lắng nghe cổng Serial (Baudrate: 115200).
- Parse chuỗi nhận được theo định dạng `TM:...:...`, tách riêng `label` và `score`.
- Sử dụng block IoT của OhStem để push giá trị đã tách lên các khóa `tm_label` và `tm_score` đã cấu hình.

## Lưu ý kỹ thuật
- Không chạy TensorFlow.js trực tiếp trên UNO do giới hạn RAM/CPU.
- Cần xử lý lỗi ngắt kết nối Serial và cơ chế reconnect nếu triển khai thực tế.
- Định dạng chuỗi truyền cần đồng bộ chính xác giữa JS và code UNO để tránh lỗi parse.
