# So sánh biểu thức trong điều kiện `if` Python: `msg == '1'` vs `(msg) == '1'`

## 💡 Khái niệm cốt lõi
Trong Python, `msg == '1'` và `(msg) == '1'` **hoàn toàn tương đương**. Dấu ngoặc tròn chỉ đóng vai trò nhóm cú pháp (grouping) để tăng tính rõ ràng, không làm thay đổi kết quả đánh giá hay luồng điều khiển khi đặt trong câu lệnh `if`.

## 📖 Phân tích kỹ thuật
- `msg == '1'` là **biểu thức so sánh**, trả về giá trị Boolean (`True`/`False`).
- `if msg == '1':` là **câu lệnh điều kiện**, sử dụng kết quả biểu thức trên để quyết định thực thi khối lệnh.
- Python bỏ qua ngoặc đơn bao quanh biến đơn lẻ vì toán tử `==` đã có độ ưu tiên xác định. Trình thông dịch đánh giá cả hai cách viết theo cùng một cây cú pháp (AST).

## 💻 Ví dụ minh họa
```python
msg = '1'

# Hai cách viết dưới đây cho kết quả y hệt nhau
if msg == '1':
    print("Cách 1")
    
if (msg) == '1':
    print("Cách 2")
```

## ⚠️ Lưu ý quan trọng
- **Kiểu dữ liệu:** `'1'` (chuỗi) khác `1` (số nguyên). `'1' == 1` luôn trả về `False`.
- **Toán tử so sánh:** Luôn dùng `==` để so sánh giá trị. Tránh dùng `is` (dùng để kiểm tra danh tính đối tượng trong bộ nhớ, dễ gây lỗi logic với kiểu dữ liệu cơ bản).
- **Ngữ cảnh kéo thả (Blockly/Visual):** Khối so sánh trả về Boolean và được cắm vào khe điều kiện của khối `if`. Khái niệm ngoặc tròn không tồn tại trong môi trường này.

## 🔍 Khi nào ngoặc tròn thực sự cần thiết?
1. **Ưu tiên toán tử phức tạp:** `(a == b) and (c > d)` giúp tránh nhầm lẫn khi kết hợp nhiều điều kiện.
2. **Toán tử gán biểu thức (Walrus `:=`):** Bắt buộc dùng ngoặc: `if (n := len(s)) > 0:`.
3. **Định dạng nhiều dòng:** Giúp code dễ đọc và bảo trì khi biểu thức điều kiện dài.
4. **Tạo Tuple:** `(msg,)` là tuple chứa 1 phần tử, khác hoàn toàn với biến `msg` thông thường.
