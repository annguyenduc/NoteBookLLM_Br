# Khắc phục cảnh báo "Enumeration of workspace source files is taking a long time" trong VS Code

**Vấn đề:** VS Code hiển thị cảnh báo khi mở thư mục chứa quá nhiều tệp, gây chậm hoặc treo giao diện do tiến trình lập chỉ mục (indexing) và theo dõi thay đổi tệp (file watching) cho các tính năng tìm kiếm, IntelliSense và Git.

**Nguyên nhân:**
- Mở thư mục gốc quá lớn (ví dụ: toàn bộ ổ đĩa, thư mục `Users/`, hoặc root của monorepo).
- Thư mục chứa nhiều tệp tự sinh, thư viện hoặc bản build (`node_modules`, `venv`, `dist`, `build`, `.git`, `__pycache__`).

**Giải pháp xử lý:**
1. **Mở đúng thư mục con:** Chỉ mở thư mục chứa mã nguồn đang phát triển thay vì thư mục cha.
2. **Loại trừ thư mục khỏi bộ theo dõi:** Cấu hình VS Code bỏ qua quét các thư mục nặng để giảm tải CPU/RAM.
3. **Tận dụng `.gitignore`:** VS Code mặc định đọc cấu hình này để loại trừ tệp khỏi tìm kiếm và theo dõi.
4. **Tái cấu trúc dự án:** Tách biệt mã nguồn (`src/`) với các tệp đầu ra/thư viện bên ngoài.

**Cấu hình nhanh (`.vscode/settings.json`):**
```json
{
  "files.watcherExclude": {
    "**/node_modules/**": true,
    "**/.git/**": true,
    "**/dist/**": true,
    "**/build/**": true,
    "**/venv/**": true,
    "**/__pycache__/**": true
  }
}
```

**Phân biệt các thuộc tính loại trừ:**
- `files.watcherExclude`: Tắt theo dõi thay đổi tệp (trực tiếp giải quyết cảnh báo này, giảm tải hệ thống).
- `files.exclude`: Ẩn tệp/thư mục khỏi Explorer (giao diện).
- `search.exclude`: Ẩn tệp/thư mục khỏi kết quả tìm kiếm toàn cục (`Ctrl+Shift+F`).
