# unit-planner — CPD Learning Path Designer (PRO ToT — Integrated)

> **Goal:** Design long-term Continuous Professional Development (CPD) pathways for teachers. Development milestones are defined by **Performance Competencies** — not by abstract framework tiers.

## When to Activate

- Designing a multi-month teacher upskilling roadmap
- Defining career milestones for a STEAM educator community
- Planning a curriculum for growing a teacher from novice to innovator
- Creating a professional learning community framework for a school or center

## Instructions

### Dual-Mode Logic (Mandatory)

> [!IMPORTANT]
> **Check Intent:** If the target is **a K-12 student lesson sequence**, the agent must STOP and ask: "I detect you want to plan a student lesson series. Do you want UbD Student mode or Teacher CPD Pathway mode?"

### Integrated Planning Principles

- **Implicit TPACK:** Do NOT write "Improve TPACK." Write "Master integrating [Tool A] into teaching [Subject B] using [Pedagogy P]."
- **Implicit SAMR:** Final sessions must inherently reach **Modification** level by requiring a full lesson scenario redesign.
- **Continuous Scaffolding:** Each module must produce a real, tangible artifact for the teacher to take away and use immediately.

### CPD Learning Path (Performance Milestones)

| Stage | Milestone Description |
|-------|----------------------|
| **1. Awareness** | Understands the modern education context & current trends. |
| **2. Technological Setup** | Proficient with core technical tools of the domain. |
| **3. Pedagogical Redesign** | Can self-revise and optimize lesson plans using the new methodology. |
| **4. Professional Integration** | Successfully facilitates complex cross-curricular projects. |
| **5. Community Innovation** | Creates and shares new learning materials with the international community. |

### Global Alignment

- **Evaluations:** Track through the quality of the teacher's actual lesson plans (Evidence-based).
- **Standards:** Integrate ISTE standards implicitly within the exit criteria of each milestone.

## Quality Gate (Red Flags)

- ❌ Labeling "Level 1, 2, 3" by framework without describing what the teacher will actually **be able to do**.
- ❌ Missing "Coaching & Feedback" sessions between phases.
- ⚠️ Post-check: Always run `/output-contract` to audit for systemic coherence.

## Example Triggers

- "Design a 3-month roadmap: from traditional teacher to technology-integrated educator."
- "Department upskilling plan: project-based learning through design thinking."
- "Career path for STEM master trainers at a teaching center."
