# Chuyển đổi và giải nén

## 🤖 AI Summary
> 

## 📝 Content
**Nguyên lý cốt lõi**
File `.docx` thực chất là một kho lưu trữ ZIP nén theo chuẩn Office Open XML. Thư mục chứa phương tiện (ảnh, GIF) luôn nằm tại đường dẫn nội bộ `word/media/`. Do đó, không cần đổi tên phần mở rộng thủ công, Python có thể đọc trực tiếp file `.docx` như một file ZIP nhờ cơ chế nhận diện magic number.

**Giải pháp tự động hóa**
Sử dụng module `zipfile` kết hợp `pathlib` để duyệt thư mục nguồn, trích xuất chọn lọc `word/media/` và lưu vào thư mục đích chỉ định:
```python
import zipfile
from pathlib import Path

def extract_docx_media(source_dir: str, output_dir: str) -> None:
    Path(output_dir).mkdir(parents=True, exist_ok=True)
    for docx_file in Path(source_dir).glob("*.docx"):
        with zipfile.ZipFile(docx_file, "r") as zf:
            for member in zf.namelist():
                if member.startswith("word/media/"):
                    zf.extract(member, output_dir)

# Cách dùng: extract_docx_media("đường_dẫn_folder_chứa_docx", "đường_dẫn_folder_đích")
```

**Lưu ý kỹ thuật**
- `zipfile.ZipFile` hoạt động trực tiếp trên file `.docx`. Việc đổi tên thành `.zip` là không bắt buộc và có thể bỏ qua để tối ưu luồng xử lý.
- Mặc định, `zf.extract()` sẽ giữ nguyên cấu trúc thư mục nội bộ (tạo ra `output_dir/word/media/...`). Nếu cần lưu ảnh dạng phẳng (flat), hãy thay `zf.extract()` bằng `zf.read(member)` và ghi file thủ công với `Path(member).name`.
- Phương pháp này xử lý hàng loạt file cục bộ, loại bỏ hoàn toàn bước upload thủ công trên Colab, phù hợp cho automation pipeline hoặc xử lý batch trên máy local.
