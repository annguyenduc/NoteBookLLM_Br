# ESP8266 & Yolo UNO trong IoT: Đặc tính & Tối ưu ổn định đa người dùng

## 1. Đặc tính kỹ thuật ESP8266
- **Kiến trúc:** CPU 32-bit Tensilica (80–160 MHz), RAM ~50 KB, Flash 512 KB–4 MB.
- **Kết nối:** Wi-Fi 2.4 GHz 802.11 b/g/n tích hợp sẵn, ~10 GPIO lập trình được.
- **Hệ sinh thái:** Tương thích Arduino IDE, MicroPython, Blockly; hỗ trợ MQTT, Blynk, Firebase, OhStem.
- **Giới hạn:** RAM thấp, không xử lý đa luồng nặng hoặc camera. Phù hợp dự án cảm biến đơn giản, điều khiển cơ bản.

## 2. Nguyên tắc triển khai ổn định
- **Nguồn cấp:** Bắt buộc ổn định 3.3V. Tách nguồn riêng khi dùng relay/motor/cảm biến công suất.
- **Chân GPIO:** Tránh dùng D0, D3, D4 cho thiết bị ngoại vi do xung đột tiến trình boot.
- **Tần suất truyền:** Giới hạn publish MQTT ở 500–1000 ms/gói. Tránh spam broker.
- **Nâng cấp phần cứng:** Chuyển sang ESP32/Yolo UNO (ESP32-S3) khi cần xử lý đồng thời nhiều cảm biến, camera hoặc kết nối mạng phức tạp.

## 3. Xử lý nghẽn cổ chai khi đa người dùng truy cập Server
Khi nhiều client đồng thời kết nối, broker dễ quá tải do bão dữ liệu và reconnect liên tục. Giải pháp tối ưu:
- **Publish theo ngưỡng thay đổi:** Chỉ gửi khi `|giá_trị_mới - giá_trị_cũ| ≥ Δ`.
- **Lọc nhiễu:** Áp dụng Hysteresis hoặc EMA (Exponential Moving Average) để loại bỏ dao động nhỏ, giảm tần suất publish không cần thiết.
- **Quản lý kết nối:** Dùng cơ chế reconnect có độ trễ tăng dần (exponential backoff), duy trì session MQTT, tránh đóng/mở liên tục.
- **Tối ưu payload & QoS:** Gửi JSON gọn, ưu tiên QoS 0 cho dữ liệu telemetry, QoS 1 cho lệnh điều khiển.

## 4. Minh họa MicroPython: Publish có điều kiện
```python
last_val = None
THRESHOLD = 0.5
while True:
    current_val = read_sensor()
    if last_val is None or abs(current_val - last_val) >= THRESHOLD:
        client.publish(b"topic/data", str(current_val))
        last_val = current_val
    time.sleep(1)
```
