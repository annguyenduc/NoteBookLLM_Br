Dưới đây là các sự kiện (Facts) được trích xuất từ nguồn dữ liệu cung cấp, tập trung vào các khía cạnh kỹ thuật liên quan đến tự động hóa (AI/Robotics) và kết nối hệ thống (IoT):

- **Fact:** [CONV] API Key là một chuỗi ký tự duy nhất dùng để xác thực, kiểm soát quyền truy cập và theo dõi hoạt động của người dùng khi giao tiếp với các dịch vụ trực tuyến.
- **Source:** [Assistant response - Section: Mục đích của API Key]
- **Tag:** [vv15]

- **Fact:** [CONV] Việc tích hợp API cho phép các ứng dụng tự động hóa quy trình quản lý tài khoản và tài nguyên từ xa thông qua các yêu cầu HTTP.
- **Source:** [Assistant response - Section: Khi nào cần API Key]
- **Tag:** [vv15]

- **Fact:** [CONV] Thư viện `requests` trong Python là công cụ phổ biến để thực hiện các phương thức HTTP như GET (lấy dữ liệu) và POST (gửi dữ liệu) tới các endpoint của API.
- **Source:** [Assistant response - Section: 2. Hiểu cách API hoạt động & Section 3]
- **Tag:** [vv15]

- **Fact:** [CONV] Biểu thức chính quy (Regular Expression) trong Python, thông qua mô-đun `re`, được sử dụng để tìm kiếm, khớp mẫu và xử lý các chuỗi văn bản phức tạp.
- **Source:** [Assistant response - Section: 1. Nhập mô-đun re]
- **Tag:** [vv15]

- **Fact:** [CONV] Trong Regular Expression, ký hiệu `r''` (raw string) được sử dụng để báo hiệu cho Python không xử lý các ký tự đặc biệt như dấu gạch chéo ngược (\).
- **Source:** [Assistant response - Section: 4. Giải thích mẫu biểu thức chính quy]
- **Tag:** [vv15]

- **Fact:** [CONV] Mẫu biểu thức chính quy `.` (dấu chấm) có chức năng khớp với bất kỳ ký tự đơn nào ngoại trừ ký tự xuống dòng.
- **Source:** [Assistant response - Section: 4. Giải thích mẫu biểu thức chính quy]
- **Tag:** [vv15]

- **Fact:** [CONV] Framework `unittest` là một thư viện tích hợp trong Python dùng để viết và chạy các bài kiểm thử tự động (unit tests) nhằm đảm bảo tính chính xác của mã nguồn.
- **Source:** [Assistant response - Section: 1. Nhập mô-đun unittest]
- **Tag:** [vv15]

- **Fact:** [CONV] Phương thức `self.assertEqual()` trong `unittest` được sử dụng để so sánh giá trị trả về của một hàm với giá trị mong đợi; nếu không khớp, bài kiểm thử sẽ thất bại (AssertionError).
- **Source:** [Assistant response - Section: 3. Định nghĩa phương thức kiểm thử test_basic & User Traceback]
- **Tag:** [vv15]

- **Fact:** [CONV] "Edge cases" (trường hợp biên) là các tình huống đặc biệt như chuỗi rỗng hoặc dữ liệu không khớp mẫu, cần được kiểm thử để tránh các hành vi không mong đợi trong lập trình hệ thống.
- **Source:** [Assistant response - Section: EDGE CASES]
- **Tag:** [vv15]

- **Fact:** [CONV] Tham số `exit=False` trong `unittest.main()` ngăn chương trình dừng đột ngột sau khi chạy kiểm thử, phù hợp khi thực thi trong môi trường tương tác như Jupyter Notebook.
- **Source:** [Assistant response - Section: 3. Chạy tất cả các bài kiểm thử]
- **Tag:** [vv15]

- **Fact:** [CONV] Để bảo mật hệ thống, API Key nên được lưu trữ trong các biến môi trường (environment variables) thay vì viết trực tiếp vào mã nguồn.
- **Source:** [Assistant response - Section: Bảo mật API Key]
- **Tag:** [vv15]
