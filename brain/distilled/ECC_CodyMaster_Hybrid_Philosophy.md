# So sánh & Tích hợp: ECC vs CodyMaster

## ⚖️ Đối chiếu Triết lý

| Tiêu chí | Everything Claude Code (ECC) | CodyMaster |
| :--- | :--- | :--- |
| **Triết lý** | Bộ khung tối ưu hiệu suất (Performance Harness). | Hệ điều hành AI khép kín (AI OS). |
| **Tính tự chủ** | Human-in-the-loop (Planner/TDD/Reviewer). | Full Lifecycle Autonomy (Auto-routing). |
| **Bộ nhớ** | Continuous Learning (Học hỏi liên tục). | Shared Memory (Bộ nhớ chia sẻ thời gian thực). |
| **Thế mạnh** | Chất lượng kỹ thuật, STEM Curriculum, Mini-apps. | Tự động hóa Pipeline từ Ý tưởng -> Triển khai. |

## 🧩 Mô hình Hybrid Antigravity (LOM v3.0)

Hệ thống Antigravity v3.0 kế thừa tinh hoa của cả hai:
1. **Kiểm soát (từ ECC)**: Áp dụng luồng `Planner -> TDD -> Reviewer` cho mọi tác vụ bài giảng STEM để đảm bảo tính sư phạm.
2. **Luồng chảy (từ CodyMaster)**: Sử dụng các Skills chuyên biệt để giải quyết các khâu nặng nhọc (Hardware Map, Rubric generation).
3. **Bộ nhớ**: Chưng cất tri thức thô từ `brain/raw/` thành `brain/distilled/` nhưng vẫn giữ liên kết chặt chẽ qua **Shared Context**.

## 🛠️ Quy tắc thực thi cho Swarm
- **Research First**: Luôn nghiên cứu và lập kế hoạch trước khi code (ECC DNA).
- **No Manual Handoffs**: Agent phải tự động chuyển giao kết quả sang giai đoạn tiếp theo nếu tiêu chuẩn chất lượng đã đạt (CodyMaster DNA).
- **Verification Loop**: Ép AI phải tự đánh giá kết quả qua các Red Flags tại `ecc_prompts.md`.
