# CONCEPT: Karpathy Idea File (DRAFT)

## Core Principle (Raw Snippets)
- “The idea of the idea file is that in this era of LLM agents, there is less of a point/need of sharing the specific code/app, you just share the idea, then the other person's agent customizes & builds it for your specific needs.”
- Traditionally: share implementation (GitHub, npm).
- Agent Era: share the *idea* (portable). The idea is abstract/vague intentionally so agents can instantiate it for specific setups (Obsidian, VS Code, OS differences).
- “The document's only job is to communicate the pattern. Your LLM can figure out the rest.”

## Pedagogical Metadata
- Source: Karpathy's GitHub gist "LLM Wiki".
- Key shift: From Open Code to Open Ideas.
